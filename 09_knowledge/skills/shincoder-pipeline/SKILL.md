# Skill: shincoder-pipeline

## このスキルが解く問題

大井が「新事業アイデアどう?」「既存事業停滞してる、ピボット?」「PMF判定して」と言った瞬間、Claude-CEO がシンコーダー11ステップで即診断・即提案できる状態を作る。

毎回手動で 11ステップを思い出すのは非効率なので、スキル化して自動化する。

---

## 入力

- **事業名** (必須): AIpaX / AIpa Web / Testall / EEMUS / AIpaX school 等
- **判定したいこと** (必須): 「新規スタート」「ピボット判断」「PMF判定」「CPA測定」のいずれか
- **現状情報** (オプション): 既知の数字 (DL数、課金者数、広告費等)

---

## 手順

### A. 新規事業スタート時

1. `[[09_knowledge/frameworks/shincoder-11steps-roadmap]]` のチェックリストをコピー
2. ステップ1 (アイデア発掘) を大井に問いかけ
   - バズり仮説1行
   - 共感ターゲット具体的1人
   - 競合3-5社の差別化
3. ステップ2 (要件定義+LP) を OpenClaw に dispatch
   - 部署: newbiz (要件定義) + marketing (LP草稿)
4. ステップ3 (広告需要検証) の準備
   - Form Spree 設定
   - X広告/メタ広告 1万円ずつ準備
5. **5日後の CPA レビュー時間を Calendar に登録**

### B. ピボット判断時

1. 該当事業の現状を `[[01_ceo/decisions/2026-05-27-shincoder-for-5businesses]]` から引く
2. 11ステップのどこにいるか診断
3. 関連 KPI を測定:
   - CPA: 直近広告測定値
   - CVR: 直近 DL→課金率
   - 継続率: 30日継続率
4. 判定基準を当てはめ (各 learning 参照)
5. ピボット推奨 or 継続推奨を 1行で大井に伝える

### C. PMF判定時

1. 3シグナル (定量・訂正・経済) を測定
2. `[[09_knowledge/learnings/2026-05-27-pmf-3-signals]]` の閾値と比較
3. 3シグナルすべて達成 → PMF達成宣言 → Step 11 (スケール) へ
4. 1-2シグナル達成 → 残りシグナル達成のための具体アクションを提示
5. 0-1シグナル達成 → Step 9 (アップデート) ループ継続 or ピボット検討

### D. CPA測定時

1. 該当事業のLP確認 (Form Spree ウェイトリスト機能あるか)
2. X広告 1万円 + メタ広告 1万円 投化計画
3. 7日後の Calendar イベント設定 (CPA計算 + GO/PIVOT判定)
4. `[[01_ceo/decisions/cpa-log.md]]` に時系列記録 (新規ファイル必要なら作成)

---

## 出力

- 該当事業の **現ステップ番号** (1-11)
- **次の1手** (具体的なアクション + 期限 + 担当)
- 関連ファイルへのリンク (フレームワーク・PMF・CPA等)
- 該当部署の `_log.md` に追記
- 必要なら Calendar イベント・Gmail下書き作成

```

## 入力
- 事業名: {事業名}
- 現状 (既知数字): DL={数字}, 課金者={数字}, 広告費={数字}, CPA={数字}, 継続率={数字}
- 判定したいこと: {新規/ピボット/PMF/CPA}

## やること
1. `[[09_knowledge/frameworks/shincoder-11steps-roadmap]]` の11ステップに対し、この事業がどこにいるか判定
2. PMF/CPA 判定基準と照合
3. 次の1手を具体的に5項目以内で提示
4. ピボット推奨か継続推奨かを1行で結論

## 出力フォーマット
- 現ステップ: {1-11}
- 状態: ✅/🟡/❌
- 次の1手 (期限付き):
  - [ ] アクション1 (期限)
  - [ ] アクション2
- 判定: {GO/HOLD/PIVOT}
- 根拠: 1-3行

約1500字以内。
```
