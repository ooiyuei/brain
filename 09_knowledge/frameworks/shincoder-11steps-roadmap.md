# シンコーダー式 11ステップ・ロードマップ

> **Claude Code 時代に「アプリを作って売上を上げる」までの再現可能な11ステップ。**
> 大井の主力5事業 (AIpaX / AIpa Web / AIpaX school / EEMUS / Testall) に適用する標準フレームワーク。

---

## チェックリスト形式 (1事業に対して埋める)

### Step 1: アイデア発掘 ☐
- [ ] バズり仮説を1行で書ける? → ____
- [ ] 共感ターゲットは具体的に1人想像できる? → ____
- [ ] 競合3-5社の差別化ポイントは? → ____

### Step 2: 要件定義 + LP作成 ☐
- [ ] ChatGPT/Claude と要件壁打ち完了
- [ ] GPT image 2.0 で UI イメージ生成
- [ ] Claude Design で LP一気生成 (3分)
- [ ] **ウェイトリスト (Form Spree)** をLP末尾に配置
- [ ] Netlify Drop で公開

### Step 3: 広告で需要検証 ☐
- [ ] X広告 1万円分 投入
- [ ] メタ広告 1万円分 投入 (オプション)
- [ ] 期間: 7日間
- [ ] ウェイトリスト獲得数を集計

### Step 4: CPA計算 ☐
- [ ] CPA = 広告費 ÷ 獲得数 を計算
- [ ] 判定:
  - CPA 100-300円 → ✅ Step 5へ進む
  - CPA 400-500円 → 🟡 判断保留 (もう1回検証 or 進む)
  - CPA 1000円+ → ❌ Step 1に戻る (ピボット)

### Step 5: Claude Code で開発 ☐
- [ ] MVP機能を絞り込み (3-5機能)
- [ ] **課金機能を必ず含める** (バズった時の取りこぼし防止)
- [ ] 認証 + DB + 課金 → 1-2日で完成 (小規模アプリ)
- [ ] 大規模 (SNS等) → 1-2ヶ月
- [ ] **PostHog 導入** (将来のファネル分析のため必須)

### Step 6: SNS運用 (開発と並行) ☐
- [ ] X アカウント (新規 or 既存) を運用開始
- [ ] 関連アカウントに「いいね回り」「コメント」(日次)
- [ ] ビルドインパブリック発信 (週3-5本)
- [ ] フォロワー牽引性のため判断 (フォロー戦略 or しない)

### Step 7: MVP ローンチ ☐
- [ ] 「完璧」より「速度」優先
- [ ] 課金機能ON
- [ ] App Store / Web 公開
- [ ] 既存SNSフォロワーへ告知

### Step 8: 分析 (課金されたか) ☐
- [ ] DL数 確認 (目標 200DL以上)
- [ ] 課金者数 確認
- [ ] CVR = 課金者 ÷ DL を計算
- [ ] 判定:
  - CVR 1%以上 → Step 9 アップデートへ
  - CVR 0% (200DL中) → 1週間アップデート → 改善なし → ピボット

### Step 9: アップデート (Step 8と並行ループ) ☐
- [ ] PostHog でファネル分析
- [ ] 離脱ポイント特定
- [ ] UI/UX 改善 (週次)
- [ ] 機能追加/削除 (週次)
- [ ] **3週間ループしても CVR 上がらない → ピボット検討**

### Step 10: PMF達成シグナル確認 ☐
詳細: [[09_knowledge/learnings/2026-05-27-pmf-3-signals]]

#### 定量
- [ ] 継続率 15%+ / 30日
- [ ] CVR 3%+
- [ ] オーガニック成長 (広告止めて増える)

#### 訂正
- [ ] Sean Ellis Test: 「なくなったら残念」40%+
- [ ] 紹介・口コミ自然発生

#### 経済
- [ ] LTV ≧ CAC × 3 (粗利ベース)

### Step 11: スケール / バイアウト / 連続起業 ☐
- [ ] マーケ強化 (広告増額 / インフルエンサー)
- [ ] 月500万超 → チーム化検討
- [ ] バイアウト検討 (連続起業派の場合)
- [ ] 連続起業 → Step 1へ戻る

## 黄金ルール (シンコーダー語録)

1. **PMF達成前は広告費かけない** (湿った木材に火つけても無駄)
2. **完璧主義 = 現実逃避** → 4-5日でMVP出す
3. **競合いる理由 = 需要証明** → 隙間狙い1機能特化
4. **CPA 300円超は即ピボット検討** → 8割の事業で発生
5. **継続率 < 15% = PMF未達** → 改善2-3週間で出ない = ピボット
6. **「アイデアあるけど忙しくて」は嘘** → 即LP作って24h以内検証
7. **「自分で完璧に」は罠** → Claude Code に丸投げが正解
8. **月収100万 = 「1日約3万円」分解** → 1事業で達成は無理・5事業合算で

## このフレームワーク使用時の Claude-CEO 動作

1. 大井から新事業案が出た時 → このチェックリストを即提示 (5分でステップ1-4の準備が整う)
2. 既存事業のピボット判断が必要な時 → 該当事業の現状を このチェックリストでスキャン
3. 月次レビュー (日曜21時) で 主力5事業の進捗をこのフレームワークで一括レビュー
