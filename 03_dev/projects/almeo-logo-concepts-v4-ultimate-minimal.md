# Almeo ロゴ v4 — 究極の引き算

> **大井確定 (2026-05-27)**:
> - シンプルを極めたい
> - とにかく美しい
> - ごちゃごちゃしてない
> - **究極の引き算**
>
> → v3 の7案・武器・サイバー要素は全部破棄。**「1要素以下」**まで削る。

---

## §1. 引き算の哲学 (大井判断軸への翻訳)

### Less, but better (Dieter Rams)
- 1要素削るたびに、残った要素の強度が増す
- 何も足さない = 何も引けない = 完成

### 究極の引き算 = 答えを2つ持つ
1. **完全プレーン**: 装飾ゼロ・文字だけ
2. **1点だけ追加**: 終止符・モノグラム

### 7案は引き算ではない (v3 の反省)
- 大井が「ごちゃごちゃしてない」と言った時点で、選択肢自体を絞るのが引き算
- v4 は **3案だけ**。これ以上は要らない。

## §2. v4 ロゴ 3案 (究極の3)

---

### 🥇 v4-A. **`almeo.`** ⭐ Claude 最終推奨

#### コンセプト
- 小文字 `almeo` + 末尾にピリオド **1点だけ**
- ピリオド = 「これで終わり」「説明はいらない」「断言」
- Apple の `Think different.` の系譜
- 1点だけ追加した時、その1点が **無限の含意** を持つ

#### 視覚仕様
```
┌──────────────────────────────────┐
│                                  │
│         almeo.                   │
│                                  │
└──────────────────────────────────┘

フォント:    Inter Display Bold (or Suisse Int'l Bold)
ウェイト:    700
字間:       -0.025em (緊密)
カラー:     #1a1a1a (純黒)
背景:       #ffffff (純白)
ピリオド:   メインカラー同じ・サイズ同等
余白:       ロゴ周辺・最低 ロゴ高さ × 2倍
```

#### 強み (究極引き算)
- ✅ **1点だけ追加で完成** (これ以下も以上も削れない)
- ✅ ピリオド = 終止符 = 「断言する」哲学
- ✅ 小文字の `al` ≈ `ai` 視覚的曖昧化 (大井元アイデア保持)
- ✅ favicon は `.` 単独 (1点) で実装可
- ✅ Apple / Aesop / Acne 系の系譜 (品の極致)
- ✅ 「説明しないけど存在感ある」 = 大井の戦闘姿勢と完璧合致

#### 弱み
- ⚠ ほぼ無い (これより削ると意味が消える)

#### 実装コード

```tsx
// components/ui/Logo.tsx (究極シンプル版)
export function Logo({ size = 'md' }: { size?: 'sm' | 'md' | 'lg' }) {
  const sizeClass = {
    sm: 'text-xl',
    md: 'text-3xl',

  }[size];

  return (
    <span 
      className={`font-bold tracking-tight text-black ${sizeClass}`}
      style={{ 
        fontFamily: "'Inter Display', 'Inter', sans-serif",
        letterSpacing: '-0.025em',
      }}

      almeo.
    </span>
  );
}
```

## §3. 削った要素 (なぜ引き算が美しいか)

### v3 の何を捨てたか
| v3 で入れた要素 | v4 で削った理由 |
|---|---|
| `>` プロンプト記号 | 「説明的すぎる」「ハッカー感は引き算じゃない」 |
| `_` カーソル | 「動的な装飾」「静止していない」 |
| `▌▐` ステンシル | 「武器要素は装飾」 |
| `Λ` ラムダ | 「暗号は引き算じゃなく加算」 |
| `/` スラッシュ | 「終止符より弱い」 |
| 7案あること自体 | 「選択肢の多さが既にごちゃごちゃ」 |

→ **「引き算」は記号も選択肢も両方削る**
