# FishOps Agent

> 釣り場・釣具屋・漁協・養殖場に散らばっている経験と勘をデータ化し、AIエージェントで運営改善する。  
> 日本の水産・釣り現場に **AI運営者を入れる事業**。

## ピボット履歴

- **初期構想（IPAK fishing）**: 釣り版チョコザップ。釣具レンタル・サブスク・港前釣具屋リメイク
- **進化**: 釣具サブスク単体は「熱量高い人には不要、低い人には重い」谷間に落ちると判断
- **現在（FishOps Agent）**: 単なるサブスクではなく、**水産レジャー・水産現場のAIオペレーション基盤**へ

## 事業の本質

| 段階 | 言い方 |
|------|-------|
| わかりやすい | 釣り場・釣具屋の運営をAIで改善するサービス |
| 強い | 勘で回していた釣り場運営を、データとAIで進化させる |
| エージェント時代 | 日本中の釣り場と釣具屋に、AI店長を配る |
| 大きいビジョン | 水産現場に散らばる経験と勘をAIでつなぎ、釣れる体験と地域資源の活用を最適化する |
| 一番強い | 釣果・来場・購買・放流・満足度を統合し、AIが釣り場と水産現場の運営を改善する「FishOps Agent」 |

## 根本課題

### 釣り初心者
何を買えばいいか/どこに行けばいいか/何が釣れるか/仕掛けの作り方/釣り場ルール/釣れなかった時の対処/成功体験なし  
→ 売るべきは **失敗せずに釣りを楽しめる体験**

### 釣具屋
価値は商品在庫ではなく「立地/地元情報/常連関係/漁協・船宿とのつながり/初心者相談/暗黙知」  
課題：後継者なし/高齢化/SNS・Googleマップ弱い/大手ECに価格負け/常連依存/初心者・観光客取り込めない/情報が店主の頭の中

### 管理釣り場
運営が経験と勘に依存（放流量/時間帯/初心者案内/原価/満足度/イベント効果）  
→ データを取りやすい閉じた空間。最適化の入口に最適。

### 水産業界全体
高齢化/人手不足/後継者不足/経験と勘依存/暗黙知のデータ化不足/DX遅れ/初心者参入障壁/地域資源活用不足/放流・養殖の効果検証難/**釣れなかったデータの不足**

## 主要プロダクト（5本）

### 1. 釣り場AI店長（管理釣り場/釣り堀/釣り体験施設）

**機能**: 予約確認・天気/水温・来場・初心者比率・放流数・釣果写真・釣れなかった記録・満足度・放流効率・売上原価・SNS自動投稿・翌日タスク提案・週次月次レポート

**価値**: 「魚をたくさん入れる」ではなく「**満足度と利益が最大になる放流量を見つける**」

**売り文句**: 勘で決めていた放流と接客を、データで改善するAI店長

### 2. 港前釣具屋AI店長（古い釣具屋/後継者不足店）

**機能**: 今日の釣れやすい場所提案/初心者向け仕掛けセット/店頭POP/LINE配信/SNS投稿/Googleマップ改善/釣果写真収集/釣れなかったデータ/売れ筋分析/餌・仕掛け発注/常連イベント/手ぶら釣りパック支援/月次売上改善

**価値**: 「商品を売る店」から「**地域の釣り体験ステーション**」へ

**売り文句**: 古い釣具屋を、AI時代の釣り体験ステーションに変える

### 3. 漁協AI事務局（漁協/自治体/遊漁管理団体）

**機能**: 遊漁券管理/釣果収集/放流効果レポート/ルール案内/釣り場情報発信/SNS・HP更新/問い合わせ対応/補助金資料/行政報告書下書き/観光施策効果測定/違反・混雑可視化

**売り文句**: 人手不足の漁協に、AI事務局を

### 4. 養殖AI副操縦士（小規模養殖/陸上養殖/管理釣り場の魚育成）

**機能**: 水温・酸素・pH記録/給餌ログ/死亡数/成長記録/異常アラート/餌コスト/出荷予測/作業日報/現場指示/ベテラン判断のログ化

**思想**: 「全自動」ではなく「人間判断を強化する副操縦士」

**売り文句**: ベテランの勘をデータ化し、若手でも養殖運営できるように

### 5. 釣り観光AIコンシェルジュ（自治体/観光地/宿泊/船宿）

**機能**: 観光客向け釣りプラン/手ぶら釣り予約/親子向け体験提案/送客/天候別代替プラン/釣果写真→思い出レポート/地域クーポン連携/宿泊飲食連携

**売り文句**: 地域の釣り資源を、AIが観光商品に変える

→ 静岡（海/川/観光/飲食あり）と相性最高

## 新規性の核

1. **釣り場・釣具屋にAI運営者を入れる**（エージェント時代の構造）
2. **釣れなかったデータを取る**（改善点は失敗にある）
3. **現場データを水産資源管理につなげる**（将来：放流/養殖最適化）

## 競合との違い

| 競合 | 立ち位置 | FishOpsとの違い |
|------|--------|---------------|
| アングラーズ | 個人向け釣果SNS | 運営側＋釣れなかったデータ |
| 予約アプリ | 予約・決済 | 予約後の体験/釣果/満足度/再来店まで改善 |
| 釣具屋DX | LINE/SNS/マップ化 | 地域の釣りデータが溜まる体験ステーション化 |

## 初期MVP（軽く始める）

### MVP1：管理釣り場AIレポート
スタック：Googleフォーム/LINE公式/スプレッドシート/Claude/Looker Studio/釣果写真アップロード  
取るデータ：来場/初心者比率/レンタル/放流/魚種/釣果写真/釣れた数/釣れなかった人/満足度/また来たいか/困ったこと  
出すもの：今日の釣果レポート/釣れなかった人の分析/放流効果分析/明日の運営提案/SNS投稿案/週次レポート

### MVP2：港前釣具屋AI店長

### MVP3：釣り体験イベント
10人集めて検証：何に困るか/いくら払うか/手ぶら需要/リピート/不満ポイント/サポート価値/写真投稿

## 市場データ

| 指標 | 数値 |
|------|------|
| 国内釣り人口（2022年） | 520万人（レジャー白書） |
| 釣具市場（2024年見込み） | 1,516億円 |
| 釣り人口減少（2017→2022） | 約120万人減 |

→ 市場は緩やかに縮小。だがFishOpsは「市場を拡大する」ではなく「**既存施設の運営効率を上げる**」事業なので、市場縮小と相関しない。

## リスクと対策

| リスク | 対策 |
|--------|------|
| 市場ニッチ（管理釣り場SaaS単体だと天井低い） | 釣具屋/船宿/漁協/自治体/観光/養殖/放流へ広げる |
| 現場がITに弱い | 「AI店長が毎週レポート出す。LINEと紙でも見れる。操作不要」レベルに落とす |
| データ入力面倒 | 入力項目を絞る（来場/放流/釣果写真/満足度のみ） |
| 釣果データの精度 | 個人投稿だけに頼らず店舗側データと組み合わせ |
| 買収は重い | 提携/運営改善/レベニューシェア/共同事業から入る |

## 最初の具体アクション

1. 管理釣り場ヒアリング（放流量決定法/釣果記録/釣れなかった人把握/SNS/原価/月3万円で使うか）
2. 釣具屋ヒアリング（客減少/初心者/若者/SNS/釣果情報/売れ筋/利益率/後継者/手ぶら釣りセット）
3. 初心者釣り体験10人実施
4. v0制作（管理釣り場AI店長v0 + 港前釣具屋AI店長v0）

## IPAKシリーズとの関係

旧IPAK fishingは「IPAKシリーズの一員」だったが、FishOpsへ進化したことで **IPAK（おひとり様レジャー）の枠組みを超えた水産AI基盤** になった。  
→ IPAK-cafe / IPAK-darts / IPAK-ofuro とは事業ロジックが切れる。
