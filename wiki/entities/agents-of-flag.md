# Agents of Flag (AOF)

> 光線銃を使った新しいリアルFPSスポーツ。  
> サバゲーの興奮を **痛み・装備差・自己申告なし** で再現し、FPSのランク・戦績・チーム・大会・観戦の中毒構造をリアル空間に持ち込む。  
> **目指すのは、世界を熱狂させるリアルeスポーツ。**

**フェーズ**: 企画書v2.0 / PoC準備中  
**オーナー**: 大井湧瑛

## 原体験

小中学生の頃、静岡の光線銃サバゲー施設に通い詰めた。
光線銃には人を夢中にさせる力があると確信。
同時に、既存光線銃が競技・文化・事業になりきれていない理由も見えていた。

**素材はいいのに、競技としての設計が足りていない**。AOFはここを本気で作り直す。

## 解決する課題

### サバゲーの問題
痛い/装備差/場所遠い/自己申告/常連強い/ミリタリー色/観戦しにくい/戦績残らない  
→ FPSゲームならキルログ/リザルト/ランク/MVP/ハイライトがあるが、リアルサバゲーにはない  
→ 「一回遊ぶ体験」止まりで「継続して競う競技」にならない

### 既存光線銃の問題
痛くない・安全・センサー判定・屋内可・公平化できる**はず**だが…  
撃った感触弱い/発射音しょぼい/ヒット音気持ちよくない/振動弱い/光演出弱い/ルール単純/チーム戦感弱い/ランクなし/戦績残らない/大会文化なし

## 基本思想

**銃を作る事業ではない。店舗を作る事業でもない。本質はリアルFPSを成立させる「競技OS」と「競技IP」を作ること。**

統合が必要：
非発射型銃型デバイス / 頭・胴体センサー / A/Bフラッグ制 / 音・振動・LED演出 / アプリ / QRチェックイン / 試合管理 / 観戦UI / 戦績保存 / ランク / チーム・クラン / 月例大会 / スポンサー / 施設導入モデル

## ゲームシステム

- **5vs5**、A/Bフラッグ制（CS:GO的）
- 3点先取ラウンド制
- HP2（頭ショット2ダメ即ダウン / 胴体1ダメ）
- ロール別スコア：Combat / Flag / Defense / Support / Clutch

### ルール種別

| ルール | 内容 |
|--------|------|
| **Formal** | AまたはBを先に3点先取で勝利。攻守3ラウンド交代。**防衛側が3回連続防衛成功で即勝利**（Defense Lock） |
| **Swift** | A/Bあり。どちらかを2点先取。OTなし（通常営業向け） |
| **Casual** | A/B区切りなし。5レグ/3レグ先取（初心者向け） |

### 少人数モード

- 1v1：**Duel**（タイマン、待機中ミニゲーム）
- 2v2：**Mini Strike**（4人で遊べる）
- 3v3：**Swift Core**（通常営業の主力候補）

### フラッグドロップルール

保持者ダウン → その場にドロップ。直接手渡し禁止。  
→ フラッグ周辺に自然と戦闘が生まれる。観戦者にも「拾うのか守るのか」の見どころ。

## フィールド設計

MOBA/FPS的：トップ/ミッド/ボット/Aフラッグ/Bフラッグ/横移動接続路/中央取ると両方に圧。  
シーズンごとにマップ変更可能（常連が飽きづらく、戦術研究が生まれる）。

## デバイス：AOF専用3Dプリント筐体

**既存エアガン外装ではなく専用筐体**を作る理由：
- ミリタリー感を残すと既存光線銃と差別化できない
- AOFが目指すのは武器ではなく **スポーツギア**
- サバゲー銃ではなく、リアルFPSの公式コントローラー

機能：赤外線発信 / トリガー / 発射音 / 胴体ヒット音 / **ヘッドショット音** / ダウン音 / 復活音 / 振動モーター / ハプティック / チームカラーLED / ダウン時赤表示 / 小型ディスプレイ / QR・端末ID / バッテリー表示 / 通信状態 / 無線通信 / 観戦UI連携

ミニHUD：HP / チーム / READY / LIVE / DOWN / RESPAWN / HIT / HEAD / バッテリー / 通信状態

## 音・振動・光（脳汁設計）

**既存光線銃の最大の弱点**：撃った感触が弱い。  
AOFでは音と振動を **報酬設計** として扱う。

| イベント | 演出 |
|---------|------|
| 発射 | 短く気持ちいい音＋軽い振動 |
| 胴体ヒット | シールド割れる音 |
| **ヘッドショット** | 重く短い決定音＋強い振動（脳汁ポイント） |
| ダウン | 長めの振動 |
| 復活 | 明るい短い振動 |
| エラー | 弱い断続振動 |

**ヘッドショット瞬間**：銃が強く震える → 相手ヘッドバンドが赤く光る → 観戦UIに「HEADSHOT」 → キルログ表示 → 会場音

## 赤外線・ヒット判定

技術：狭角赤外線LED＋遮光筒＋（必要に応じレンズ/コリメータ）＋受信側遮光カバー＋信号ID＋ショットID＋同チーム無効＋ダウン中無効＋重複排除

判定の分担：
- 銃 = 「撃った」を検知
- センサー = 「当たった」を検知
- サーバー = 「試合結果として有効か」を確定

## フラッグドック（AOF独自）

**磁石極性による判定方式**：

| Flag | 磁極 |
|------|------|
| A Flag | N極下向き |
| B Flag | S極下向き |
| 台座 | 極性判別ホールセンサー＋マイクロスイッチ |

判定：
- スイッチON + N極 = A Flag inserted
- スイッチON + S極 = B Flag inserted
- スイッチON + 磁場なし = Error

サーバー確認：試合中か / 保持中か / 保持者誰か / 生存中か / 得点可能か / 二重処理ではないか

観戦UI：
- 通常：「FLAG A CAPTURED / BLUE +1」
- リーチ：「BLUE A FLAG - MATCH POINT」
- 勝利：「BLUE WINS BY FLAG A」

→ フラッグを刺す行為が「ただの得点」ではなく **会場全体が反応する儀式** になる。

## 観戦UI

表示：BLUE/ORANGE / プレイヤー名 / ランク / HP / 生存・ダウン / A・Bポイント / フラッグ状況 / キルログ / HEADSHOT / FLAG DROPPED / Defense Lock / 残り時間 / MVP候補 / 次試合案内 / スポンサー

**脳汁演出ワード**：
HEADSHOT / ACE / CLUTCH / LAST AGENT STANDING / FLAG DROPPED / FLAG CAPTURED / MATCH POINT / DEFENSE LOCK 2/3 / COMEBACK CHANCE / HERO MOMENT

> 観戦UIの目的：待機時間を退屈ではなく、次の試合への熱狂を高める時間に変える。

## データ・リザルト・クリップ

イベント記録：SHOT / HIT / HEADSHOT / BODY HIT / DOWN / RESPAWN / FLAG PICKED / FLAG DROPPED / FLAG CAPTURED / DEFENSE SUCCESS / ROUND END / MATCH END

リザルト：勝敗 / キル / デス / ヘッドショット数 / フラッグ取得 / フラッグ防衛 / 防衛成功 / MVP / チーム貢献

**クリップ自動生成**：HEADSHOT発生 → 前後10秒切り出し / FLAG CAPTURE → 該当ゾーンカメラ切り出し → アプリで見返し / SNSで共有  
→ サバゲーで残せない瞬間が、AOFでは残る。継続率と拡散に効く。

## ランク・戦績・チーム

**ロール別スコア（K/Dだけで評価しない）**：

| スコア | 内容 |
|--------|------|
| Combat | 撃ち合いの強さ |
| Flag | 取得・運搬・刺し込み |
| Defense | 防衛・リーチ阻止 |
| Support | カバー・味方支援 |
| Clutch | 重要局面の活躍 |

→ 撃ち合いが強くなくても、運んだ人・カバーした人・止めた人・最後に刺した人が活躍できる

チーム/クラン：名前 / ロゴ / メンバー / 戦績 / ランキング / 大会履歴 / ライバル

継続ループ：**遊ぶ → 記録 → 所属 → 競う → 観戦される → 称えられる → 戻ってくる**

### 最重要な姿勢

熱量だけで押し切らない。**世界を取りたい熱量** と **まず小さく検証する現実感** の両方を持つ。

> 僕は本気で世界を熱狂させるリアルeスポーツを作りたいです。  
> ただ、いきなり大きく作るのではなく、まずは御社の知見や既存機材を活かしながら、小さくPoCを作って、本当に人が熱狂するか検証したいです。
