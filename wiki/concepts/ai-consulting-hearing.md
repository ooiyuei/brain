# AIコンサル ヒアリング手法（完全版）

> 相手の業務を聞いて、AIツールとして提案できる形に変換するための確認資料。
> 「どんなAIが欲しいか」を考えさせるのではなく、今の業務・記録・保存先・困りごとを聞き、こちらで商品案に変換する。

## コア原則

- **相手にAI商品を設計させない** — 聞くのはAIの仕様ではなく、今の業務
- **商品名から入らない** — 「営業後処理パック」などの商品名で聞かない。現場で起きていることを先に聞く
- **AIの話は最初にしない** — まず業務を理解することに集中
- **「困っていること」より「時間を使っていること」を聞く** — 課題に気づいていないケースが多い
- **具体的なエピソードを引き出す** — 「先週のこの作業で何時間かかりましたか？」
- **現場担当者と経営者の両方** — 温度差が重要な情報

## ヒアリング10セクション（詳細）

### 2-1. 面倒な業務・軽くしたい業務

**聞くこと:** 今、繰り返し発生していて、記録・返信・共有・確認などが面倒な業務は何か

**聞き方の例**
- 普段の業務で、毎回発生していて面倒な作業はありますか？
- 記録、返信、確認、共有、引き継ぎあたりで時間を取られているものはありますか？
- 人によって対応の質が変わる作業はありますか？
- 新人がやるとミスりやすい作業はありますか？

**判断できること:** AIツール化する価値があるか / どの業務カテゴリに近いか / 個別深掘りが必要か

---

### 2-2. 今の業務の流れ

**聞くこと:** その業務が今どんな順番で行われているか

**聞き方の例**
- 今はその作業をどんな流れでやっていますか？
- 最初に何を見て、次に何をして、最後にどこへ残していますか？
- その作業のあと、誰かに共有したり、次の作業につながったりしますか？

**判断できること:** AIをどこに差し込めるか / 入力と出力の位置 / 保存先 / 単体フローか、パックにした方がよいか

---

### 2-3. 誰が、何を、どこに記録しているか

**聞くこと:** 今誰が、何を、どこに記録しているか。記録した情報を誰が見て、何に使っているか

**聞き方の例**
- その記録は誰が書いていますか？
- 何を記録していますか？どこに残していますか？
- その記録はあとで誰が見ますか？記録した情報は、次に何に使われますか？

**判断できること:** 保存先 / 利用者 / 見る人 / 出力形式 / 管理系商品に発展しそうか

---

### 2-4. AIに入れられそうな元データ

**聞くこと:** AIに入れられそうな元データがあるか

**聞き方の例**
- その作業の元になる文章や記録はありますか？
- 文字起こし、メール、メモ、一覧、FAQなどは残っていますか？
- 今ある情報をそのままAIに入れられそうですか？

**候補:** 会話文字起こし / 通話メモ / 問い合わせメール / フォーム送信内容 / チャット履歴 / 案件メモ / スプレッドシート / Excel / CRMの記録 / kintoneのレコード / FAQ / マニュアル / 過去返信文 / 日報 / まだ残していない

**判断できること:** 入力起点 / AI化のしやすさ / 商品化できるかどうか

---

### 2-5. 今の保存・管理先

**聞くこと:** その業務の情報を今どこに保存・管理しているか

**聞き方の例**
- その情報は今どこに残していますか？
- スプレッドシート、Excel、CRM、kintoneなどに残していますか？
- チャットやメールに流れて終わっていますか？

**候補:** Google Sheets / Excel / kintone / Notion / Salesforce / HubSpot / Zoho CRM / Google Drive / SharePoint / Gmail / Outlook / Slack / Chatwork / LINE WORKS / Microsoft Teams / 紙・個人メモ

**判断できること:** 保存連携先 / 連携難易度 / 保存まで商品に含めるか / 管理系商品に発展しそうか

---

### 2-6. 困っていることの種類

**聞くこと:** その業務のどこに困っているか

**聞き方の例**
- その作業で一番面倒なのはどこですか？
- 時間がかかる部分はどこですか？
- 人によって品質が変わるところはありますか？

**候補:** 時間がかかる / 毎回書くのが面倒 / 共有が遅れる / フォロー漏れがある / 人によって品質に差がある / 上司が全体を把握しにくい / 保存先はあるが見返しづらい

**判断できること:** 出力タイプ / 価値の出し方（時間削減型・品質安定型・管理改善型）

---

### 2-7. AIから出せると便利そうなもの

**聞くこと:** AIから何が出ると使えそうか（候補を見せて絞る）

**聞き方の例**
- 必要そうなものを3つまで選ぶならどれが近いですか？
- これは後回しでいいものはありますか？

**出力タイプ6分類**

| タイプ | 内容 | 候補 |
|--------|------|------|
| **A. 整理系** | 長い情報を見やすくする | 要約、重要ポイント、決定事項、未決定事項、顧客要望 |
| **B. 行動系** | 次にやることを明確にする | 次アクション、確認すべきこと、担当者、期限、抜けている対応 |
| **C. 文面系** | そのまま使える文章を作る | フォロー文面、一次返信文、上司報告文、引き継ぎ文 |
| **D. 判定系** | 優先度や種類を判断する | 緊急度、優先度、担当振り分け、クレーム度合い、上司確認要否 |
| **E. チェック系** | 不足や危ない点を見つける | 不足情報、ヒアリング漏れ、NG表現、ルール違反 |
| **F. 管理系** | 保存済み情報を管理しやすくする | 放置案件一覧、今日見るべき案件、管理者向け日次レポート |

**判断できること:** ベースに含める出力 / 追加に回す出力 / パック向きかどうか / 管理系商品に発展しそうか

---

### 2-8. AIに任せる範囲

**聞くこと:** 最初は、人が確認して使う形で問題ないか。通知や下書き、自動実行まで必要そうか

**自動化段階（1〜6）**
1. 整理だけ（要約や記録を出す）
2. 下書きまで（メール文面や返信文を作る）
3. 保存まで（スプレッドシートやCRMなどに書き込む）
4. 通知まで（Slack、Chatwork、Teamsなどに通知する）
5. 人が承認して実行（人が確認したあと送信・登録する）
6. 条件付きで自動実行（決めた条件を満たした時だけ自動送信・自動通知する）

**判断できること:** 標準範囲で足りるか / 通知・下書きが必要か / 実行系として高めに見るべきか

---

### 2-9. 頻度・量・時間感

**聞くこと:** その業務がどれくらい発生しているか。1件あたりどれくらい時間がかかっているか（ざっくりでよい）

**聞き方の例**
- その作業はどれくらいの頻度で発生しますか？
- 月に何件くらいありますか？1件あたり、だいたい何分くらいかかっていますか？

**候補**
- 頻度: 毎日 / 週数回 / 週1回 / 月数回 / 案件ごと
- 件数: 月10件未満 / 10〜30件 / 30〜100件 / 100件以上
- 時間: 5分未満 / 5〜10分 / 10〜30分 / 30分以上

**判断できること:** 価値の大きさ / 価格説明のしやすさ / 優先して提案すべき業務か

---

### 2-10. 最初に使う人・見る人

**聞くこと:** 最初に誰が使い、誰が見るのか

**聞き方の例**
- 最初に使うとしたら、誰が使いそうですか？
- 現場担当だけですか？リーダーや上司も見ますか？
- 使う人数はどれくらいになりそうですか？

**候補:** 営業担当 / 営業リーダー / 新人 / 問い合わせ担当 / 管理者 / チーム全体

**判断できること:** 画面の複雑さ / 管理系が必要か / 権限やログインが必要か

## こちらで判断する基準

### 3-1. AI化しやすさ

| 判定 | 条件 |
|------|------|
| **高い** | 元データあり + 繰り返し発生 + 出力の形が見える + 保存先もあり + 困りごとが明確 |
| **普通** | 元データはあるが出力がまだ曖昧 / 業務の流れはあるが追加確認が必要 |
| **低い** | 元データなし / 単発作業に近い / 出力の形が見えない / AIを入れる位置が分かりにくい |

### 3-2. 入力起点 → 商品系統

| 入力起点 | 商品系統 |
|----------|----------|
| 会話文字起こし | 営業後処理、新人レビュー、引き継ぎ |
| 問い合わせ文 | 一次返信、振り分け、緊急度判定 |
| 案件一覧 | 追客漏れ、管理者向けまとめ、優先度整理 |
| FAQ / マニュアル | 返信テンプレ、NG表現チェック、社内ルール確認 |

### 3-3. 出力セット

- ベースに含める出力: **2〜3個に絞る**
- ベースに入れやすい: 要約 / 社内記録 / 次アクション / 保存用メモ / 不足情報の簡易チェック
- 追加に回しやすい: フォロー文面 / 一次返信文 / 緊急度判定 / NG表現チェック / 通知文
- 管理系に分けやすい: 放置案件一覧 / 今日見るべき案件 / 管理者向け日次レポート

### 3-4. 保存・連携先の難易度と価格感

| レベル | 対象 | 価格感 |
|--------|------|--------|
| **低** | Google Sheets / Slack / Chatwork / Google Chat | ベースに含めやすい / +10〜20万円 |
| **中** | Notion / kintone / Gmail下書き / LINE WORKS / Teams / Excel on SharePoint | 単体50〜90万 / +20〜40万円 |
| **高** | Salesforce / HubSpot / Zoho CRM / 社内独自システム / 紙・PDF中心 | 単体80〜150万以上 / パック120〜250万以上 |

### 3-5. 商品タイプの仮判定

| タイプ | 条件 | 価格目安 |
|--------|------|----------|
| **A. ベース商品** | 1業務 / 出力2〜3種 / 保存先1つ / 自動送信なし / 管理系なし | 40万円〜 |
| **B. 追加モジュールあり** | 文面も欲しい / 判定も欲しい / チェックも欲しい / 通知や下書きも必要 | +20〜30万/出力 / +10〜40万/通知系 |
| **C. パック向き** | 同じ入力から複数出力が必要 / 最初から保存まで含めた方がよい | 80〜100万以上 |
| **D. 管理系商品に発展** | 保存済み情報から一覧やレポートを出したい / 上司が全体を見たい | 40〜150万以上 |

## 持ち帰る情報の最小セット（10項目）

1. どの業務か
2. 今どういう流れか
3. 誰が何を記録しているか
4. 元データは何か
5. 保存先はどこか
6. 困っていることは何か
7. 最初に必要そうな出力は何か
8. AIに任せる範囲はどこまでか
9. どれくらいの頻度・量か
10. 誰が使うか、誰が見るか

## Go/Hold/No-Go 判断

| 判定 | 条件 |
|------|------|
| **Go** | 課題明確 + データ存在 + 意思決定者参加 |
| **Hold** | 課題あるが現場/経営の優先度ズレ or データ散在 |
| **No-Go** | 業務フロー未整備 or AIより先にやることあり or 不信感強い |
