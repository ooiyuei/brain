# 大井湧瑛 深層プロファイル

> これは診断ではなく、これまでの会話・事業案・悩み方・言葉選びから見える思考パターンの仮説集。

---

## 0. 一番深い核

**「自分は普通に終わる人間じゃないと証明したい」**

金を稼ぎたいだけでも、有名になりたいだけでもない。
自分の人生が、ただの不運や傷や停滞で終わらないことを、事業・名前・成果で証明したい。

→「売上」「有名」「静岡で1番」「世界のユウエイ」「ビジコンを荒らす」「テレビ」「スーツ」「カリスマ」みたいな言葉に強く反応するのはこれ。
**浅い承認欲求だけじゃなく、人生を取り返すための証明欲求。**

## 1. 思考の大きな癖

### 1. 美しい構造に異常に反応する
「儲かりそう」より先に、事業構造が美しいかを見る。好きな構造：
入力がある → 変換する → 行動に落ちる → データが貯まる → 改善される → ネットワーク化する → 文化になる

### 2. 「点」ではなく「OS」として考える
- Testall = 受験OS / AOF = リアルFPS OS / Almeo = 企業情報OS
- 何を見ても「単体ツールではなく、全体の運用基盤にできないか？」と考える
- **ただし最初からOSを作ろうとして重くなりがち。まず電卓から作れ。**

### 3. 「誰の、どんな痛みか」に戻ると強い
抽象概念より、人間のダサい瞬間を掴んだ時に強い。
（例：「点数を見るのが怖い」「答えを見て分かった気になる」「スマホ触って2時間消える」）

### 4. 綺麗な理想より汚い現実が好き
綺麗な教育論より「ワーク全部丸で満足してんじゃん」みたいな現実の情けない瞬間に価値を感じる。**人は本音を言語化された時に刺さる。あなたはそこを掴むのがうまい。**

### 5. 「英雄感」に異常に価値を置く
自分が強くなった感じ・主人公感・成り上がり感・ランクアップ・称号・見られる・記録が残る・誰かに認められるに強く反応する。**人を主人公にする構造が入ると強い。**

### 6. 事業を"メダルゲーム"として見る
資源配分・撤退ライン・勝てる盤面・持ち玉の余裕を本能的に気にする。
**危険：テンションが上がると「この台、絶対出る」と思い込みやすい。**

### 7. 「勝ち筋」という言葉が好き
勝ち筋が見えた瞬間にすでに勝った気になる。
**正しい変換：「勝った」→「勝てる仮説を見つけた。証拠を積む。」**

## 2. 好みの傾向

### 8. 例え話が好き
パソコン・メダルゲーム・OS・右腕・AI社員・武器・刀・焼け野原。
概念を絵と構造で掴む。Obsidianは文章より「事業をゲーム盤にする」「自分をPCスペックで見る」整理が合う。

### 9. "かっこよさ"をかなり重視する
事業は便利なだけでは足りない。名前・ロゴ・語感・世界観・スーツ・カリスマ全部大事。
**正解：かっこよく見せるために盛ることではなく、途中経過ごとかっこよく編集すること。**

### 10. 「無双」「ぶち抜く」「焼け野原」系の言葉が燃料になる
**使い分け：外向き＝社会課題を変える言葉 / 内向き＝自分を燃やす戦闘語**

### 11. "Show, Don't Tell" が好き
画面・デモ・プロトタイプ・動画・事例・数字・Before/Afterがあると燃える。

### 12. "データでサイエンスする" が好き
感覚型に見えてかなりスコア化・構造化・比較が好き。カオスな発想をスコアとDBに落とすと急に強くなる。

## 3. 強い思考パターン

### 13. 本人性のある事業に強い
Testall（高校生のリアル）/ AOF（英雄感）/ EEMUS（高校生の実務機会）。
**EcoKanやフルーツソースは面白いが、あなた自身の切実さとは少し距離がある。**

### 14. "弱者の違和感"を事業化できる
既存の仕組みの間に落ちている人を見つけるのが得意。

### 15. "既存カテゴリの再定義"が好き
塾ではない→受験OS / AIツールではない→企業の右腕 / 光線銃ではない→リアルFPSスポーツ。
**市場を作るタイプの発想。新しいものをゼロから作るより、既存カテゴリをズラして新しい名前をつけるのが得意。**

### 16. "顧客の行動が変わる瞬間"に反応する
「便利です」だけでは満足しない。人の行動が変わる瞬間に価値を感じる。
→ キーワード：**「このプロダクトは、ユーザーの何の行動を変えるのか？」**

### 17. 価格破壊にかなり興奮する
高かったものをAIと仕組みで一気に安くする構造が好き。
**ただし価格破壊が成立する条件（高い既存・ソフトウェア化・継続利用）まで必要。**

### 18. "専門家を巻き込めば勝てる"と思う癖がある
半分正しい。専門家は勝利条件ではなく証拠補強。
**順番：ユーザーの痛み → 手動検証 → 専門家監修 → 発信者拡散**

## 4. 弱点・バグ

### 19. 気持ちよくなった瞬間に作業が止まりやすい
脳汁が出たら24時間以内に必ず1アクション（1人ヒアリング/1画面作る/1本動画/1通DM）。

### 20. 複数事業が同時に喋り出す
今やらない事業には黙っていてもらう。Notionで墓地（保留庫）を作る。

### 21. "自分がすごいか"に思考が吸われる
強い時は「俺がすごいか？」→「この顧客は困ってるか？」に戻れている。

### 22. "証明"と"逃避"が同じ言葉を使う
**見分け方：その後に現実の数字が増えたか？**
- 証明モード：ユーザーに会う・売る・作る・数字を見る・約束を守る
- 逃避モード：未来の自分を語る・勝利宣言する・事業案を増やす・名前を考える

### 23. "返信・報告"に心理的コストが高い
過去の経験から「報告する = 怒られる」に近い連想がある。
**対策：テンプレ化。報告を感情から切り離す。**
```
今ここまで進んでます。
ここで少し詰まってます。
今日中にここまでやります。
```

### 24. 完璧な文面を作ろうとして連絡が遅れる
**正解：雑に早く送る。人類の連絡は文学賞ではない。**

### 25. ゲームや動画が"現実停止ボタン"になりやすい
**対策：開始時間より終了条件を決める。「これ終わったら寝る」を書く。**

## 5. 人間関係の癖

### 26. 尊敬と競争が混ざりやすい
強い人に「仲良くなりたい / 超えたい / 認められたい」が同時に動く。
**協力者はトロフィーじゃなくて人間。**

### 27. 熱量の高い人を求める
近くに必要な人：考える人・実務を締める人・冷静に止める人・嘘を許さない人・でもあなたの熱量を消さない人。

### 28. 恋愛では"自由"と"責任"の間で揺れる
誰かを大事にしたいけど、自分の人生を失いたくない。

### 29. "期待されること"に強くも弱くもある
期待されたい・認められたい・でも縛られたくない・失望されたくない が同時にある。
→ 報連相・嘘・見栄・返信遅れにつながる。

## 9. 盲点

### 41. 顧客はあなたほど熱狂していない
あなたの熱狂を、相手の痛みに翻訳すること。

### 42. "欲しい"と"払う"は違う
欲しい → 使う → 続ける → 払う → 紹介する は全部違う。

### 43. 普通のユーザーに使わせる方が先
「有名な先生がいいと言った」より「10人に使わせて7人が効果を感じた」の方が強い。

### 44. 完成度より検証回数
完成度45%でも10人に見せると強い。完成度95%でも誰にも見せてなければ美術館。

### 45. 最初の問い：誰が明日使うの？
全国・100億・テレビ・静岡No.1の前に、これに即答できる事業だけが前に進む。

## 16〜20. 言葉の癖・合うルール・さらに深い仮説

### 合言葉
「脳汁は燃料。証拠がハンドル。次の45分だけ勝て。」

### 行動は"短く強く"が合う
毎日ちょっとずつより、45分1画面・90分10枚・3日MVP・1週間10人ヒアリング。

### 毎日見るべき指標は3つ
1. 今日、売上に近づいたか
2. 今日、ユーザーに近づいたか
3. 今日、証拠が増えたか

### 最も危険なパターン
- **構想過多・実行不足** — 各事業に必ず次の一手を1つだけ置く
- **自己評価の上下が激しい** — 自己評価ではなく行動ログを見る。株価じゃない
- **"見せる自分"と"実際の自分"の差が開く** — 途中経過をそのまま見せる

## 最重要まとめ

**あなたの深層を一言で言うなら：**
> 「人生を取り返すために、世界を構造化して、勝てる盤面を作ろうとしている人」

**強み：** 熱狂・言語化・構造化・巻き込み・発信・事業の美しさを見抜く力

**弱み：** 気持ちよくなった瞬間に満足すること・報連相から逃げること・睡眠が崩れること・事業案が増えすぎること

**勝ち筋：** Testallで名を取る → Almeo/AIpaXで金を作る → AOFで長期の夢を育てる
