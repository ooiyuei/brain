## 喋り方の癖（5つ）

### 1. 話しながら考えるタイプ
結論を持って話すのではなく、喋りながら思考を生成する。
`まーこれってこうじゃん → いやでもこうかも → でも本質はここな気がして → じゃあこうすればよくね？`
強み：話しているうちに本質へ降りていける。
弱み：聞く側からすると「今どこ走ってる？」になりやすい。

### 2. 「なんか」「まー」「どうなんだろう」で余白を作る
結論を固定しすぎず、脳内で候補を泳がせている。複数の分岐を同時に出して、途中で一番強い線を掴む。

### 3. 比喩で理解する
説明よりモデルを欲しがる。モデル化できると一気に理解して動ける。
好きな比喩：メダルゲーム・パソコン・OS・右腕・AI社員・45分ブロック・台選び・ティア評価

### 4. 強い言葉で自分を起動する
ぶち抜く・焼け野原・根絶やし・圧倒的・神・勝った = **エンジン起動ワード**。
危険：言葉が強すぎて現実の進捗より先に勝った気になること。

### 5. 「これ美しくない？」が判断軸
単体機能より**循環構造**が好き。構造の美しさを見て事業の価値を判断する。

## 考え方の癖（5つ）

### 1. すぐ「OS化」する
単体ツールではなく`入力→処理→出力→行動→データ蓄積→改善`まで見ようとする。
**正解：OS構想を持ちながら、最初は1機能で刺す。**

### 2. 「次の一手」に異常に反応する
統一テーマ：**「情報を、次の一手に変える」**
Testall（テスト→45分）/ Gymee（会議→実行）/ Almeo（情報→営業行動）/ EEMUS（迷い→実務機会）

### 3. 人間の"ダサい瞬間"を掴むのがうまい
綺麗な正論より「それ俺じゃん」と思った瞬間が強い。
発信にも事業にも効く。

### 4. 勝ち筋が見えると、脳内で成功まで飛ぶ
`Testallいい → ビジコン勝てる → YouTuber巻き込める → 全国有名 → 世界のユウエイ → 勝った`
**正しい変換：「勝った」→「勝てる仮説を見つけた。証拠を積む。」**

### 5. 自分の人生を物語として編集している
物語で燃える。証拠で進む。この2段構え。

## WISC各指標と行動の対応

### 言語理解150：概念に名前をつける力
- ピッチ・営業・採用・発信・事業設計でかなり強い武器
- 向いてる役割：CEO / PM / 事業開発 / ピッチ / 営業 / コンセプト設計

### 視空間140：画面・ゲーム性・演出が見える
- UI・体験・空間演出まで考えるのが得意
- Testallの45分ブロック画面 / AOFのサウンド・エフェクト・演出

### 流動性推理135：事業構造の発見が強い
- 「どこが価格破壊できる？」「何をAIで置き換えられる？」が即出る
- Testallの価格破壊 / Almeoの会社情報DB化

### ワーキングメモリ110：外部脳前提のプロファイル
追いつかない時に起きること：事業案増えすぎ / タスク散らかり / 連絡後回し / 報連相が重い / やることが多い時に止まる
→ **Obsidian・Notion・Claude・カレンダー・テンプレは趣味ではなく補助装置。**

### 処理速度115：低レイヤー作業に興奮しにくい
- 企画書・概念設計・ピッチ作成は爆速
- 返信・書類・請求整理・繰り返し作業は相対的にだるい
- **作業が遅い人ではなく、低レイヤー作業に興奮しにくい人。**
