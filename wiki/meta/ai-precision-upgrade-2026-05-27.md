# AI Dispatch Precision Upgrade 2026-05-27

## 背景と目的

現状の prompt-prefix.md (v2) はAI臭バン6項目・声調指定が薄い・部署別指示なし。
qwen3.6出力に「依然として有効」「ニーズが高まっている」「珍しくない」等のAI定型表現が混入。
Sonnet vs qwen3.6の委任基準がCLAUDE.mdの散文だけで機械的判定できない。

**目標**: 品質スコア平均 24/40 → 32/40 に引き上げる。

---

## Sonnet vs qwen3.6 委任決定マトリクス

| タスクタイプ | Sonnet | qwen3.6 | 判断軸 |
|---|---|---|---|
| 新規事業の可否判断（GO/NO GO） | ✅ | ❌ | 戦略判断は大井クローンでないと意味なし |
| /idea-judge 10項目採点 | ✅ | ❌ | 評価基準の解釈に深さが必要 |
| 大井の発言・ツイートの代筆 | ✅ | ❌ | 声調・一人称・原体験の再現が要る |
| 数字の矛盾検出・整合性判定 | ✅ | ❌ | 複数ソースの突き合わせ |
| 複数成果物の統合・昇格判定 | ✅ | ❌ | 全体文脈の把握が必要 |
| 競合調査の生データを表に整形 | ❌ | ✅ | テンプレ埋め。判断不要 |
| 事業カードのMarkdown整形 | ❌ | ✅ | フォーマット合わせるだけ |
| 議事録の要約・3セクション化 | ❌ | ✅ | 構造化だけ。判断なし |
| LP・X原稿の量産（10本以上） | ❌ | ✅ | 量産。個別品質より速度 |
| README・APIドキュメント整形 | ❌ | ✅ | 既存情報の構造化 |
| 市場調査データの一次整形 | ❌ | ✅ | 拾ったデータを表にする |
| 翌日の日報自動生成 | ❌ | ✅ | テンプレ通り。判断なし |
| 撤退判断・優先度変更 | ✅ | ❌ | 大井の意思決定の代理 |
| 既存wiki entityの更新提案 | ❌ | ✅ | 差分チェック。判断はSonnetが後でする |

**決定ルール（3秒で判断）:**
- 「大井ならどう言うか？」が問われる → Sonnet
- 「この情報をこの形式に変換せよ」が問われる → qwen3.6
- 「複数の観点を統合して1つの結論を出す」 → Sonnet
- 「同じ処理を10件繰り返す」 → qwen3.6

## 品質スコアリングルーブリック（40点満点）

4次元 × 10点。`quality_score.ps1` で自動採点。

### 次元1: フォーマット準拠（10点）

| 項目 | 点数 |
|---|---|
| YAMLフロントマターが存在し必須フィールドが揃っている | 2点 |
| Markdown見出し（##, ###）が適切に使われている | 2点 |
| 指定wikilinkのみ使用（架空[[]]ゼロ） | 3点 |
| 字数が指定範囲内（指定なし時: 500〜2500字） | 3点 |

### 次元2: 声調マッチ（10点）

| 項目 | 点数 |
|---|---|
| AI臭禁止12パターンが出力に含まれない | 4点（1パターン検出で-1点、最低0点） |
| 大井向けの語調（前置き・挨拶なし） | 2点 |
| 「証拠は？」「誰が明日使う？」視点が含まれている | 2点 |
| 大井の原体験・文脈（自立/仕組み/OS化など）への言及がある場合 +2点（任意加点） | 2点 |

### 次元3: 数字の誠実さ（10点）

| 項目 | 点数 |
|---|---|
| 数字に「推測:」「不明」「[要確認]」のどれかが付いている（根拠なし数字ゼロ） | 4点 |
| 架空企業名・架空固有名詞がない | 3点 |
| 架空ニュース・架空法律がない | 3点 |

### 次元4: 実用価値（10点）

| 項目 | 点数 |
|---|---|
| 大井が明日そのまま使えるアクションが1つ以上書かれている | 4点 |
| 撤退条件またはリスクが明記されている（newbiz系の場合） | 3点 |
| 指示外のスコープを勝手に足していない | 3点 |

**採点基準:**
- 32〜40点: 即昇格可
- 24〜31点: 軽修正で昇格（Sonnetが差分追記）
- 16〜23点: dispatch再投入（指示文を改善）
- 0〜15点: 削除

## v2 vs v3 の主な改善点

| 項目 | v2 | v3 |
|---|---|---|
| AI臭禁止パターン数 | 6項目 | 12項目（代替表現付き） |
| 大井プロフィール深度 | 名前・クセ4行 | 原体験含む7行 |
| 部署別指示 | なし | 7部署それぞれに注意事項 |
| 数字ルール | 「捏造禁止」1行 | 推測/不明/[要確認]の使い分け明記 |
| 声調サンプル | なし | 禁止表現に代替案セット |

## 失敗モードと回復手順

| 失敗モード | 検出方法 | 回復 |
|---|---|---|
| AI臭混入（禁止パターン検出） | quality_score.ps1 声調次元 < 6 | 禁止表現をプロンプトに具体例付きで追記してredispatch |
| 架空数字混入 | 数字次元 < 6 | ContextFilesに一次データを追加してredispatch |
| 架空wikilink生成 | フォーマット次元 wikilink項目 < 3点 | prompt先頭に「以下のwikilink以外の[[]]は絶対に使うな」を強調 |
| 字数超過（3000字超） | フォーマット次元 字数項目 < 3点 | 「〜のみ」「〜に限定」で出力範囲を明示してredispatch |
| スコープ外追加 | 実用価値次元 スコープ項目 < 3点 | 「○○のみ出力せよ。それ以外は書くな」に変更 |

## A/B テスト計画

### 対象
直近30日のdispatch履歴から department=newbiz のタスク20件を抽出。
同一プロンプトを v2 prefix と v3 prefix で各10件ずつ再実行。

### 計測指標
1. quality_score.ps1 合計点の平均値
2. Sonnetが即昇格判定した割合（redispatch不要率）
3. AI臭パターン検出件数の平均

### 成功基準
- 合計点平均: v2 ≥ 24 → v3 ≥ 32（+8点）
- 即昇格率: v2 ≥ 40% → v3 ≥ 65%
- AI臭件数: v2 ≥ 2.5件 → v3 ≤ 0.5件

### スケジュール
- 2026-05-27: v3を scripts/prompt-prefix.md として本番適用
- 2026-05-28〜2026-06-03: 7日間の運用データ収集
- 2026-06-04: 計測・判定（スコアが向上していれば v3 確定、横ばいなら v4 設計）

### フォールバック
v2 は `scripts/prompt-prefix-v2.md` として保存済み。
dispatch.ps1 の `$prefixPath` を v2 パスに戻せば即ロールバック。
