# AIpaX school 入学後30日の生徒・親対応プラン  
**「動くアプリ1個」作らせる + 週次進捗レポート**  
**総字数：約3500字**

---

## はじめに：AIpaX schoolの教育理念と30日間の目標

AIpaX schoolは、中高生向けのAI教育スクールであり、**「AIを手足のように使えるスキル」**を実践で身につけることを目的としています。  
本スクールは、**17歳現役AI起業家の大井**が直接教えることで、実際の現場での学びを再現し、**「学校より現場で学ぶ」**スタイルをカリキュラムに組み込んでいます。

本プランでは、**入学後30日間の生徒・親対応**を軸に、**「動くアプリ1個」**を作らせる目標を達成し、**親への週次進捗レポート**を含む、**30日間の教育・サポートプラン**を設計します。

---

## 1. 30日間の教育目標とスケジュール概要

### 1.1 教育目標

- **生徒**：30日間で「動くアプリ1個」を完成させる。
- **親**：週次進捗レポートを受信し、生徒の学習状況を把握し、サポートを提供する。
- **全体**：AI教育の実践的な学びを体験し、AIリテラシーを高める。

### 1.2 スケジュール概要

| 日数 | 日付 | 作業内容 | 目標 |
|------|------|----------|------|
| 1日 | 入学日 | 入学手続き・環境設定 | 環境を整える |
| 2日 | 3日 | AIツール入門・プロンプト設計 | AIツールの基本を学ぶ |
| 4日 | 5日 | コーディング入門・簡単なアプリ制作 | ノーコード〜ローコードでアプリ作成 |
| 6日 | 7日 | プロジェクトのテーマ設定 | 自分のアイデアを形にする |
| 8日 | 9日 | サービス企画・アイデアの整理 | アイデアを具体化 |
| 10日 | 11日 | アプリの設計・UI/UXの作成 | アプリの形を決める |
| 12日 | 13日 | プログラミング・コードの作成 | アプリの実装 |
| 14日 | 15日 | テスト・改善・修正 | アプリをテストし、改善 |
| 16日 | 17日 | プレゼン資料の作成 | アプリの説明資料を作成 |
| 18日 | 19日 | プレゼン準備・実施 | アプリを発表 |
| 20日 | 21日 | 週次進捗レポートの作成・送信 | 親に進捗を報告 |
| 22日 | 23日 | フィードバックの収集・改善 | 生徒の意見を反映 |
| 24日 | 25日 | 最終的なアプリの完成・公開 | 動くアプリを完成 |
| 26日 | 27日 | 週次進捗レポートの送信 | 親に最終進捗を報告 |
| 28日 | 29日 | フィードバックの整理・改善 | 最終的な改善 |
| 30日 | 31日 | 最終進捗レポートの送信 | 30日間の成果を報告 |

## 2. 週次進捗レポートの内容と送付方法

### 2.1 週次進捗レポートの内容

週次進捗レポートは、**生徒の学習進捗・成果・課題**を親に報告するための重要なツールです。  
以下は、週次進捗レポートの内容と送付方法の例です。

#### 週次進捗レポートの内容

| 項目 | 内容 |
|------|------|
| 1. 週間の学習内容 | その週に学んだ内容を簡潔に説明 |
| 2. 完成した作業 | その週に完成した作業をリストアップ |
| 3. 進捗状況 | 30日間の目標達成度を評価 |
| 4. 課題と改善点 | その週の課題と改善点を記載 |
| 5. 次の週の目標 | 次の週に達成したい目標を記載 |
| 6. 親へのメッセージ | 親に向けたメッセージやサポート要望 |

#### 週次進捗レポートの送付方法

- **送付頻度**：毎週月曜日に送信
- **送信形式**：PDF形式のメールまたはオンラインフォーム
- **送信対象**：生徒の保護者
- **送信時間**：毎週月曜日の午前10時までに送信

## 3. 30日間の教育実施計画

### 3.1 1〜7日：AIツール入門とアプリ制作の基礎

- **目的**：AIツールの基本を学び、簡単なアプリを作成する。
- **内容**：
  - AIツール（ChatGPT、Claude、Canva AIなど）の使い方を学ぶ。
  - プロンプト設計の基礎を学ぶ。
  - ノーコード〜ローコードで簡単なアプリを作成する。
- **成果**：
  - AIツールの基本的な使い方を理解。
  - 簡単なアプリを完成させる。

### 3.2 8〜14日：プロジェクトのテーマ設定とサービス企画

- **目的**：自分のアイデアを形にし、サービス企画を立てる。
- **内容**：
  - 自分のアイデアを整理し、プロジェクトのテーマを設定。
  - サービス企画を立て、アイデアを具体化。
- **成果**：
  - 自分のアイデアを整理し、プロジェクトのテーマを設定。
  - サービス企画を立てる。

### 3.3 15〜21日：アプリの設計とプログラミング

- **目的**：アプリの設計を完了し、プログラミングを実施する。
- **内容**：
  - アプリのUI/UXを設計し、デザインを完成。
  - プログラミングを行い、アプリの実装を完了。
- **成果**：
  - アプリのUI/UXを完成。
  - アプリの実装を完了。

### 3.4 22〜28日：テスト・改善・プレゼン準備

- **目的**：アプリをテストし、改善し、プレゼン資料を作成する。
- **内容**：
  - アプリをテストし、改善点を確認。
  - プレゼン資料を作成し、発表準備を進める。
- **成果**：
  - アプリをテストし、改善を完了。
  - プレゼン資料を作成し、発表準備を進める。

### 3.5 29〜30日：最終進捗レポートと成果発表

- **目的**：最終的な進捗レポートを送信し、成果を発表する。
- **内容**：
  - 最終的な進捗レポートを作成し、親に送信。
  - 成果を発表し、フィードバックを収集。
- **成果**：
  - 最終的な進捗レポートを送信。
  - 成果を発表し、フィードバックを収集。

## 4. 親へのサポートとフィードバック

### 4.1 親へのサポート

- **週次進捗レポート**：毎週月曜日に送信し、生徒の学習進捗を把握。
- **フィードバックの収集**：生徒の意見を反映し、改善を進める。
- **サポート要望の反映**：親の要望を反映し、教育内容を調整。

### 4.2 フィードバックの収集

- **フィードバックの方法**：オンラインフォームやメールでフィードバックを収集。
- **フィードバックの内容**：
  - 学習内容の理解度
  - 進捗状況
  - 課題と改善点
  - 親からのサポート要望
