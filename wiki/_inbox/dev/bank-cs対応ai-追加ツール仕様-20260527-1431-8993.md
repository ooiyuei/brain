# CS対応AI 追加ツール仕様

## 1. 概要：なぜ今「CS対応AI」か

[[entities/aipax]] は現在、4社契約済み（月収20万円）。
月収100万円目標（2026-06-21期限）に対し、残り33日。
既存の「保存庫コア＋営業支援」に加え、**「CS対応AI」**を標準ツールとして追加する。

**理由：**
1. **導入障壁が最も低い**：営業支援は「リアルタイム性」や「社内ルール」の調整が必要だが、CS（問い合わせ対応）は「定型質問への回答」が中心で、保存庫コアの恩恵を最も受けやすい。
2. **即効性がある**：問い合わせの属人化・遅延は中小企業共通の痛み。AIが24時間対応することで、顧客満足度が即向上する。
3. **月間100万円のピースになる**：通常価格（初期100-150万/月額10-15万）の顧客獲得において、CS対応は「最初の入り口」として最適。

## 2. 対象顧客と課題

| 対象 | 課題 | AIpaXでの解決 |
|------|------|-------------|
| **中小企業（30-300名）** | 問い合わせの属人化（特定の社員しか答えられない） | 保存庫コアからAIが即時回答 |
| **営業・CS担当者の負担** | 夜間・休日の問い合わせ対応漏れ | 24時間AI対応 |
| **新人教育コスト** | 基本的な質問に新人が対応できない | AIが標準回答を提示 |

## 3. ツール仕様

### 3.1 コア機能

1. **問い合わせ受信**：
   - メール、Webフォーム、LINE公式アカウントからの問い合わせを受信。
   - 保存庫コア内の資料（FAQ、契約条件、料金表、顧客別条件）を参照。

2. **AI回答生成**：
   - 保存庫コアから関連情報を検索。
   - 顧客の質問に対して、保存庫コアの情報を基に回答を生成。
   - **重要**：保存庫コアに情報が無い場合、または矛盾がある場合は、その旨を提示し、人間へのエスカレーションを促す。

3. **回答カードの自動作成**：
   - 生成された回答を、顧客向けにフォーマットされた「回答カード」として出力。
   - 担当者が確認・修正後、送信可能。

4. **学習・改善**：
   - 担当者が回答を修正した場合、その修正内容を保存庫コアに反映（またはフィードバックとして記録）。
   - 回答できなかった質問を収集し、保存庫コアの不足部分を特定。

### 3.2 保存庫コアとの連携

- **検索精度の向上**：保存庫コアが整備されているほど、AIの回答精度が向上する。
- **更新の自動化**：保存庫コアが更新されると、AIの回答知識も自動的に更新される。
- **例外条件の扱い**：保存庫コアに「顧客別条件」や「例外条件」が保存されている場合、AIはそれを考慮して回答を生成する。

### 3.3 技術スタック

- **Supabase**：問い合わせデータ、回答履歴、保存庫コアのメタデータを管理。
- **OpenAI / File Search / Embedding**：保存庫コアからの情報検索、意味検索。
- **Claude / GPT**：回答生成。
- **Webhook / API**：メール、Webフォーム、LINE公式アカウントとの連携。

## 4. 価格設定

| 段階 | 初期費用 | 月額費用 | 内容 |
|------|---------|---------|------|
| **初回事例価格** | 50-80万円 | 5万円 | 保存庫コア構築 + CS対応AI 1ツール |
| **通常価格** | 100-150万円 | 10-15万円 | 保存庫コア構築 + CS対応AI 1ツール + 追加ツール |
| **本格導入** | 200-500万円 | 20-50万円 | 全社展開、複数ツール、CRM連携等 |

## 5. 導入プロセス

1. **ヒアリング**：
   - 顧客の問い合わせ種類、頻度、現在の対応プロセスをヒアリング。
   - 保存庫コアに反映すべき資料を特定。

2. **保存庫コア構築**：
   - 顧客の資料（FAQ、契約条件、料金表等）を保存庫コアに反映。
   - 顧客別条件、例外条件を整理。

3. **CS対応AIの設定**：
   - 保存庫コアへの連携設定。
   - 回答のトーン、フォーマットの設定。
   - エスカレーションルール（回答できない場合の対応）の設定。

4. **テスト・検証**：
   - 実際の問い合わせデータを用いて、AIの回答精度を検証。
   - 担当者のフィードバックを反映。

5. **ローンチ**：
   - 本番環境での運用開始。
   - 運用開始後のサポート（回答カードの改善、保存庫コアの更新等）。

## 6. 3ヶ月以内の検証アクション

1. **既存4社への提案**：
   - 既存4社（[[entities/aipax]]契約済み）に、CS対応AIの追加ツールとして提案。
   - 初期費用50-80万円、月額5万円の条件で、1社でも契約を獲得。

2. **新規顧客への提案**：
   - CS対応AIを「最初の入り口」として、新規顧客に提案。
   - 通常価格（初期100-150万円/月額10-15万円）での契約獲得を目指す。

3. **保存庫コアの標準化**：
   - 全顧客共通の汎用基盤として、保存庫コアの標準化を進める。
   - 保存庫コアの整備が、CS対応AIの精度向上に直結することを証明。

## 7. 注意点

- **保存庫コアとツール出力を混ぜない**：保存庫コアは「知識の源」、CS対応AIは「出力ツール」。役割を明確にする。
- **顧客資料の正しさは顧客側が決める**：AIが生成した回答は、最終的に顧客側が確認・承認する。
- **AIが勝手に正式情報を作らない**：保存庫コアに情報が無い場合は、その旨を提示し、人間へのエスカレーションを促す。
- **不足・矛盾・最新版不明を検出して提示する**：保存庫コアの不足部分を特定し、顧客にフィードバックする。
- **対象領域内の顧客別条件・例外条件は削らない**：保存庫コアには、顧客別条件、例外条件を保存し、AIがそれを考慮して回答を生成する。

## 8. 結び

[[entities/aipax]] の月収100万円達成には、既存4社の維持・拡大と、新規顧客の獲得が不可欠。
CS対応AIは、そのための「強力なツール」となる。
保存庫コアの整備と、CS対応AIの標準化を進め、[[entities/aipax]] の価値を最大化する。
