# FishOps Agent 機能仕様v1

## 1. 概要と設計思想

**FishOps Agent** は、管理釣り場・港前釣具屋・漁協・養殖場といった「水産・釣り現場」に散らばる **経験と勘をデータ化** し、AIエージェントが運営改善を行う基盤。

**核心**: 「魚をたくさん入れる」のではなく、「満足度と利益が最大になる放流量を見つける」こと。
**ターゲット**: 管理釣り場（閉じた空間でデータ取りやすい）→ 港前釣具屋（地域密着）→ 漁協（行政・観光連携）

**大井の脳汁ポイント**:
- 単なるサブスクではなく、**「AI店長」** を配る。
- 釣れなかったデータを蓄積し、改善に使う。
- 現場のITリテラシーの低さを逆手に取り、「操作不要・自動レポート」で浸透させる。

## 2. 主要プロダクトライン（5本柱）

### 2.1 管理釣り場AI店長（MVP1：最優先）

**ターゲット**: 管理釣り場・釣り堀・釣り体験施設
**課題**: 放流量の決定が経験・勘依存。釣果記録が手書きまたはなし。初心者比率の把握が困難。

**機能仕様**:
1. **データ入力（簡易）**
   - Googleフォーム/LINE公式から、店側が「来場者数」「初心者比率」「放流数」「魚種」「釣果写真（任意）」を入力。
   - 顧客側（釣り人）からも「釣れた数」「釣れなかった理由」「満足度」をQRコード経由で投稿。
2. **AI分析・レポート**
   - 釣果データと満足度を紐付け、「どの魚種をどのくらい放流すれば、満足度が最大化するか」を提案。
   - 「釣れなかった人」の分析：天候・時間帯・仕掛け別の失敗パターンを特定。
3. **運営支援**
   - 翌日の放流提案：前日の釣果データから、明日の魚種・数をAIが提案。
   - SNS投稿案自動生成：釣果写真を基に、SNS用のキャプションを生成。
   - 週次/月次レポート：売上原価・釣果・満足度のトレンドを可視化。

**価値**: 勘で決めていた放流と接客を、データで改善。

### 2.2 港前釣具屋AI店長（MVP2）

**ターゲット**: 古い釣具屋・後継者不足店
**課題**: 立地・地元情報・常連関係・漁協・船宿とのつながり・初心者相談・暗黙知が店主の頭の中にある。

**機能仕様**:
1. **地域情報連携**
   - 今日の釣れやすい場所・天候・水温情報を自動収集・整理。
2. **顧客向け提案**
   - 初心者向け仕掛けセットの提案。
   - 店頭POP・LINE配信・SNS投稿の自動生成。
   - Googleマップの改善提案（写真・レビュー対応）。
3. **在庫・売上分析**
   - 売れ筋分析・餌・仕掛けの発注提案。
   - 常連イベントの企画提案。

**価値**: 「商品を売る店」から「地域の釣り体験ステーション」へ。

### 2.3 漁協AI事務局

**ターゲット**: 漁協・自治体・遊漁管理団体
**機能**: 遊漁券管理・釣果収集・放流効果レポート・ルール案内・釣り場情報発信・SNS・HP更新・問い合わせ対応・補助金資料・行政報告書下書き・観光施策効果測定・違反・混雑可視化。

**価値**: 人手不足の漁協に、AI事務局を。

### 2.4 養殖AI副操縦士

**ターゲット**: 小規模養殖・陸上養殖・管理釣り場の魚育成
**機能**: 水温・酸素・pH記録・給餌ログ・死亡数・成長記録・異常アラート・餌コスト・出荷予測・作業日報・現場指示・ベテラン判断のログ化。

**価値**: ベテランの勘をデータ化し、若手でも養殖運営できるように。

### 2.5 釣り観光AIコンシェルジュ

**ターゲット**: 自治体・観光地・宿泊・船宿
**機能**: 観光客向け釣りプラン・手ぶら釣り予約・親子向け体験提案・送客・天候別代替プラン・釣果写真→思い出レポート・地域クーポン連携・宿泊飲食連携。

**価値**: 地域の釣り資源を、AIが観光商品に変える。

## 3. 技術スタックと実装方針

**基本方針**: 現場のITリテラシーの低さを考慮し、**「操作不要・自動」** を徹底。
**入力**: Googleフォーム/LINE公式/QRコード。
**処理**: Claude AI（分析・提案・文章生成）。
**出力**: Looker Studio（レポート）/ LINE公式（通知）/ SNS（投稿案）。

**MVP1（管理釣り場AI店長）の具体的な実装**:
1. **データ収集**
   - Googleフォーム：来場者数・初心者比率・放流数・魚種・釣果写真。
   - LINE公式：顧客からの釣果投稿・満足度アンケート。
2. **データ蓄積**
   - Googleスプレッドシート：一時データ蓄積。
   - Supabase：構造化データ蓄積（将来的に）。
3. **AI分析**
   - Claude API：釣果データと満足度の相関分析。
   - 釣れなかった人の分析：天候・時間帯・仕掛け別の失敗パターン特定。
4. **レポート生成**
   - Looker Studio：週次/月次レポートの自動生成。
   - LINE公式：翌日の放流提案・SNS投稿案の自動配信。

## 6. 初期MVP（軽く始める）

### MVP1：管理釣り場AIレポート
- **スタック**: Googleフォーム/LINE公式/スプレッドシート/Claude/Looker Studio/釣果写真アップロード
- **取るデータ**: 来場/初心者比率/レンタル/放流/魚種/釣果写真/釣れた数/釣れなかった人/満足度/また来たいか/困ったこと
- **出すもの**: 今日の釣果レポート/釣れなかった人の分析/放流効果分析/明日の運営提案/SNS投稿案/週次レポート

### MVP2：港前釣具屋AI店長
- **スタック**: LINE公式/QR釣果投稿/Googleマップ整備/SNSテンプレ/初心者マップ/手ぶら釣りパック設計/店頭POP/釣果週次/売れ筋分析

### MVP3：釣り体験イベント
- **内容**: 10人集めて検証：何に困るか/いくら払うか/手ぶら需要/リピート/不満ポイント/サポート価値/写真投稿

## 7. 市場データ

| 指標 | 数値 |
|------|--|
| 国内釣り人口（2022年） | 520万人（レジャー白書） |
| 釣具市場（2024年見込み） | 1,516億円 |
| 釣り人口減少（2017→2022） | 約120万人減 |

→ 市場は緩やかに縮小。だがFishOpsは「市場を拡大する」のではなく「**既存施設の運営効率を上げる**」事業なので、市場縮小と相関しない。

## 8. リスクと対策

| リスク | 対策 |
|--------|------|
| 市場ニッチ（管理釣り場SaaS単体だと天井低い） | 釣具屋/船宿/漁協/自治体/観光/養殖/放流へ広げる |
| 現場がITに弱い | 「AI店長が毎週レポート出す。LINEと紙でも見れる。操作不要」レベルに落とす |
| データ入力面倒 | 入力項目を絞る（来場/放流/釣果写真/満足度のみ） |
| 釣果データの精度 | 個人投稿だけに頼らず店舗側データと組み合わせ |
| 買収は重い | 提携/運営改善/レベニューシェア/共同事業から入る |
