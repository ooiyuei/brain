# AIpaX school 1年カリキュラム

## 🎯 5モジュール構造

### Module 1 (Month 1-2): AIマインドセット + バイブ体験
- **学習目標**: AIを手足のように使えるマインドセットを構築し、AIツールの基本操作を習得
- **週次トピック**:
  1. AIとは何か？（AIの定義・歴史・今後の展望）
  2. AIツールの選び方（ChatGPT / Claude / Canva AIなど）
  3. プロンプト設計の基本（AIに何を聞けばいいか）
  4. AIで作れるもの（AIで作れるもの・例：プレゼン資料・文章作成）
  5. AIと人間の関係（AIの限界・人間の役割）
  6. AIで学ぶ習慣（毎日AIを使う習慣をつける）
  7. AIと社会（AIが社会に与える影響）
  8. AIの未来（AIがどう進化するか）
- **課題と発表**: AIツールを使って「AIで作れるもの」を1つ作成し、プレゼン資料を作成
- **必要ツール**: Antigravity / Cursor / Claude / Canva AI
- **評価方法**: 作成したプレゼン資料の完成度・AIツールの使い方の理解度

### Module 2 (Month 3-4): 日常業務自動化
- **学習目標**: 日常業務をAIで自動化し、効率化するスキルを習得
- **週次トピック**:
  1. 日常業務の自動化とは？（自動化の定義・メリット）
  2. AIで自動化できる業務（例：メール作成・スケジュール管理）
  3. AIツールの自動化機能（例：ChatGPTの自動文章作成）
  4. 自動化の実践（AIツールを使って日常業務を自動化）
  5. 自動化の限界（AIができないこと・人間の役割）
  6. 自動化のコスト（時間・労力・コストの比較）
  7. 自動化の未来（AIがどう進化するか）
  8. 自動化の実践（AIツールを使って日常業務を自動化）
- **課題と発表**: 日常業務をAIで自動化し、その結果をプレゼン資料で発表
- **必要ツール**: Antigravity / Cursor / Claude / Notion AI
- **評価方法**: 自動化の実践結果の完成度・AIツールの使い方の理解度

### Module 3 (Month 5-6): エンジニアリング思考 + 外部連携
- **学習目標**: エンジニアリング思考を身につけ、外部と連携するスキルを習得
- **週次トピック**:
  1. エンジニアリングとは？（エンジニアの役割・スキル）
  2. AIとエンジニアリング（AIがエンジニアリングに与える影響）
  3. 外部連携の重要性（外部と連携するメリット）
  4. 外部連携の実践（AIツールを使って外部と連携）
  5. 外部連携の限界（AIができないこと・人間の役割）
  6. 外部連携のコスト（時間・労力・コストの比較）
  7. 外部連携の未来（AIがどう進化するか）
  8. 外部連携の実践（AIツールを使って外部と連携）
- **課題と発表**: 外部と連携し、AIツールを使ってプロジェクトを実行し、その結果をプレゼン資料で発表
- **必要ツール**: Antigravity / Cursor / Claude / Figma
- **評価方法**: 外部連携の実践結果の完成度・AIツールの使い方の理解度

### Module 4 (Month 7-8): 本格アプリ開発 + リリース
- **学習目標**: 本格的なアプリを開発し、リリースするスキルを習得
- **週次トピック**:
  1. アプリ開発とは？（アプリ開発の定義・メリット）
  2. アプリ開発の流れ（アイデア・設計・開発・テスト・リリース）
  3. AIツールのアプリ開発機能（例：Cursorのコード生成）
  4. アプリ開発の実践（AIツールを使ってアプリを開発）
  5. アプリ開発の限界（AIができないこと・人間の役割）
  6. アプリ開発のコスト（時間・労力・コストの比較）
  7. アプリ開発の未来（AIがどう進化するか）
  8. アプリ開発の実践（AIツールを使ってアプリを開発）
- **課題と発表**: 本格的なアプリを開発し、リリースし、その結果をプレゼン資料で発表
- **必要ツール**: Antigravity / Cursor / Claude / Figma
- **評価方法**: アプリ開発の実践結果の完成度・AIツールの使い方の理解度
