## 🎯 5モジュール構造 (1年間)

### Module 1: AIマインドセット + バイブ体験 (Month 1-2)
- **学習目標**: AIを手足のように使えるマインドセットを構築し、AIツールの基本操作を習得
- **週次トピック**:
  1. AIとは？AIの歴史と今後の展望
  2. AIツールの選定と使い方（ChatGPT / Claude / Canva AI）
  3. プロンプト設計の基礎（例: 画像生成 / テキスト生成）
  4. AIと人間の協働（AIができないこと / AIが得意なこと）
  5. AIで何か作れる体験（AIで動画作成 / AIで音楽作成）
  6. AIと社会の関係（AI倫理 / AIと雇用）
  7. AIで学ぶ習慣（AIを活用した学習法）
  8. AIマインドセットのまとめと振り返り
- **課題と発表**: AIで作成した動画・音楽・プレゼン資料を提出
- **必要ツール**: Antigravity / Cursor / Claude / Canva AI
- **評価方法**: 作成した作品の完成度 + 週次振り返りの質

### Module 2: 日常業務自動化 (Month 3-4)
- **学習目標**: 日常業務をAIで自動化し、効率化のスキルを習得
- **週次トピック**:
  1. 日常業務の自動化とは？
  2. AIで自動化できる業務の種類（例: メール作成 / データ整理）
  3. AIツールの自動化機能（例: Notion AI / Gamma）
  4. 自動化の実践（メール自動作成 / データ整理の自動化）
  5. 自動化のコストと効率（時間の節約 / エラー防止）
  6. 自動化の限界と注意点（例: プライバシー / データの正確性）
  7. 自動化の実践（複数業務の自動化）
  8. 自動化のまとめと振り返り
- **課題と発表**: 自動化した業務の成果をプレゼン資料で発表
- **必要ツール**: Notion AI / Gamma / Antigravity
- **評価方法**: 自動化の成果の完成度 + 週次振り返りの質

### Module 3: エンジニアリング思考 + 外部連携 (Month 5-6)
- **学習目標**: エンジニアリング思考を身につけ、外部との連携を学ぶ
- **週次トピック**:
  1. エンジニアリング思考とは？
  2. プロジェクトの設計（例: アプリの設計 / サービスの設計）
  3. チームでの協働（例: チームでのアイデア出し / チームでの作業）
  4. 外部との連携（例: 企業との連携 / 他者との連携）
  5. プロジェクトの実践（アイデアの実現 / チームでの作業）
  6. プロジェクトの評価（成果の評価 / チームの評価）
  7. プロジェクトの改善（改善点の検討 / 改善後の実践）
  8. プロジェクトのまとめと振り返り
- **課題と発表**: プロジェクトの成果をプレゼン資料で発表
- **必要ツール**: Antigravity / Cursor / Figma
- **評価方法**: プロジェクトの成果の完成度 + 週次振り返りの質

### Module 4: 本格アプリ開発 + リリース (Month 7-8)
- **学習目標**: 本格的なアプリを開発し、リリースのスキルを習得
- **週次トピック**:
  1. アプリ開発の基礎（例: アプリの設計 / アプリの構築）
  2. アプリの設計（例: ユーザーインターフェース / データベース）
  3. アプリの構築（例: プログラミング / データベースの構築）
  4. アプリのテスト（例: テストの設計 / テストの実施）
  5. アプリのリリース（例: リリースの準備 / リリースの実施）
  6. アプリの改善（例: ユーザーからのフィードバック / 改善後の実践）
  7. アプリの評価（例: ユーザーからの評価 / チームからの評価）
  8. アプリのまとめと振り返り
- **課題と発表**: アプリの成果をプレゼン資料で発表
- **必要ツール**: Antigravity / Cursor / Figma
- **評価方法**: アプリの成果の完成度 + 週次振り返りの質

### Module 5: データ分析 + マネタイズ (Month 9-12)
- **学習目標**: データ分析とマネタイズのスキルを習得
- **週次トピック**:
  1. データ分析とは？
  2. データ分析の種類（例: データの整理 / データの分析）
  3. データ分析の実践（例: データの整理 / データの分析）
  4. マネタイズの基礎（例: マネタイズの方法 / マネタイズの種類）
  5. マネタイズの実践（例: マネタイズの方法 / マネタイズの種類）
  6. マネタイズの改善（例: マネタイズの改善 / マネタイズの実践）
  7. マネタイズの評価（例: マネタイズの成果 / マネタイズの改善）
  8. マネタイズのまとめと振り返り
- **課題と発表**: データ分析とマネタイズの成果をプレゼン資料で発表
- **必要ツール**: Antigravity / Cursor / Figma
- **評価方法**: データ分析とマネタイズの成果の完成度 + 週次振り返りの質
