# AIpaX school 1年カリキュラム (5モジュール構成)

## 🎯 5モジュール構成 (2ヶ月 × 5 = 1年)

### Module 1: AIマインドセット + バイブ体験 (Month 1-2)
- **学習目標**: AIを手足のように使えるマインドセットを完成
- **週次トピック**:
  1. AIとは？人間とAIの関係
  2. AIの歴史と現在のトレンド
  3. AIの倫理と社会への影響
  4. AIツールの基礎（ChatGPT / Claude / Canva AI）
  5. プロンプト設計の基本
  6. AIで何か作れる体験（バイブコーディング）
  7. AIと人間の協働の未来
  8. AIマインドセットのまとめ
- **課題と発表**: AIツールで「AIで何か作れる」体験レポート
- **必要ツール**: Antigravity / Cursor / Claude / Canva AI
- **評価方法**: 体験レポートの完成度 + プロンプト設計の実践度

### Module 2: 日常業務自動化 (Month 3-4)
- **学習目標**: 日常業務をAIで自動化するスキルを習得
- **週次トピック**:
  1. 日常業務の自動化とは？
  2. AIで自動化できる業務の種類
  3. ノーコード・ローコードツールの使い方
  4. Replit / Cursor の基礎
  5. 自動化の実践（スケジュール管理・メール自動化）
  6. データの整理とAI活用
  7. 自動化の限界と倫理
  8. 日常業務自動化のまとめ
- **課題と発表**: 自分の日常業務をAIで自動化する実践報告
- **必要ツール**: Replit / Cursor / Google Apps Script
- **評価方法**: 実践報告の完成度 + 自動化の実行力

### Module 3: エンジニアリング思考 + 外部連携 (Month 5-6)
- **学習目標**: エンジニアリング思考を身につけ、外部と連携するスキルを習得
- **週次トピック**:
  1. エンジニアリング思考とは？
  2. プロジェクトの構成と設計
  3. チームでのアイデア形成
  4. 外部との連携方法
  5. プロジェクトの実行計画
  6. チームでの協働とフィードバック
  7. プロジェクトの評価と改善
  8. エンジニアリング思考のまとめ
- **課題と発表**: 自分のプロジェクトをチームで設計する実践報告
- **必要ツール**: Figma / Slack / GitHub
- **評価方法**: プロジェクト設計の完成度 + チーム協働の実践度

### Module 4: 本格アプリ開発 + リリース (Month 7-8)
- **学習目標**: 本格的なアプリ開発を実践し、リリースするスキルを習得
- **週次トピック**:
  1. アプリ開発の基本と設計
  2. プロジェクトの構成と設計
  3. ローコードツールの実践
  4. アプリのUI/UX設計
  5. アプリのテストと改善
  6. アプリのリリース準備
  7. プロジェクトの評価と改善
  8. アプリ開発のまとめ
- **課題と発表**: 自分のアプリをリリースする実践報告
- **必要ツール**: Figma / Replit / GitHub
- **評価方法**: アプリの完成度 + リリース準備の実践度

### Module 5: データ分析 + マネタイズ (Month 9-12)
- **学習目標**: データ分析とマネタイズのスキルを習得
- **週次トピック**:
  1. データ分析とは？
  2. データの収集と整理
  3. データの可視化と分析
  4. マネタイズの基本と方法
  5. データを活用したビジネスモデル
  6. データ分析の実践
  7. マネタイズの実践
  8. データ分析とマネタイズのまとめ
- **課題と発表**: 自分のデータ分析とマネタイズの実践報告
- **必要ツール**: Google Sheets / Tableau / Notion
- **評価方法**: データ分析の完成度 + マネタイズの実践度

## 📌 評価基準
- **完成度**: 課題の完成度と実践力
- **実践度**: 実際に手を動かして作るスキル
- **創造性**: 自分のアイデアを形にする力
- **チーム協働**: チームでの協働とフィードバック
