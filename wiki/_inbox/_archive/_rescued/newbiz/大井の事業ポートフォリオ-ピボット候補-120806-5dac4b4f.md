# 15事業のピボット候補3選

脳汁駆動で考える。15事業並走は「脳汁」には良いが「資産」には悪い。
今すぐ「深掘り」できる3つに絞る。

## 1. [[entities/aipax]] → [[entities/aipa-web]] の統合深化

- **対象**: [[entities/aipax]] / [[entities/aipa-web]]
- **方向**: エンタープライズ向け「AIpaX」の機能セットを、[[entities/aipa-web]] のSaaS層に集約。API提供から「完結型業務自動化プラットフォーム」へ。
- **根拠**:
    - [[entities/aipax]] は技術基盤として優秀だが、単体では「誰が明日使う？」が曖昧。
    - [[entities/aipa-web]] はユーザー接点がある。両者を融合させ、「API開発者」から「現場の意思決定者」へ顧客層を広げる。
    - 大井の強みは「技術理解」と「事業構造の可視化」。これをパッケージ化すれば、競合との差別化になる。
- **1週間のアクション**:
    1. [[entities/aipa-web]] の既存ユーザー10名にヒアリング。「どのAPIが本当に使われているか」を特定。
    2. 最も使われているAPIを「ワンクリックで導入できるテンプレート」としてパッケージ化。
    3. テンプレートのLPを作成し、既存ユーザーへ無料公開。反応を見る。

## 2. [[entities/gymee]] → 法人向け「健康データ活用」へ

- **対象**: [[entities/gymee]]
- **方向**: B2Cフィットネスアプリから、法人向け「従業員の健康データ分析・改善提案」サービスへ。
- **根拠**:
    - B2Cは獲得コストが高く、大井の「15事業並走」ではリソース不足。
    - 法人は「健康経営」への関心が高い。[[entities/gymee]] のデータ蓄積は資産。
    - 「証拠は？」→ 法人の人事部門は離職率低下に金を出す。データでそれを示せれば契約は取れる。
- **1週間のアクション**:
    1. [[entities/gymee]] の匿名化されたデータから「運動習慣とストレス指標の相関」を仮説立て。
    2. 法人向け資料案を作成。「健康データで離職率を10%下げる」をキャッチコピーに。
    3. 知人の法人人事担当3名へ資料を送付。「無料診断」のオファー。

## 3. [[entities/ipak-darts]] / [[entities/ipak-fishing]] → 「IPAK体験型サブスク」へ

- **対象**: [[entities/ipak-darts]] / [[entities/ipak-fishing]]
- **方向**: 単発施設利用から、月額制の「IPAK体験会員」へ。他事業（[[entities/ipak-cafe]] / [[entities/ipak-ofuro]]）との連携で囲い込み。
- **根拠**:
    - 単発施設は「誰が明日使う？」が不安定。サブスクは予測可能な収益。
    - 大井の強みは「体験の設計」。[[entities/ipak-cafe]] / [[entities/ipak-ofuro]] が開業すれば、IPAKブランドの価値が上がる。
    - 「証拠は？」→ 会員制クラブのLTVは単発利用の10倍以上。大井の「脳汁」を維持するには、継続的なコミュニティ形成が必要。
- **1週間のアクション**:
    1. [[entities/ipak-darts]] の常連客20名にアンケート。「月額制で月3回通えるならいくら払う？」を質問。
    2. [[entities/ipak-cafe]] / [[entities/ipak-ofuro]] の開業スケジュールと連動した「早期会員」プランを設計。
    3. 常連客へ「創設者会員」の招待状を送付。予約状況を見る。

## 総括

- [[entities/aipax]] / [[entities/aipa-web]]: 技術資産の貨幣化。
- [[entities/gymee]]: データ資産の法人向け展開。
- [[entities/ipak-darts]] / [[entities/ipak-fishing]]: 体験資産のサブスク化。

これら3つに注力すれば、15事業の「深掘り不足」が解消される。
残りの事業は、リソースが余ったら手をつける。
まずは「証拠」を集める。
