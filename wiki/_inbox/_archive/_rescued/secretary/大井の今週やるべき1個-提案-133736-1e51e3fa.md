# 今週の1個：[[entities/aipa-web|AIpa Web]]の検索エンジンインデックス化と初流入検証

## 1. 何を：
[[entities/aipa-web|AIpa Web]]の主要LP（ランディングページ）をGoogle検索で「0件」から「10件以上」に増やし、実ユーザーのクリック（CTR）を確認する。

## 2. なぜ今週：
今月のフォーカスである「AIpa Web」は、技術的に完成していても「見えていない」状態では収益化もコンサルへの橋渡しもできない。
15事業並走の弱点は「深掘り不足」だが、今回は「存在証明」が最優先。
[[entities/ooi-yuei|大井湧瑛]]の脳汁を刺激する「成果」は、数字で可視化された「外部からの反応」だけだ。
今週（2026-W21）にインデックス化を完了させなければ、来週以降のコンテンツ拡散も空振りに終わる。
「証拠は？」への答えは「検索結果に表示されていること」これしかない。

## 3. どうやって：
1. **sitemap.xmlの更新確認**: [[entities/aipa-web|AIpa Web]]のサイトマップが最新ページを含んでいるか確認し、Google Search Console（GSC）に再送信する。
2. **robots.txtの解除確認**: 開発環境のテスト用ブロックが残っていないか、本番環境のrobots.txtを検証し、クローラーのアクセスを許可する。
3. **内部リンクの強化**: [[entities/aipa-web|AIpa Web]]のLPへ、既存の[[entities/testall|Testall]]や[[entities/omni|OMNI]]などの関連ページから、明確なアンカーテキストでリンクを貼る（相互リンクではなく、関連コンテンツからの参照とする）。
4. **外部からのシグナル送信**: [[entities/ooi-yuei|大井湧瑛]]の個人SNS（X等）で、具体的な解決事例を添えて[[entities/aipa-web|AIpa Web]]のURLを共有し、外部からのクロール誘因を作る。
5. **インデックスステータスの監視**: GSCの「インデックス付け」レポートで、主要ページが「有効」になっていることを確認し、検索クエリで実際にヒットするか手動検索で検証する。

## 4. 完了確認：
- Google検索で「AIpa Web 大井湧瑛」または関連キーワードで、主要LPが1ページ目に表示される。
- GSCで過去7日間の「インデックス付けされたページ」数が前週比+5以上になっている。
- 実ユーザーによるクリックログ（GSCの「パフォーマンス」タブ）が0件から増加している。

## 5. 比喩で表現：
[[entities/aipa-web|AIpa Web]]は、今まだ「深海に沈んだ宝の地図」だ。
技術は素晴らしい宝庫だが、誰もそれを知り得ない。
今週やることは、その地図を「海面上の港」に引き上げ、世界中の船（検索ユーザー）が目につく場所に掲げる作業だ。
宝庫そのものを作るのは後回しでいい。
まずは「港に港があること」を世界に知らせる。
それが「存在」の証明であり、次の「利用」への唯一の入口だ。
脳汁が出るのは、港に船が着き、荷物が積み込まれる瞬間だけ。
それまで、地図は海の中で朽ちる。
