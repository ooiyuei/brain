# 今週の1個：[[entities/aipa-web|AIpa Web]]の「検索インデックス化」突破

## 1. 何を：
[[entities/aipa-web|AIpa Web]]のコンテンツを、Google検索のインデックス対象かつ「AI検索（SGE）の回答ソース」として認識させるための「構造化データ埋め込み＋外部被リンク取得」アクション。

## 2. なぜ今週：
今月のフォーカスである「AIpa Web」の死活問題である「認知の欠如」を解消する唯一の物理的根拠を作るため。
15事業並走の現状、脳汁が冷めないうちに「成果が見える形」にする必要がある。ビジコン応募やSFC AO構想は「準備」段階だが、AIpa Webの検索流入は「収益・認知」の直結ライン。今週、インデックス登録が完了し、来週以降の有機的流入の種をまく。遅れれば、他の事業に意識が散漫になり、この「唯一の公開資産」が放置される。

## 3. どうやって：
1. **スキーママークアップの適用**: [[entities/aipa-web|AIpa Web]]の主要ページ（LP、サービス詳細）に、`SoftwareApplication`および`FAQPage`のJSON-LD構造化データを埋め込む。Googleに「これは何ぞや」を教える。
2. **内部リンクの強化**: 既存のコンテンツから、AIpa Webの核心ページへ向かう内部リンクを3つ以上新規作成・強化する。
3. **外部被リンクの確保**: 関連する技術ブログやニュースサイト、あるいは[[entities/ipak-cafe|Cafe IPAK]]などの既存資産から、AIpa Webへの明確な被リンクを取得する（またはSNSで公開し、Googlebotのクロールを促す）。
4. **インデックスリクエスト**: Google Search Consoleより、主要ページのインデックス登録を直接リクエストする。
5. **結果の監視**: 24時間後、Search Consoleの「インデックス登録」ステータスが「有効」になっていることを確認する。

## 4. 完了確認：
- Google Search Consoleで、AIpa Webの主要ページが「インデックス済み」と表示されていること。
- Google検索で、AIpa Webの固有のキーワード（例：「AIpa Web 機能」）で、サイトが1ページ目に表示されること。
- 構造化データのテストツールでエラーがゼロであること。

## 5. 比喩で表現：
**「砂漠に井戸を掘る」**
15事業は砂漠を歩く旅に似ている。脳汁は水だ。しかし、水を持っていても、誰にも見えていなければ意味がない。AIpa Webは、その砂漠に掘った「井戸」だ。今週は、その井戸の蓋を開け、水を汲むためのバケツ（検索インデックス）を準備する。井戸が公開されて初めて、通りすがりの人（ユーザー）が水を求めて集まる。蓋を閉じたままでは、井戸はただの穴でしかない。今週、その蓋を開ける。
