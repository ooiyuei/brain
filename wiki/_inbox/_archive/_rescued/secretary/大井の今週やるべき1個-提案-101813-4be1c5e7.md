# 今週の1個：[[entities/aipa-web|AIpa Web]]の「最小機能版」公開

## 1. 何を
[[entities/aipa-web|AIpa Web]]の「問い合わせフォーム＋LP」のみを公開し、**初回コンタクト（リード）を1件獲得する**。

## 2. なぜ今週
今月のフォーカス「AIpa Web / AIコンサル」において、**「見えない仕事」は存在しない**。
15事業並走の弊害で、どの事業も「準備中」で終わっている。
[[entities/aipa-web|AIpa Web]]は、他の事業（[[entities/gymee|Gymee]]や[[entities/eemus|EEMUS]]）の横展開や、[[entities/aipax-school|AIpaX School]]の集客源として機能する可能性を秘めている。
しかし、サイトがない＝信用ゼロ＝商談の起点なし。
今週中に「公開＝完了」にすることで、**「脳汁駆動」の燃料である「反応（フィードバック）」を得る**必要がある。
反応がないと、[[entities/ooi-yuei|大井湧瑛]]の判断軸である「証拠」が得られない。
「誰が明日使う？」という問いに対し、今の状態では「誰も使わない（見えない）」が正解。
これを打破するには、**「不完全でも出す」**しかない。

## 3. どうやって（5ステップ）

1.  **LPの骨子決定（30分）**
    *   ヘッドライン：「AIで業務を自動化する｜大井湧瑛のAIpa Web」
    *   サブコピー：「15事業を回す1人起業家が選んだ、実証済みのAI導入ノウハウ」
    *   CTA（行動喚起）：「無料相談予約」ボタン
    *   *注意：デザインはテンプレート（CocoonやWix等）のデフォルトでOK。カスタマイズ禁止。*

2.  **問い合わせフォーム設置（1時間）**
    *   [[entities/aipa-web|AIpa Web]]のドメインまたはサブドメインに接続。
    *   フォーム項目：「お名前」「会社名」「相談したい内容（チェックボックス）」
    *   送信先：[[entities/ooi-yuei|大井湧瑛]]のメールのみ。

3.  **コンテンツの「嘘」排除（1時間）**
    *   実績欄に「15事業並走」と書く。
    *   信頼性を担保するため、[[entities/testall|Testall]]や[[entities/gymee|Gymee]]のリンクをフッターに貼る（証拠提示）。
    *   技術的な詳細は書かない。「どうやるか」ではなく「何ができるか」のみ。

4.  **公開とSNSへの「放置」投稿（30分）**
    *   サイト公開。
    *   X（Twitter）に「[[entities/aipa-web|AIpa Web]]を公開しました。15事業の知見を凝縮。相談受け付けます。」と投稿。
    *   詳細な説明はしない。興味がある人にクリックさせる。

5.  **反応の監視と対応（残り時間）**
    *   メールが来たら、即座に電話または返信。
    *   来なかったら、来週は「コンテンツ（ブログ記事）」で補強する。

## 4. 完了確認
*   **測定基準**: [[entities/aipa-web|AIpa Web]]のLPにアクセスし、問い合わせフォームが正常に送信できること。
*   **成果基準**: 今週中に**1件以上の問い合わせ（または予約）**があること。
    *   *注：0件でも「公開」は完了とするが、次週のアクションは「集客施策」へ変更する。*

## 5. 比喩で表現
**「砂浜に『ホテル』の看板を立てる」**

今、[[entities/ooi-yuei|大井湧瑛]]は砂浜でホテルの設計図を描き、内装の素材を15事業から集めている。
しかし、客（クライアント）は設計図を見ても泊まりに来ない。
客は「ここにある」という**看板**を見て、初めて「泊まろう」と思う。

今週やることは、**「豪華なホテル」ではなく、「簡易テント＋看板」を建てること**。
テントは貧弱でいい。内装は後でやる。
重要なのは、**「ここに宿泊施設がある」という事実を、世界に通知すること**だ。

脳汁が出るのは、設計図を描く時ではなく、**「客がテントの前で立ち止まり、中を覗き込む瞬間」**だ。
その瞬間を今週作れ。
