# 今週やるべき「1個」：AIpa Webの「検索窓」を動かし、最初の1件を返す

## 1. 何を
[[entities/aipa-web|AIpa Web]]のフロントエンド検索機能を実装し、バックエンドのLLM呼び出しと連携させて、**実際にユーザーが入力したキーワードに対して、意味のある回答（または関連コンテンツへのリンク）が1件でも返ってくる状態**を作る。

## 2. なぜ今週
大井の弱点は「深掘り不足」だが、[[entities/aipa-web|AIpa Web]]は「証拠」がないと始まらない。
15事業並走の中で、コードが動いていないプロダクトは「妄想」に過ぎない。
今月フォーカスの「AIコンサル」や「SFC AO構想」においても、**「自分の頭で考えたこと」を外部に出力する媒体**としての[[entities/aipa-web|AIpa Web]]の価値は、中身が空っぽならゼロ。
今週中に「動く証拠」を出さないと、来週も「準備」で終わる。脳汁を出すには、ボタンを押して結果が出る快感が必要だ。

## 3. どうやって（5ステップ）

1.  **検索APIの定義**: `POST /api/search` を作成。パラメータは `query` (string) のみ。
2.  **LLM連携の最小実装**: 検索クエリをそのまま、または少し整形してLLMに投げる。返却値はJSON（`title`, `url`, `summary`）。
3.  **フロントエンドの検索窓**: 画面上部に検索バーを設置。EnterキーでAPIを叩く。
4.  **結果表示の簡素化**: 検索結果をリスト表示するだけのHTML/JS。デザインは後回し。
5.  **テスト実行**: 大井が実際に「大井湧瑛」や「AIpaX」などで検索し、結果が返ってくることを確認。

## 4. 完了確認
- [ ] ブラウザで[[entities/aipa-web|AIpa Web]]にアクセスし、検索窓に文字を入力してEnterを押す。
- [ ] 画面に検索結果が1件以上表示される。
- [ ] 表示された結果のリンクをクリックして、該当ページ（またはLLMの回答画面）に遷移できる。

## 5. 比喩で表現
**「脳みその外付けハードディスクの読み書きテスト」**

大井の脳はRAMのように高速だが、容量が有限だ。
[[entities/aipa-web|AIpa Web]]は、そのRAMに載せきれない知識や思考を保存し、必要に応じて呼び出す「外付けSSD」だ。
今週やることは、SSDを接続して、ファイルを読み書きできるか確認する「ドライバのインストール」に過ぎない。
中身がどうであれ、**「入力したら出力がある」**という物理的な接続が確立されれば、大井の思考は無限に拡張可能になる。
これをやらないと、大井は常に「自分の頭の中」だけで戦わなければならない。それは、15事業を並走する1人起業家にとって、最も非効率な「オフラインモード」だ。
