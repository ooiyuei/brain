# AIpaX 紹介スライド (15枚構成)

## 1. タイトル
**「AIツール」ではなく「会社別AI保存庫コア」を。**
中小企業の「知の資産」を、AIが使える形にする。

- 現在は15の事業を並走。月収100万円達成に向けてスプリント中。

## 3. 課題：中小企業の「AI導入」の壁
- 「AIって何？」ではなく「どう使う？」が壁。
- ツール導入は簡単だが、**「社内の知識」がAIに伝わっていない**。
- 営業のノウハウ、顧客情報、過去の商談記録が「個人の頭の中」や「紙」にある。
- AIは「正解」を出せない。入力される「情報」が古かったり、欠けていたりするから。

## 4. 解決策：会社別AI保存庫コア
**「まず保存庫を作る。その上にツールを載せる。」**
- 各社ごとに、資料・契約・FAQ・顧客情報をAIが検索・参照できる形に整理。
- これが「マザーボード」。
- その上に、必要な業務AIツール（拡張機能）を載せる。
- **毎回ゼロから作るのではなく、共通基盤＋拡張機能の構造。**

## 5. AIpaXの構造
```
会社別AI保存庫コア（マザーボード）
+
顧客ごとの業務AIツール（拡張機能）
```
- 保存庫コア = 会社ごとの資料・契約・料金・FAQ・顧客情報・顧客別条件・例外条件・やり取り履歴を、AIが検索・参照・再利用できる形に整理する共通基盤。
- その上に必要に応じて載せるツール群：
  - リアルタイム電話 / 営業回答支援
  - 社内AIチャット / 新人教育AI
  - 議事録整理AI
  - フォロー文生成AI
  - CS対応AI
  - 管理レポートAI
  - CRM補助ツール

## 6. 売り方の基本形
**保存庫コア構築 + 最初の業務ツール1つ**

| 顧客の課題 | 最初に提案するツール |
|----------|------------------|
| 営業電話で回答がズレる | リアルタイム電話/営業回答支援 |
| 新人がルールを何度も聞く | 社内AIチャット/新人教育AI |
| 商談後の整理が遅い | 議事録整理AI |
| 問い合わせ属人化 | CS対応AI |
| フォロー漏れ多い | フォロー文生成AI |
| 管理者が現場見たい | 管理レポートAI |

**最有力の初手商品**：保存庫コア + リアルタイム電話/営業回答支援ツール

## 9. 月額運用の範囲
### 含むもの
対象領域内の資料更新 / 入庫前処理 / 顧客確認後の保存庫反映 / 旧情報の非アクティブ化 / 検索条件の軽微調整 / 回答カードの軽微改善 / 利用ログ確認 / 回答できなかった質問の整理 / 軽微な不具合修正

## 10. 5原則
1. **保存庫コアとツール出力を混ぜない**
2. **顧客資料の正しさは顧客側が決める**
3. **AIが勝手に正式情報を作らない**
4. **不足・矛盾・最新版不明を検出して提示する**
5. **対象領域内の顧客別条件・例外条件は削らない**

## 11. 技術構成
- **Supabase**：構造化DB（companies/sources/source_chunks/knowledge_items/entities/relationships/events/reviews/versions/tool_logs/user_accounts/permissions）
- **OpenAI / File Search / Embedding**：資料検索・意味検索
- **アプリ**：URL+ログイン（初期）。デスクトップアプリ不要

## 12. OpenAI/Supabase 管理方式（顧客選択）
| 方式 | 内容 | 採用しやすさ |
|------|------|------------|
| A. 自社側管理（標準） | 導入早い・修正しやすい・実費管理あり | デフォルト |
| B. 顧客側管理 | 機密に強い・情シスある会社向け | 追加費用オプション |
| C. 自社→顧客側移管 | 最初早く導入、後で移管 | 別料金 |
