# FishOps Agent 10人初心者体験イベント 設計書

## 目的と戦略

[[entities/ipak-fishing]] (FishOps Agent) のMVP検証と、[[entities/aipax-school]] や [[entities/eemus]] への橋渡しを兼ねた「釣り体験イベント」を設計する。

**核心**: 単なる「釣り」ではない。「釣れなかったデータ」をどう取るか、それをどうAIで可視化するか、というFishOps Agentの核（Value Prop）を体感させること。
**大井の現状**: 月収100万期限（6/21）。このイベント自体で即金性は薄い（むしろコスト）。だが、**「水産業界のDX」という切り口で、漁協・自治体・大手釣具店への営業資料（Proof of Concept）として使える**のが最大の資産。
**明日使う1人**: 大井湧瑛（営業用スライドの「実証データ」として）。

## イベント概要

| 項目 | 内容 |
|------|------|
| **名称** | FishOps Agent 検証イベント #01: 「釣れなかった」をデータにする |
| **日時** | 2026-06-13（土）10:00-15:00（雨天決行・屋内併用） |
| **場所** | 静岡県内の管理釣り場（例：富士山麓の某フィールド、または浜名湖近郊） |
| **人数** | 10名（募集：学生・大学生・社会人1年目・釣り初心者） |

| **目的** | 1. FishOps Agent MVP（釣果・不満データ収集）の検証<br>2. 漁協/自治体向け提案用の「リアルな失敗データ」獲得<br>3. EEMUS連携候補の発掘 |

## 参加者募集ターゲット（明日使う1人視点）

1. **[[entities/eemus]] 連携候補**: 静岡の高校生・大学生。地元企業インターンに興味がある層。
2. **[[entities/aipax-school]] 潜在生徒**: AIに興味はあるが、何をしていいか分からない中高生・保護者。
3. **一般釣り初心者**: 「釣りたいけど、何を揃えればいいか分からない」層。FishOpsの「手ぶら釣り」コンセプトの検証。

## イベントフロー（5時間）

### 1. 受付・データセットアップ（10:00-10:30）
- **QRチェックイン**: 参加者全員がLINE公式アカウントにQRで登録（FishOps MVP1の基盤）。
- **デバイス配布**: 専用タグ（または既存のウェアラブル）を装着。釣果写真用スマホ貸出（必要に応じて）。
- **ヒアリングシート**: 「なぜ釣りを始めたか」「過去の失敗体験」を紙orスマホで入力。

### 2. 釣り体験A（10:30-12:00）
- **ルール**: 3時間枠の前半。
- **FishOps要素**:
  - 釣れた魚の種類・数・サイズを写真撮影してLINEに送信（自動記録）。
  - **釣れなかった人**は「なぜ釣れなかったか」（場所・餌・時間）をフォームで入力。
  - 運営側（大井）が「放流数」「水温」「天気」を記録。

### 3. ランチ・データ可視化体験（12:00-13:00）
- **昼食**: 地元食材を使ったBBQまたは弁当。
- **リアルタイムダッシュボード**: 会場にプロジェクターor大型モニターを設置。
  - 「現在の釣果ランキング」
  - 「釣れなかった理由のワードクラウド」（例：「浅い」「餌が動かない」）
  - 「今日の水温vs釣果相関」
- **大井のプレゼン**: 「このデータが溜まると、明日の釣果予測や最適放流が可能になる」と説明。FishOps Agentの価値を体感させる。

### 4. 釣り体験B（13:00-14:30）
- **データ活用実験**: 午前中のデータに基づき、運営側が「ここが狙い目」と提案。
- **参加者行動**: 提案された場所へ移動し、釣果を比較。
- **フィードバック収集**: 「提案は役立ったか？」「逆に迷った点は？」をリアルタイムで収集。

### 5. 振り返り・クロージング（14:30-15:00）
- **リザルト発表**: 各人の釣果レポートをLINEで配信。
- **アンケート**: 「FishOps Agentのようなサービスがあれば使うか？」「月額いくらまで払うか？」
- **EEMUS/AIpaX School 紹介**: 「地元企業で働くには？」（EEMUS）、「AIスキルで未来を変える」（AIpaX School）の紹介。

## 収集すべきデータ（FishOps Agent MVP1の要件）

1. **釣果データ**: 魚種、サイズ、釣れた時間、使用餌。
2. **失敗データ**: 釣れなかった理由、滞在時間、満足度。
3. **運営データ**: 放流数、水温、天気、来場者数、初心者比率。
4. **心理データ**: 「釣れなかった時の気分」「次に釣りに来るか？」

## 大井のアクション（今週〜来週）

1. **釣り場交渉**: 静岡県内の管理釣り場3社に連絡。「データ検証のための協賛パートナー」を募集。
2. **LINE公式アカウント設定**: FishOps MVP1の基盤となるLINE公式アカウントの作成とQRコード生成。
3. **参加者募集**: [[entities/eemus]] のSNS、[[entities/aipax-school]] の待機者リスト、地元の釣りコミュニティへ告知。
4. **データ収集フォーム作成**: Googleフォーム（釣果、失敗理由、満足度）。

## 注意点

- **安全確保**: 初心者向けなので、ライフジャケットの着用義務、水深の浅いエリアでのみ実施。
- **データプライバシー**: 釣果写真のSNS投稿は任意とする。個人情報は漁協への提供は匿名化。
- **雨天対策**: 屋内の待機スペースと、雨天時の代替プログラム（例：仕掛け作りワークショップ）を準備。

## 結論

このイベントは、FishOps Agentの「データ収集」を目的とするが、**「釣り」という体験を通じて、参加者にFishOpsの価値を体感させる**ことが重要。
大井は、このイベントを「漁協への営業デモ」として位置づけ、[[entities/eemus]] や [[entities/aipax-school]] への集客も同時に図る。
