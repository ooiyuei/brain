主力5事業を横断した「紹介プログラム（リファラルプログラム）」の設計案を提案します。

このプログラムは、**「紹介者の動機付け（報酬）」**と**「被紹介者のハードル低下（特典）」**、そして**「法的リスクの回避（契約書）」**の3要素をバランスよく設計することが重要です。

※※注意：以下の内容は一般的なビジネスモデルに基づくテンプレート・指針です。実際の導入にあたっては、貴社の財務状況、業界法規、および弁護士による法的審査を必ず行ってください。

---

# 主力5事業横断型 紹介プログラム設計書

## 1. プログラムの基本概念
*   **対象事業:** 主力5事業（例：SaaS製品A、コンサルティングB、クラウドインフラC、教育プラットフォームD、ECソリューションE）
*   **基本方針:** 事業間で共通のルールを適用し、管理コストを削減するとともに、顧客のライフサイクル全体での囲い込みを図る。
*   **対象者:** 既存顧客、パートナー企業、一般ユーザー（B2B/B2Cに応じて調整）

---

## 2. 紹介者報酬率（インセンティブ設計）

報酬は「初回契約金」または「継続課金」のいずれか、または両方をベースに設定します。横断プログラムでは、**「事業間クロスセル」を促進するため、高付加価値事業ほど報酬率を高くする**のが一般的です。

### 案A: 固定金額型（シンプル・認知度重視）
| 事業カテゴリ | 例 | 紹介者報酬（1件あたり） | 備考 |
| :--- | :--- | :--- | :--- |
| **低単価・高頻度** | 教育プラットフォームD | ¥5,000 - ¥10,000 | 導入ハードルを下げるため |
| **中単価・標準** | SaaS製品A / ECソリューションE | ¥20,000 - ¥50,000 | 標準的なリファラル報酬 |
| **高単価・長期** | コンサルティングB / クラウドインフラC | ¥100,000 - ¥300,000 | 契約金額の10%〜20%程度 |

*   **支払い条件:** 被紹介者の契約が成立し、かつ**初回決済が完了してから30日後**に支払う（キャンセルリスク回避）。
*   **上限:** 1人あたり年間¥1,000,000までなど、不正防止の上限を設ける。

### 案B: 継続コミッション型（LTV重視・SaaS向け）
*   **報酬率:** 被紹介者の**月間課金額の10%〜20%**を、継続期間（例：最大12ヶ月間）にわたり支払う。
*   **メリット:** 紹介者が長期的な顧客関係維持に関心を持ち、質の高い紹介を促す。
*   **横断適用:** 5事業すべてで「月間課金額の15%」など統一ルールにすると管理が容易。

### 案C: 混合型（推奨）
*   **初回ボーナス:** 契約成立で¥50,000（即金または翌月払い）
*   **継続ボーナス:** 月間課金額の10%（最大6ヶ月間）
*   **クロスセルボーナス:** 紹介された顧客が、別の主力事業を追加契約した場合、紹介者に追加で¥30,000を支払う。

---

## 3. 被紹介者特典（新規顧客獲得促進）

紹介された側（被紹介者）にも魅力がないと契約に繋がらないため、強力な特典を用意します。

| 特典タイプ | 内容例 | 目的 |
| :--- | :--- | :--- |
| **割引特典** | 初月無料 / 初回契約料30%OFF / 年額契約で2ヶ月分無料 | 導入の心理的・金銭的ハードルを下げる |
| **機能アップ** | 通常プランから上位プランを1ヶ月間無料体験 | 高単価プランへのアップセル機会を作る |
| **サービス特典** | 初期設定サポート無料 / 専門コンサルタント1時間無料相談 | コンサルティング系事業での差別化 |
| **ギフト券** | Amazonギフト券¥5,000分 | 即効性のあるインセンティブ（B2C向け） |

**横断ルール:**
*   5事業すべてで「初月無料」または「30%OFF」など、**統一された特典枠**を用意することで、顧客が迷わず契約できるようにする。
*   特典は**紹介コード入力時**に自動適用される仕組みにする。

---

## 4. 契約書テンプレート（紹介プログラム規約）

以下の条項を「紹介プログラム規約」として、紹介者（紹介者）と貴社（運営者）の間で締結します。

---

### **紹介プログラム参加規約**

**第1条（目的）**
本規約は、[会社名]（以下「当社」という）が実施する紹介プログラム（以下「本プログラム」という）への参加条件、紹介者の権利・義務、および報酬の支払い条件等を定めるものとする。

**第2条（対象事業）**
本プログラムの対象となる事業は、以下の主力5事業とする。
1. [事業A名]
2. [事業B名]
3. [事業C名]
4. [事業D名]
5. [事業E名]
（※必要に応じて事業名を記載）

**第3条（紹介の定義）**
「紹介」とは、紹介者が被紹介者に対し、本プログラム専用の紹介リンクまたは紹介コードを伝達し、当該被紹介者が当社が提供する対象事業の新規契約を締結することをいう。

**第4条（報酬の算定と支払い）**
1. 紹介報酬は、被紹介者の契約が成立し、かつ初回決済が完了したことを条件として算定する。
2. 報酬額は、第2条で定める対象事業および契約種別に応じて、当社が別途定める「報酬率表」による。
3. 報酬の支払いは、条件を満たした月の翌月末日に、銀行振込にて行う。
4. 被紹介者の契約解除、キャンセル、または代金未払いが生じた場合、既に支払った報酬の返還を求める権利を当社は留保する。

**第5条（被紹介者特典）**
被紹介者は、紹介コードを提示することで、当社が定める新規顧客特典（割引、無料体験等）を受けることができる。ただし、他のキャンペーンと重複適用できない場合がある。

**第6条（禁止行為）**
紹介者は、以下の行為を行ってはならない。違反した場合、報酬の支払いを停止し、既に支払われた報酬の返還を求めることができる。
1. 虚偽の情報や不正な手段による紹介。
2. 自らのアカウントや関連アカウントを用いた自己紹介。
3. 他社との提携プログラムとの併用（競合するプログラム）。
4. 本プログラムの内容を無断で第三者に開示すること。

**第7条（規約の変更）**
当社は、必要に応じて本規約および報酬率を変更することができる。変更後は、当社ウェブサイト等にて公告し、効果を生じるものとする。

**第8条（準拠法及び管轄裁判所）**
本規約に定めのない事項については、日本法に従い、[会社所在地]を管轄する裁判所を専属的合意管轄裁判所とする。

## 5. 実装・運用上のポイント

1.  **管理プラットフォームの選定**
    *   手動管理は非効率です。ReferralCandy、PartnerStack、または自社開発のダッシュボードを活用し、クリック・コンバージョン・報酬額を自動集計してください。
2.  **クロスセルのトラッキング**
    *   5事業横断の場合、「A事業で紹介された顧客が、後からB事業も契約した」場合の報酬分配ルールを明確にしてください（例：最初の紹介者に全報酬、または折半）。
3.  **コンプライアンス対応**
    *   **景品表示法:** 報酬額や特典内容を正確に開示し、誤解を招かない表現を使用してください。
    *   **個人情報保護法:** 紹介者と被紹介者の情報を適切に管理し、同意なく第三者提供しないよう注意してください。
4.  **周知・啓発**
    *   メールマガジン、Webサイトバナー、営業担当者からの案内など、多角的なチャネルで紹介プログラムを告知してください。
