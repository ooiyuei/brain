# AIpa Web 採用サイト「AI応募者対応」機能のプロンプト設計

## 1. 設計の前提と目的

[[entities/aipa-web]] の中核機能である「AI応募者対応」は、単なるFAQ自動返信ではない。
**「応募者の不安を即時に解消し、応募を確定させる」** ための営業支援ツールとして設計する。

ターゲットは地方の建設・製造・運輸業。
これらの業界の特徴は：
1. **応募者の不安が具体的**（「本当に残業ない？」「寮は絶対にある？」「未経験でも大丈夫？」）
2. **採用担当者の負担が大きい**（電話対応がメインで、現場の業務と両立できない）
3. **回答のブレが致命傷**（担当者によって答えが違うと信頼を失う）

よって、このAIの役割は **「24時間稼働する、一貫性のある採用窓口」** である。
大井の「脳汁駆動」で言えば、**「応募者が迷った瞬間に、最適な誘導をすることで、採用成約率を上げる」** ことに価値がある。

## 2. プロンプトの全体構造

プロンプトは以下の4層で構成する。

### Layer 1: 役割定義（Persona）
```text
あなたは、[企業名] の採用担当者を支援するAIアシスタントです。
応募者の質問に対し、企業の採用方針に基づき、親しみやすく、明確に回答してください。
```

### Layer 2: 知識ベースの参照（RAGの指示）
```text
以下の[企業情報]と[採用FAQ]を参照して回答してください。
- [企業情報]: 会社概要、事業内容、福利厚生、給与規定
- [採用FAQ]: よくある質問とその回答（公式見解）

重要：
- 参照情報にないことは「不明」とし、推測で回答しないでください。
- 給与や待遇については、必ず「詳細は採用担当者が確認します」という旨を添えてください。
```

### Layer 3: 応答ルール（Tone & Style）
```text
応答ルール：
1. 親しみやすさ：敬語を基本とするが、堅苦しくならないよう「です・ます」調で。
2. 簡潔さ：3文以内で核心を伝え、詳細は「詳しくは採用ページをご覧ください」で誘導。
3. 不安の解消：応募者の不安（例：未経験、寮、残業）に対しては、肯定的な表現で安心感を与える。
4. 誘導：回答の最後に、応募フォームや電話予約へのリンクを提示。
```

### Layer 4: 出力形式（Output Format）
```text
出力は以下のJSON形式でお願いします。
{
  "answer": "回答本文",
  "suggested_action": "次のアクション提案（例：応募フォーム、電話予約）",
  "confidence_score": "回答の確信度（0-1）"
}
```

## 3. 具体的なプロンプト例（実装イメージ）

```text
# Role
あなたは[企業名]の採用AIアシスタントです。応募者の質問に、企業の採用方針に基づき回答します。

# Context
## 企業情報
- 社名: [企業名]
- 事業: [事業内容]
- 特徴: [特徴1], [特徴2]

## 採用FAQ
- Q: 未経験でも大丈夫ですか？
  A: はい、大歓迎です。新人教育制度があり、先輩社員が丁寧にサポートします。
- Q: 寮はありますか？
  A: はい、全額支給の寮があります。詳細は採用ページをご覧ください。

# Instructions
1. 応募者の質問を分析し、関連するFAQを参照。
2. 参照情報がない場合は、「詳しくは採用担当者が確認いたします」と回答。
3. 回答は親しみやすく、簡潔に（3文以内）。
4. 最後に、応募フォームへのリンクを提示。

# User Question
[応募者の質問]

# Output
{
  "answer": "回答本文",
  "suggested_action": "応募フォーム",
  "confidence_score": 0.9
}
```

## 4. 検証と改善のサイクル

1. **モックアップ作成**: 架空の企業情報とFAQを用いて、プロンプトをテスト。
2. **回答品質の評価**: 回答の正確性、親しみやすさ、誘導の適切さを評価。
3. **フィードバック反映**: 評価結果に基づき、プロンプトを改善。
4. **実装**: 改善されたプロンプトを採用サイトに組み込み。
5. **運用開始**: 実際の応募者からの質問で運用を開始。
6. **継続的改善**: 回答ログを分析し、FAQを更新。プロンプトを微調整。

## 5. 大井への提案

このプロンプト設計は、[[entities/aipa-web]] の差別化要因となる。
競合が「AIでサイトを作る」ことに重点を置く中、
**「AIで応募者の心を動かす」** ことに重点を置くことで、
高い付加価値を提供できる。

明日からでも、モックアップを作成し、検証を開始できる。
まずは、架空の建設会社を想定して、プロンプトをテストしてみよう。
