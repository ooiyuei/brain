# AIpaX school 1年カリキュラム 5モジュール完成版

本ドキュメントは、中高生向けオンラインAIスクール「AIpaX school」の1年間（5モジュール×2ヶ月）のカリキュラム詳細を示す。本カリキュラムは、大井がHR高等学院で習得した思考フレームワーク（F1〜F5）を基盤とし、現役AI起業家による実践的な「バイブコーディング」体験と、卒業後の事業化（マネタイズ）までを見据えた構成となっている。

## モジュール1：AIマインドセットとバイブコーディング体験（Month 1-2）

**学習目標**: AIを「道具」ではなく「思考の拡張」として捉え、プロンプトエンジニアリングの基礎と、AI支援による高速なコード生成（バイブコーディング）の感覚を身につける。

### 週次トピック（8週間）
1.  **AIリテラシーと倫理**: AIの限界、ハルシネーション、著作権、プライバシー。AI時代の「人間にしかできないこと」の定義。
2.  **プロンプトエンジニアリング基礎**: 役割付与、文脈提供、出力形式指定。F1「ストーリーテリング3軸」の導入（普通×武器×提案）。
3.  **チャットAIの高度活用**: ChatGPT/Claudeを用いたリサーチ、要約、ブレインストーミング。メタプロンプトの作成。
4.  **バイブコーディング入門**: Cursor/Replit環境構築。自然言語でコードを生成するプロセスの理解。
5.  **コードのデバッグと修正**: AIが生成したコードの検証方法。エラーメッセージの読み解き方とAIへの指示出し。
6.  **小規模スクリプト制作**: 日常のタスクを自動化するスクリプト（例：ファイル整理、データ変換）の作成。
7.  **AIツール連携**: Notion AIやGamma AIとの連携。情報収集からプレゼン資料作成までのワークフロー構築。
8.  **モジュール1総括と振り返り**: 作成した成果物の共有。AIとの協業における「自分らしさ」の発見。

### 課題
*   **最終課題**: 「自分の日常を10倍効率化するAIワークフロー」の設計と実装。
*   **提出物**: 使用したプロンプト履歴、生成されたコード、改善点の記録。

### 必要ツール
*   ChatGPT Plus / Claude Pro
*   Cursor / Replit
*   Notion / Gamma

### 評価方法
*   プロンプトの精度（指示の明確さ）
*   生成コードの動作確認
*   ワークフローの効率化率の定量的評価

## モジュール2：日常業務自動化とチーム合意形成（Month 3-4）

**学習目標**: AIを活用したデータ処理と自動化を学び、F2「合意形成4ステップ」を用いてチームでアイデアを具現化する力を養う。

### 週次トピック（8週間）
1.  **データ分析基礎**: CSV/Excelデータの読み込み、Pandasライブラリの基礎。AIによるデータ可視化。
2.  **APIの基礎**: REST APIの概念。外部サービス（天気、為替、SNS）からデータを取得する仕組み。
3.  **自動化スクリプトの深化**: Webスクレイピングの基礎と倫理。定期実行タスクの設定。
4.  **チームプロジェクト開始**: F2「合意形成4ステップ」の導入。課題の定義、役割分担、合意形成プロセス。
5.  **アイデアの構造化**: ブレインストーミング結果の整理。マインドマップツールの活用。
6.  **プロトタイピング**: FigmaやFramer AIを用いたUI/UXのアイデア出し。ワイヤーフレームの作成。
7.  **バックエンド連携**: フロントエンドとバックエンドの接続。データベースの基礎（SQLite/Firebase）。
8.  **中間発表とフィードバック**: チーム間の進捗共有。他チームからの質問と回答を通じた相互学習。

### 課題
*   **最終課題**: チームで「AIを活用した解決策」を企画し、プロトタイプを作成。
*   **提出物**: プロジェクト企画書、プロトタイプURL、合意形成プロセスの記録。

### 必要ツール
*   Python (Pandas, Requests)
*   Figma / Framer AI
*   Firebase / Supabase
*   Miro / FigJam

### 評価方法
*   チーム内の合意形成プロセスの適切さ
*   プロトタイプの機能性とUI/UX
*   データ処理の正確性

## モジュール3：エンジニアリング思考と個人探究テーマ設定（Month 5-6）

**学習目標**: F3「11項目ロジックモデル」を用いて個人探究テーマを設定し、技術的な課題解決能力を高める。

### 週次トピック（8週間）
1.  **ロジックモデルの理解**: F3「11項目ロジックモデル」の解説。インプット、アクティビティ、アウトプット、アウトカムの関係性。
2.  **課題発見と定義**: 社会課題や個人の悩みから、AIで解決できる課題の抽出。問題文の具体化。
3.  **技術選定とアーキテクチャ**: 使用する技術スタックの決定。クラウドサービス（AWS/GCP）の基礎。
4.  **データベース設計**: リレーショナルデータベースの設計。ER図の作成と正規化。
5.  **認証とセキュリティ**: ユーザー認証の実装。OAuth2.0の基礎。セキュリティベストプラクティス。
6.  **API開発**: RESTful APIの設計と実装。ドキュメント作成（Swagger/OpenAPI）。
7.  **テスト駆動開発**: ユニットテストの基礎。AIによるテストケースの生成と実行。
8.  **個人探究テーマの確定**: F3ロジックモデルに基づく最終的なテーマ決定。技術的課題の特定。

### 課題
*   **最終課題**: 「個人探究テーマ」の企画書と技術アーキテクチャ図の提出。
*   **提出物**: ロジックモデル図、技術選定理由書、ER図。

### 必要ツール
*   AWS/GCP（無料枠）
*   Swagger / Postman
*   GitHub / GitLab

### 評価方法
*   課題定義の明確さ
*   技術選定の妥当性
*   ロジックモデルの整合性

## モジュール4：本格アプリ開発とサービス企画（Month 7-8）

**学習目標**: F4「CHINTAI型5項目企画」を用いてサービスを企画し、フルスタックのWebアプリケーションを開発・デプロイする。

### 週次トピック（8週間）
1.  **サービス企画の深化**: F4「CHINTAI型5項目企画」の適用。ターゲット、価値提案、収益モデルの具体化。
2.  **フロントエンド開発**: React/Vue.jsの基礎。コンポーネント指向の理解。
3.  **バックエンド開発**: Node.js/Python(FastAPI)を用いたサーバーサイドロジックの実装。
4.  **データベース連携**: リアルタイムデータ処理。WebSocketsの基礎。
5.  **デプロイとCI/CD**: Vercel/Netlifyへのデプロイ。GitHub Actionsを用いた自動デプロイ。
6.  **UI/UXの磨き上げ**: アクセシビリティ、レスポンシブデザイン。ユーザーテストの実施。
7.  **マーケティング基礎**: ランディングページの作成。SEOの基礎。SNSでのプロモーション戦略。
8.  **サービス公開準備**: ドメイン取得、SSL証明書、プライバシーポリシーの作成。

### 課題
*   **最終課題**: 公開可能なWebアプリケーションの完成と公開。
*   **提出物**: 公開URL、サービス企画書、ユーザーテスト結果。

### 必要ツール
*   React / Vue.js
*   Node.js / FastAPI
*   Vercel / Netlify
*   GitHub Actions

### 評価方法
*   アプリケーションの完成度とバグの少なさ
*   サービス企画書の説得力
*   デプロイの円滑さ

## モジュール5：データ分析とマネタイズ（Month 9-12）

**学習目標**: F5「産業課題×AI 4項目」を用いて業界分析を行い、収集したデータを分析して収益化（マネタイズ）の戦略を立て、卒業課題を発表する。

### 週次トピック（8週間）
1.  **産業分析の基礎**: F5「産業課題×AI 4項目」の適用。市場規模、競合分析、参入障壁の評価。
2.  **データ収集とクリーニング**: Webスクレイピングの応用。APIからの大量データ取得。データの前処理。
3.  **統計分析と可視化**: 記述統計、相関分析。Tableau/Power BIを用いたダッシュボード作成。
4.  **機械学習入門**: 教師あり学習の基礎。scikit-learnを用いた予測モデルの構築。
5.  **収益モデルの設計**: 課金モデル（サブスクリプション、広告、手数料）の比較と選択。
6.  **財務基礎**: 損益計算書の作成。CAC（顧客獲得コスト）、LTV（顧客生涯価値）の計算。
7.  **卒業課題の準備**: 全モジュールの成果の統合。プレゼンテーション資料の作成。
8.  **卒業発表会**: 企業関係者、保護者、他受講生を対象とした発表。フィードバックの収集。

### 課題
*   **最終課題**: 卒業課題としての「AI活用事業計画書」と「プロダクト」の完成。
*   **提出物**: 事業計画書、データ分析レポート、プレゼンテーション資料。

### 必要ツール
*   Tableau / Power BI
*   scikit-learn
*   Excel / Google Sheets

### 評価方法
*   産業分析の深さと妥当性
*   データ分析の結果とインサイト
*   事業計画の実現可能性
*   プレゼンテーションの質

## 総括

本カリキュラムは、単なるツールの操作方法の習得に留まらず、AIを駆使して社会課題を解決し、価値を生み出すまでの一連のプロセスを体験することで、受講生の「AIリテラシー」と「起業家マインドセット」の育成を目指す。各モジュールでHR高等学院由来の思考フレームワークを統合することで、論理的かつ創造的な問題解決能力を養う。
