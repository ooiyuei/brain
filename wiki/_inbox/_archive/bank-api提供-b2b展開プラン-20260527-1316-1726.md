# API提供 B2B展開プラン：薬機チェック特化型

## 1. 結論：何を作るか

**「AIpaX 保存庫コア」の上に載せる「薬機法（医薬品医療機器等法）違反チェックAPI」** を商品化する。

*   **製品名**: AIpaX Compliance API（仮）
*   **ターゲット**: 医薬品・医療機器・化粧品を扱うECサイト運営会社、広告代理店、マーケティング部門
*   **価値**: 「薬機法違反の広告文面」を、投稿前にAIがリアルタイムで検知・修正提案する
*   **価格感**: 初期構築費 50〜80万円 / 月額 5〜10万円（API利用量依存）

**なぜ今か？**
2026年現在、厚生労働省の薬機法違反摘発は年々厳格化しており、特にSNS広告（X, Instagram, TikTok）での「効能効果の誇大表現」や「医療機器の適応外表示」が狙われている。企業側は「専門の法務担当」を置けない中小〜中堅企業が多く、**「AIpaXの保存庫（社内ルール）」と「薬機法」とを掛け合わせたチェック機能」の需要が急増している。**

## 2. 市場と競合の現実

### 競合実態
*   **大手法務SaaS（例: LexisNexis等）**: 価格が数百万円〜で、中小企業には手が届かない。また、AIのリアルタイム性よりも「文書レビュー」が主。
*   **一般的なAIチェックツール**: 薬機法特有の「表現のグレーゾーン」（例：「〜のような効果があります」vs「〜の効果があります」）の判定精度が低い。
*   **AIpaXの優位性**:
    1.  **保存庫連携**: 顧客企業の「承認済み文面パターン」を保存庫に持たせ、それとの差分でチェックできる。
    2.  **既存顧客の横展開**: 既にAIpaX契約済の4社の中に、薬機関連の顧客が潜んでいる可能性が高い（または、AIpaXの営業で獲得した顧客に「法務チェック」をオファリングできる）。

### 市場規模感
*   医薬品・医療機器・化粧品関連のEC市場は巨大だが、**「コンプライアンス違反リスク」への支出意欲**は非常に高い。
*   違反1件で数十万〜数百万円の損害賠償や行政指導リスクがあるため、月額数万円の保険として捉えられやすい。

## 3. プロダクト設計：APIとしてどう売るか

### 機能要件（MVP）
1.  **テキスト入力**: 広告文面、SNS投稿文、LPコピーなどをAPIに送信。
2.  **違反検知**: 薬機法第4条（誇大広告）、第6条（適応外表示）、第7条（効果・効能の表示）に抵触する可能性のある箇所をハイライト。
3.  **修正提案**: 「『治ります』→『サポートします』に変更推奨」など、具体的な修正案を返す。
4.  **保存庫連携**: 顧客企業の「承認済み表現リスト」を参照し、それと矛盾しないかチェック。

### 技術スタック
*   **基盤**: [[entities/aipax|AIpaX]] 保存庫コア（顧客固有のルール管理用）
*   **AIモデル**: Claude Sonnet（文脈理解に優れるため、グレーゾーンの判定に使用）
*   **API形式**: RESTful API（JSON形式でテキストを送信し、違反フラグと修正案を返す）
*   **連携先**: ECプラットフォーム（Shopify, BASE, EC-CUBE）、CMS（WordPress）、SNS投稿ツール

### 価格設計
*   **初期構築費**: 50〜80万円
    *   顧客固有の「承認済み表現リスト」の保存庫への組み込み
    *   API連携の設定
*   **月額利用料**: 5〜10万円
    *   基本料金 + API呼び出し回数（例：10,000回/月まで含む、超えた分は追加課金）
    *   **根拠**: 法務担当者の時給（5,000円/時）で月10時間分の作業を代替する価値。

## 4. GTM戦略：誰に、どう売るか

### ターゲットセグメント
1.  **既存AIpaX顧客への横展開（最優先）**
    *   既にAIpaX契約済の4社の中に、医薬品・医療機器・化粧品関連の顧客がいるか確認。
    *   もしいれば、既存の「保存庫」に「薬機チェック」モジュールを追加する形で提案。
    *   **メリット**: 契約交渉が楽、導入コストが低い、信頼関係がある。
2.  **医薬品・医療機器EC運営会社**
    *   中堅〜大企業のEC部門。
    *   **アプローチ**: 「薬機法違反リスクをAIでリアルタイムにチェック」を訴求。
    *   **チャネル**: 医薬品流通協会、EC協会、法務コンサルティング会社との提携。
3.  **広告代理店・マーケティングエージェンシー**
    *   顧客企業の広告文面をチェックする業務を外部委託されている会社。
    *   **アプローチ**: 「チェック業務の自動化」でコスト削減を提案。

### 営業プロセス
1.  **ヒアリング**: 顧客の「承認フロー」を把握。誰が、何を、どのようにチェックしているか。
2.  **PoC（Proof of Concept）**: 既存の広告文面100件をAPIにかけ、違反検知精度を確認。
3.  **提案**: 検知結果と、保存庫連携による「自動化のメリット」を提示。
4.  **契約**: 初期構築費 + 月額契約。

## 5. リスクと対策

### リスク1: 誤検知・見逃し
*   **対策**: AIの出力は「提案」であり、最終判断は人間が行うことを明記。契約書に「AIの判断による損害に対する責任は免れる」条項を入れる。
*   **対策**: 保存庫に「承認済み表現リスト」を持たせ、それとの差分でチェックすることで、誤検知を減らす。

### リスク2: 法改正への追従
*   **対策**: 薬機法の改正情報は、AIpaXの保存庫に定期インポートし、チェックルールを自動更新する仕組みを作る。
*   **対策**: 法務専門家との提携（外部アドバイザー）を検討。

### リスク3: 競合の参入
*   **対策**: 「保存庫連携」による顧客固有のルール管理を差別化ポイントとする。一般的なAIチェックツールではできない「社内ルールとの整合性チェック」が強み。

## 6. 3ヶ月以内の検証アクション

### 1ヶ月目：要件定義とPoC設計
*   **アクション**: 既存AIpaX顧客4社に、薬機関連の顧客がいるか確認。
*   **アクション**: 薬機法の違反事例100件を収集し、テストデータを作成。
*   **アクション**: Claude Sonnetを用いたプロンプト設計（違反検知の精度向上）。

### 2ヶ月目：PoC実施と精度検証
*   **アクション**: テストデータを用いてAPIの検知精度を検証。
*   **アクション**: 既存顧客（もしいれば）または見込み顧客2社にPoCを提供。
*   **アクション**: 検知結果のフィードバックを受け、プロンプトと保存庫ルールを調整。

### 3ヶ月目：契約と本格導入
*   **アクション**: PoC結果を基に、契約提案。
*   **アクション**: 最初の顧客にAPIを提供し、運用開始。
*   **アクション**: 運用結果を記録し、次の顧客への提案資料として活用。

## 7. 大井湧瑛としての判断

*   **なぜこのプロジェクトか？**
    *   AIpaXの「保存庫」の価値を最大化できる。
    *   薬機法は「専門知識」が必要な分野であり、AIpaXの「社内ルール管理」の強みが活きる。
    *   既存顧客への横展開が可能で、営業コストが低い。
*   **なぜ今か？**
    *   薬機法の摘発が厳格化しており、企業のニーズが高い。
    *   AIpaXの保存庫が成熟し、API提供の基盤が整った。
*   **次の一手**:
    *   既存AIpaX顧客4社へのヒアリング。
    *   薬機法違反事例の収集とテストデータ作成。
