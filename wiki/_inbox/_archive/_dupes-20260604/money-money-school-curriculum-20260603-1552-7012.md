# AIpaX school 1年カリキュラム (5モジュール構造)

## 📌 Module 1: AIマインドセット + バイブ体験 (Month 1-2)

### 学習目標
- AIを「手足」のように使えるマインドを育てる
- バイブコーディング体験を通じてAIの可能性を実感
- プロンプト設計の基礎を習得

### 週次トピック
1. AIとは何か？（AIの定義・歴史・現状）
2. AIの3つの使い方（代替・拡張・創造）
3. プロンプト設計の基本（目的・対象・形式）
4. AIツールの選定（ChatGPT / Claude / Canva AI）
5. バイブコーディング体験（Replit / Cursor）
6. AIと人間の協働（AIの限界と補完）
7. プロンプトの実践（課題解決型プロンプト作成）
8. AIマインドセットのまとめ（自己評価・振り返り）

### 課題と発表
- プロンプト設計課題：「AIで何ができるか？」の3つの例
- バイブコーディング体験：簡単なアプリ制作（例：QRコード生成）

### 必要ツール
- Antigravity
- Cursor
- Claude
- Figma

### 評価方法
- プロンプト設計の完成度
- バイブコーディングの成果物
- 週次振り返りの質

---

## 📌 Module 2: 日常業務自動化 (Month 3-4)

### 学習目標
- 日常業務をAIで自動化するスキルを習得
- チームでアイデアを共有・合意形成する能力を育てる
- プロジェクト管理の基礎を学ぶ

### 週次トピック
1. 日常業務のAI化（例：メール自動作成・スケジュール管理）
2. チームでのアイデア共有（F1 ストーリーテリング 3軸）
3. チーム合意形成（F2 合意形成 4ステップ）
4. プロジェクト管理の基礎（Ganttチャート・タスク管理）
5. AIツールの連携（例：Notion AI + Google Calendar）
6. 自動化の実践（スクリプト作成・ワークフロー設計）
7. チームでの成果発表（プレゼン作成・資料作成）
8. 自動化のまとめ（自己評価・振り返り）

### 課題と発表
- 自動化課題：日常業務の1つをAIで自動化
- チーム合意形成：グループでアイデアを共有・合意形成

### 必要ツール
- Antigravity
- Cursor
- Claude
- Notion
- Google Calendar

### 評価方法
- 自動化の完成度
- チーム合意形成の成果
- プレゼン資料の質

## 📌 Module 3: エンジニアリング思考 + 個人探究テーマ設定 (Month 5-6)

### 学習目標
- エンジニアリング思考を身につける
- 個人探究テーマを設定し、課題を明確にする
- ロジックモデルを活用した問題解決を学ぶ

### 週次トピック
1. エンジニアリング思考の基本（問題解決のステップ）
2. ロジックモデルの理解（F3 11項目ロジックモデル）
3. 個人探究テーマの設定（目的・対象・形式）
4. ロジックモデルの実践（課題解決型ロジック作成）
5. プロジェクトの構築（スケジュール・リソース管理）
6. ロジックモデルの検証（フィードバック・修正）
7. 個人探究テーマのまとめ（自己評価・振り返り）
8. ロジックモデルの応用（実践型課題）

### 課題と発表
- 個人探究テーマ設定：自分の興味に合わせた課題
- ロジックモデルの実践：課題解決型ロジック作成

### 必要ツール
- Antigravity
- Cursor
- Claude
- Notion

### 評価方法
- 個人探究テーマの明確さ
- ロジックモデルの完成度
- 週次振り返りの質

---

## 📌 Module 4: 本格アプリ開発 + サービス企画 (Month 7-8)

### 学習目標
- 本格的なアプリ開発を実践
- サービス企画を立案し、実行する能力を育てる
- 産業課題とAIの連携を学ぶ

### 週次トピック
1. アプリ開発の基礎（設計・開発・テスト）
2. サービス企画の立案（F4 CHINTAI型 5項目企画）
3. 産業課題の理解（F5 産業課題×AI 4項目）
4. アプリの設計（UI/UX・機能設計）
5. アプリの開発（コード作成・テスト）
6. サービス企画の実践（企画書作成・プレゼン）
7. 産業課題の実践（AIを活用した解決策）
8. アプリとサービス企画のまとめ（自己評価・振り返り）

### 課題と発表
- アプリ開発課題：簡単なアプリ制作
- サービス企画：自分のアイデアを形にする

### 必要ツール
- Antigravity
- Cursor
- Claude
- Figma

### 評価方法
- アプリ開発の完成度
- サービス企画の質
- 産業課題の解決策

## 📌 Module 5: データ分析 + マネタイズ + 卒業課題発表 (Month 9-12)

### 学習目標
- データ分析のスキルを習得
- マネタイズの方法を学び、ビジネスモデルを構築
- 卒業課題を発表し、成果を評価される

### 週次トピック
1. データ分析の基礎（データ収集・整理・可視化）
2. マネタイズの方法（収益モデル・価値提案）
3. ビジネスモデルの構築（価値・収益・コスト）
4. データ分析の実践（実データの分析）
5. マネタイズの実践（収益モデルの設計）
6. ビジネスモデルの検証（フィードバック・修正）
7. 卒業課題の準備（成果物の整理・プレゼン資料作成）
8. 卒業課題発表（成果発表・自己評価・振り返り）

### 課題と発表
- データ分析課題：実データの分析
- マネタイズ課題：収益モデルの設計
- 卒業課題発表：成果物の発表

### 必要ツール
- Antigravity
- Cursor
- Claude
- Notion
- Google Sheets

### 評価方法
- データ分析の完成度
- マネタイズの質
- 卒業課題発表の成果
