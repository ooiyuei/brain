## アプリ名・1行説明  
**「やる気スイッチ」**  
高校生が毎日勉強を続けるための、ゲーム要素を組み込んだ学習管理アプリ

## ターゲット  
**明日使う1人: 高校2年生の女子（16歳）**  
偏差値60台の公立高校に通う。勉強を続けるのが苦手で、スマホにゲームをやってしまう。親に「勉強をしなさい」と言われても、やる気にならない。  
**課金はしない**。広告収益で月3万円を達成する。

## 功能リスト v0.1 (MVP・3機能のみ)  
1. **勉強タスクの設定**  
   - 今日の勉強時間・科目・目標を設定  
   - タスクを「ゲームのステージ」として表示  
   - 完成すると「ステージクリア」のアニメーションが表示  

2. **勉強時間の記録と統計**  
   - 毎日勉強時間を記録  
   - 週ごとの勉強時間のグラフ表示  
   - 「今週の達成率」を表示  

3. **ゲーム要素の報酬**  
   - 勉強時間に応じて「ポイント」を獲得  
   - ポイントで「ゲーム内アイテム」を購入（例: キャラクターの変更）  
   - ポイントは広告で獲得できる（例: 広告をクリックすると10ポイント）

## 画面構成 (3画面)  
1. **ホーム画面**  
   - 今日のタスク一覧  
   - 今週の勉強時間グラフ  
   - 「今週の達成率」表示  

2. **タスク設定画面**  
   - 科目・時間・目標を入力  
   - 「ゲームのステージ」に表示  

3. **報酬画面**  
   - 獲得したポイントを表示  
   - ポイントで購入できるアイテム一覧  
   - 広告クリックでポイントを獲得するボタン  

## データモデル  
| テーブル名 | フィールド | 説明 |
|------------|----------|------|
| users | id, name, created_at | ユーザー情報 |
| tasks | id, user_id, subject, time, goal, created_at | 勉強タスク |
| study_logs | id, user_id, task_id, start_time, end_time, duration | 勉強時間記録 |
| rewards | id, user_id, points, created_at | 獲得ポイント |
| ads | id, user_id, points_earned, created_at | 広告クリックによるポイント獲得 |

## 技術スタック  
- **Frontend**: React Native + Expo  
- **Backend**: Supabase（リアルタイムデータベース）  
- **広告**: AdMob（バナー広告）  
- **課金**: Stripe（広告収益をStripeで管理）  
- **UI**: Figmaでデザインし、Expoで実装  

## 月3万円達成のロジック  
- **DAU（日間アクティブユーザー）**: 100人  
- **広告単価**: 1人あたり1日10円（AdMobの平均）  
- **月収入**: 100人 × 10円 × 30日 = **30,000円**  
- **課金率**: 0%（広告収益のみ）
