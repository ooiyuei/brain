# AIpaX school 1年カリキュラム

## 🎯 5モジュール構造 (2ヶ月 × 5 = 1年)

### Module 1: AIマインドセット + バイブ体験 (Month 1-2)
- **学習目標**: AIを手足のように使えるマインドセットを構築し、AIツールの基本操作を習得
- **週次トピック**:
  1. AIとは何か？AIの現状と未来
  2. AIツールの選び方と使い方（ChatGPT / Claude / Canva AI）
  3. プロンプト設計の基礎（例: 「AIで動画作成」のプロンプト作成）
  4. AIで作る「バイブ体験」（AIで動画・音楽・デザインを生成）
  5. AIと人間の協働の価値
  6. AIツールの限界と倫理
  7. AIで学ぶ「思考のフレームワーク」（F1 ストーリーテリング 3軸）
  8. AIで学ぶ「実践のスタイル」（大井の実体験をもとに）
- **課題と発表**: AIで作る「バイブ体験」作品（動画・音楽・デザインなど）を発表
- **必要ツール**: Antigravity / Cursor / Claude / Canva AI
- **評価方法**: 作品の完成度 + プロンプト設計の工夫 + フレームワークの理解度

### Module 2: 日常業務自動化 (Month 3-4)
- **学習目標**: 日常業務をAIで自動化するスキルを習得し、チームでアイデアを共有
- **週次トピック**:
  1. 日常業務の自動化とは？（例: メールの自動返信、スケジュール管理）
  2. AIツールで自動化する方法（Replit / Cursor / Notion AI）
  3. チームでアイデアを共有する方法（F2 合意形成 4ステップ）
  4. 自動化の実践（例: メールの自動返信スクリプト作成）
  5. 自動化の限界と倫理
  6. 自動化で学ぶ「実践のスタイル」（大井の実体験をもとに）
  7. 自動化で学ぶ「思考のフレームワーク」（F2 合意形成 4ステップ）
  8. 自動化で学ぶ「実践のスタイル」（大井の実体験をもとに）
- **課題と発表**: 自動化の実践成果（スクリプト・ツール・成果物）を発表
- **必要ツール**: Replit / Cursor / Notion AI / Antigravity
- **評価方法**: 実践成果の完成度 + チームでのアイデア共有 + フレームワークの理解度

### Module 3: エンジニアリング思考 + 個人探究テーマ設定 (Month 5-6)
- **学習目標**: エンジニアリング思考を身につけ、個人の探究テーマを設定
- **週次トピック**:
  1. エンジニアリングとは？（例: プログラミング・設計・実装）
  2. エンジニアリングの実践（例: サービスの設計・実装）
  3. 個人探究テーマの設定（F3 11項目ロジックモデル）
  4. 個人探究テーマの実践（例: サービスの設計・実装）
  5. 個人探究テーマの限界と倫理
  6. エンジニアリングで学ぶ「実践のスタイル」（大井の実体験をもとに）
  7. エンジニアリングで学ぶ「思考のフレームワーク」（F3 11項目ロジックモデル）
  8. エンジニアリングで学ぶ「実践のスタイル」（大井の実体験をもとに）
- **課題と発表**: 個人探究テーマの実践成果（サービス・設計・実装）を発表
- **必要ツール**: Cursor / Antigravity / Figma / Notion AI
- **評価方法**: 実践成果の完成度 + フレームワークの理解度 + チームでのアイデア共有

### Module 4: 本格アプリ開発 + サービス企画 (Month 7-8)
- **学習目標**: 本格アプリ開発のスキルを習得し、サービス企画を実践
- **週次トピック**:
  1. アプリ開発とは？（例: メイン画面・機能設計・実装）
  2. アプリ開発の実践（例: サービスの設計・実装）
  3. サービス企画の設計（F4 CHINTAI型 5項目企画）
  4. サービス企画の実践（例: サービスの設計・実装）
  5. サービス企画の限界と倫理
  6. アプリ開発で学ぶ「実践のスタイル」（大井の実体験をもとに）
  7. アプリ開発で学ぶ「思考のフレームワーク」（F4 CHINTAI型 5項目企画）
  8. アプリ開発で学ぶ「実践のスタイル」（大井の実体験をもとに）
- **課題と発表**: サービス企画の実践成果（サービス・設計・実装）を発表
- **必要ツール**: Cursor / Antigravity / Figma / Notion AI
- **評価方法**: 実践成果の完成度 + フレームワークの理解度 + チームでのアイデア共有

### Module 5: データ分析 + マネタイズ + 卒業課題発表 (Month 9-12)
- **学習目標**: データ分析とマネタイズのスキルを習得し、卒業課題を発表
- **週次トピック**:
  1. データ分析とは？（例: データの収集・分析・可視化）
  2. データ分析の実践（例: データの収集・分析・可視化）
  3. マネタイズの設計（F5 産業課題×AI 4項目）
  4. マネタイズの実践（例: データの収集・分析・可視化）
  5. マネタイズの限界と倫理
  6. データ分析で学ぶ「実践のスタイル」（大井の実体験をもとに）
  7. データ分析で学ぶ「思考のフレームワーク」（F5 産業課題×AI 4項目）
  8. データ分析で学ぶ「実践のスタイル」（大井の実体験をもとに）
- **課題と発表**: 卒業課題の実践成果（サービス・設計・実装）を発表
- **必要ツール**: Cursor / Antigravity / Figma / Notion AI
- **評価方法**: 実践成果の完成度 + フレームワークの理解度 + チームでのアイデア共有
