# AIpaX β顧客向け提案書

## 🎯 顧客課題の言語化

中小企業のAI導入率は5%にとどまっている。その理由は、AI導入のハードルが高く、導入後の運用コストが見込まれるため。AI導入の初期費用が高額で、月額の運用コストも不明瞭なため、中小企業は導入をためらっている。

## 🔍 AIpaX が解決すること

AIpaXは、**会社別AI保存庫コア**と**業務AIツール**を組み合わせることで、中小企業のAI導入を実現する。

### 会社別AI保存庫コア
- 会社の資料・契約・料金・FAQ・顧客情報・顧客別条件・例外条件・やり取り履歴を整理
- AIが検索・参照・再利用できる形に整える
- 顧客ごとの条件を保存し、AIがその条件を反映して回答する

### 業務AIツール
- リアルタイム電話 / 営業回答支援
- 社内AIチャット / 新人教育AI
- 議事録整理AI
- フォロー文生成AI
- CS対応AI
- 管理レポートAI
- CRM補助ツール

## 📌 導入後のBefore/After（業務シーン3つ）

### Before
- 営業電話で回答がズレる
- 新人がルールを何度も聞く
- 商談後の整理が遅い

### After
- リアルタイム電話で正確な回答を提供
- 社内AIチャットで新人がルールを確認
- 議事録整理AIで商談後の整理を自動化

## 💰 価格構造の根拠

### 初回価格（50〜80万円）
- 会社別AI保存庫コアの構築
- 初期の業務AIツール1つ（リアルタイム電話/営業回答支援）

### 月額価格（5万円前後）
- 保存庫コアの運用
- 業務AIツールの運用
- 月額の運用コストは、資料更新・入庫前処理・旧情報の非アクティブ化・検索条件の調整・回答カードの改善・利用ログ確認・回答できなかった質問の整理・不具合修正を含む

### 通常価格（100〜150万円）
- 会社別AI保存庫コアの構築
- 通常の業務AIツール（複数ツール）

### 本格導入（200〜500万円）
- 会社別AI保存庫コアの構築
- 複数の業務AIツール
- 外部連携・紙資料・OCR・大量旧資料整理・法務・契約条件整理代行

## 📅 導入スケジュール（3ヶ月）

## 📌 月額運用の範囲

### 含むもの
- 対象領域内の資料更新 / 入庫前処理 / 顧客確認後の保存庫反映 / 旧情報の非アクティブ化 / 検索条件の軽微調整 / 回答カードの軽微改善 / 利用ログ確認 / 回答できなかった質問の整理 / 軽微な不具合修正

### 含まないもの（追加料金）
- 新ツール作成 / 対象領域外の追加 / 外部連携 / 紙資料・OCR / 大量旧資料整理 / FAQ新規作成代行 / 契約条件整理代行 / 社内ルール作成 / 緊急対応 / 別部署展開 / 別商品ライン追加 / 電話回線連携 / CRM連携 / Zoom・Meet連携 / 手書き資料 / 顧客側アカウント構築 / 自社→顧客側移管 / 現地作業 / 法務・契約条件整理代行

## 📌 5原則

1. 保存庫コアとツール出力を混ぜない
2. 顧客資料の正しさは顧客側が決める
3. AIが勝手に正式情報を作らない
4. 不足・矛盾・最新版不明を検出して提示する
5. 対象領域内の顧客別条件・例外条件は削らない

## 📌 技術構成

- Supabase：構造化DB（companies/sources/source_chunks/knowledge_items/entities/relationships/events/reviews/versions/tool_logs/user_accounts/permissions）
- OpenAI / File Search / Embedding：資料検索・意味検索
- アプリ：URL+ログイン（初期）。デスクトップアプリ不要

## 📌 OpenAI/Supabase 管理方式（顧客選択）

| 方式 | 内容 | 採用しやすさ |
|------|------|------------|
| A. 自社側管理（標準） | 導入早い・修正しやすい・実費管理あり | デフォルト |
| B. 顧客側管理 | 機密に強い・情シスある会社向け | 追加費用オプション |
| C. 自社→顧客側移管 | 最初早く導入、後で移管 | 別料金 |

## 📌 通話ログ保存（顧客選択）

| 方式 | 内容 |
|------|------|
| A. 保存しない | セキュリティ軽い |
| B. 文字起こし＋検知質問＋回答候補＋検索ログのみ保存（標準） | 改善に使える |
| C. 音声＋文字起こし両方保存 | 追加設計必要 |
