# Testall vs Studyplus Planning — 機能・価格・ユーザー層・勝ち筋の比較

> 結論を先に: 両者は「同じ高校生」を狙うが**起点が逆**。Studyplus は「計画を立てる」ところから始まり、Testall は「テスト結果」から始まる。計画は気合いで続かないが、テスト結果は否応なく返ってくる。**入口を相手の弱点に置く**のが Testall の勝ち筋。

---

## 1. 市場位置

### Testall
- **何**: 受験生向け AI 学習マネージャー（旧称・受験OS から進化）。
- **誰に**: 高校1〜3年の独学受験生、特に塾なし層（推計70万人）。スマホメイン。
- **コアバリュー**: テスト返却 → 弱点特定 → **今日の45分タスクに自動変換**。
- **スタック**: Next.js + Supabase + Claude AI（`claude-sonnet-4-6`、深い分析は Opus）。
- **JUDGE**: 75点 / Tier A。
- **独自資産**: 2,888冊の参考書DB、Claude Vision でテスト画像をそのまま入力可。
- **Founder-Market Fit**: 大井自身が受験世代 = 当事者設計。

### Studyplus Planning
- **何**: Studyplus が展開する学習計画・管理機能。母体の Studyplus は学習記録SNSとして高校生に広く浸透。
- **誰に**: 高校生・大学生（受験層〜資格層）。
- **コアバリュー**: 学習時間の記録と、計画の作成・管理。**計画は基本的に手動で立てる**。
- **スタック**: 《要記入: 公開情報なし。推測で埋めない》。
- **AI 診断**: 《要記入: 自動診断機能の有無・深度は要確認。現時点で未確認》。

## 2. 機能比較

| 領域 | Testall | Studyplus Planning |
|------|---------|--------------------|
| 入口 | テスト結果入力（科目・点数・分野別ミス） | 学習計画・目標を手動設定 |
| 診断 | Claude AI が弱点分析・優先度付け | 《要記入: AI診断の有無 要確認》 |
| 実行支援 | 集中モード（今日の45分タスク提示＋タイマー） | タスク管理（追加・完了チェック） |
| 計画 | 週単位スケジュール（診断から自動） | 計画作成・管理（主に手動） |
| 記録 | 成長記録・連続学習日数 | 学習時間記録（母体の主力機能） |
| ソーシャル | なし（意図的に単独完結） | 学習記録の共有・仲間機能（母体の強み） |

→ Testall の機能はすべて「テスト結果」という1点から枝分かれする。Studyplus は「記録＋計画＋ソーシャル」の総合学習プラットフォーム。**重なるのは計画・タスク管理の層だけ**で、診断起点かどうかが分水嶺。

## 3. UX の違い（一文で）

- **Testall**: 入力したら、あとはやることが出てくる。判断をアプリに預けられる。
- **Studyplus Planning**: 何をやるかは自分で決める。記録と仲間がモチベーションを支える。

「自分で決める負担」を取り除けるかどうか。ここがやる気依存層と仕組み依存層の境目。

## 6. 勝ち筋（3本）

### 勝ち筋1: 起点が「テスト結果」＝続けるための気合いが要らない
Studyplus の計画機能は「まず計画を立てる」ことを前提にする。だが計画は気合いで続かない。Testall は**テスト結果という、誰のもとにも否応なく届くトリガー**を入口にする。入力 → AI診断 → 45分タスクまで一気通貫で、ユーザーの意思決定をゼロにできる。やる気ではなく仕組みで成績を上げたい層に直接刺さる。

### 勝ち筋2: 受験戦略OS という再定義
Studyplus は「学習記録・管理アプリ」。Testall は「受験戦略OS」を名乗る。記録するだけでなく、**何を・いつ・どの順でやるかを戦略として処方する**立ち位置。同じ計画機能でも、ツールか OS かでユーザーの期待値が変わる。カテゴリを自分で命名して上に立つ。

### 勝ち筋3: Claude AI 診断＋画像入力の精度とスピード
Claude Vision でテスト画像をそのまま読み込み、2,888冊の参考書DBに紐づけて弱点 → 優先度 → 45分タスクを生成する。手動入力の摩擦が小さく、診断の解像度が高い。Studyplus の自動診断の有無は《要記入: 要確認》だが、母体の主力は記録・ソーシャルであり、診断起点の設計思想ではない。**ここで真っ向勝負しなくていい**——別カテゴリとして並走できる。

| 競合 | 弱点 | Testall の優位 |

| Studyplus Planning | **最大脅威**。計画機能を強化中 | テスト結果起点の自動診断 |

→ スタサプの塾向け撤退は需要の空白。Studyplus との正面戦だけでなく、**この空白を取りにいく**動きも同時に張れる。

## 8. 結論

Studyplus が「**自分で計画を立て、記録し、仲間と続ける**」プラットフォームなら、Testall は「**テスト結果を渡せば、次の45分が出てくる**」マシン。狙う高校生は重なるが、勝負する軸はずらせる。

- 真っ向勝負しない。Studyplus が弱い「**計画を立てられない・続かない塾なし層**」を入口で取る。
- 価格では差がつかないので、**払う理由（自動診断の精度）**で差をつける。
- カテゴリは「学習管理アプリ」ではなく「受験戦略OS」で名乗る。
