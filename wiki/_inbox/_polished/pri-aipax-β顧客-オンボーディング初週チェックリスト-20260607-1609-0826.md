# AIpaX β顧客 オンボーディング初週チェックリスト

> 対象: 初回事例価格（初期50〜80万 / 月額5万）で契約した β顧客。契約締結から7日間の実務。
> 範囲: [[sources/aipax-spec-v1-2026-05-19]] の仕様内のみ。架空ツール名・未確定機能は書かない。
> ゴール: **保存庫コア + 最初の業務ツール1つ** が顧客の業務フローで動き、初期成果が1件確認できる状態にする。

## 前提（このチェックリストが守る5原則）

仕様書の5原則をオンボ全工程の判断基準にする。

1. 保存庫コアとツール出力を混ぜない
2. 顧客資料の正しさは顧客側が決める
3. AIが勝手に正式情報を作らない
4. 不足・矛盾・最新版不明を検出して提示する
5. 対象領域内の顧客別条件・例外条件は削らない

---

## Day 0: キックオフ（契約締結当日）

| タスク | 担当 | 完了条件 |
|--------|------|----------|
| 契約書・確認シートの最終締結 | AIpaX | 電子署名済み。契約前確認シートの全項目に顧客回答が入っている |
| 対象領域の確定 | AIpaX × 顧客 | 「保存庫が扱う範囲」を1枚に定義（管理対象領域 / 更新頻度 / 含む含まないの境界）。対象外＝追加料金であることを顧客と合意 |
| 最初の業務ツール1つを確定 | 顧客 | 顧客の課題から初手ツールを1つ選定。最有力は「リアルタイム電話 / 営業回答支援」。複数同時導入はしない |
| 顧客選択メニューの3点を回収 | 顧客 | ① OpenAI/Supabase管理方式（A:自社側標準 / B:顧客側 / C:後移管）② 通話ログ保存（A:保存なし / B:文字起こし＋検知＋回答候補＋ログ＝標準 / C:音声も保存）③ アカウント管理方式 |
| 必要資料リストの提出依頼 | AIpaX → 顧客 | 対象領域内の資料・契約・料金表・FAQ・顧客別条件・例外条件・やり取り履歴の提出ルールを渡し、提出窓口担当を1名決める |

> 注意: 対象外領域の追加、外部連携（CRM / Zoom・Meet / 電話回線）、紙資料・OCR、大量旧資料整理は **月額に含まない**。Day 0で線を引いておかないと初週で溶ける。

---

## Day 1-3: 保存庫コア構築 + ツール初期設定

| タスク | 担当 | 完了条件 |
|--------|------|----------|
| 入庫前処理 | AIpaX | 提出資料を構造化DB（companies / sources / source_chunks / knowledge_items / entities / events）に整形。旧情報は非アクティブ化。原則4に従い、不足・矛盾・最新版不明を一覧化して顧客に提示 |
| 保存庫コアの初期構築 | AIpaX | File Search / Embedding で検索・参照可能化。対象領域内の顧客別条件・例外条件は削らずに保持（原則5） |
| 最初の業務ツールの設定 | AIpaX | 確定した初手ツールを構築。営業回答支援なら回答カード・検索条件を初期投入。出力は保存庫コアと分離（原則1） |
| 操作レクチャー | AIpaX → 顧客 | 顧客担当者がログイン（URL+ログイン、デスクトップアプリ不要）し、基本操作を一通り実行できる |

---

## Day 4-7: 顧客確認 → 保存庫反映 → 初期成果

| タスク | 担当 | 完了条件 |
|--------|------|----------|
| 顧客確認シートの回収 | AIpaX → 顧客 | 構築した保存庫の中身を顧客が確認。正誤・最新版の判断は顧客側が下す（原則2）。未確認のまま反映しない |
| 保存庫コアの本反映 | AIpaX | 顧客確認後にのみ反映。これで保存庫が日常利用可能になる |
| ツールの軽微調整 | AIpaX | 利用ログを見て、回答カード・検索条件を顧客の実業務に合わせて調整（月額運用範囲内の軽微改善） |
| 初期成果の確認と報告 | AIpaX → 顧客 | 初手ツールの成果を1件、具体で確認（例: 営業電話で回答がズレなくなった通話を1本）。回答できなかった質問を整理し、簡潔な初週レビューを顧客へ提出 |

---

## 完了の定義（初週が「成功」と言える条件）

- 対象領域が1枚で定義され、顧客と合意済み
- 保存庫コアが顧客確認を経て反映され、検索・参照できる
- 最初の業務ツール1つが業務フローで動いている
- 初期成果が**具体で1件**確認できている（抽象的な「効果がありそう」は不可）
- 回答できなかった質問・不足資料が一覧化され、次の月額運用に渡せる

---

## 初週でやらないこと（月額に含まない＝別見積）

新ツール追加 / 対象領域外の取り込み / 外部連携（電話回線・CRM・Zoom/Meet）/ 紙資料・OCR / 大量旧資料整理 / FAQ新規作成代行 / 社内ルール作成 / 別部署展開 / 緊急対応。
顧客から要望が出たら、断るのではなく「追加メニュー」として見積に回す。

---

## 引き継ぎ（初週 → 月額運用）

初週レビュー提出時に、月額運用で回す項目を顧客と握る:
対象領域内の資料更新 / 入庫前処理 / 顧客確認後の保存庫反映 / 旧情報の非アクティブ化 / 検索条件の軽微調整 / 回答カードの軽微改善 / 利用ログ確認 / 回答できなかった質問の整理 / 軽微な不具合修正。

> 《要記入: β顧客の具体社名・業種・初手ツール選定の実際（4社それぞれで埋める）》
> 《要記入: 初期成果の測定指標を顧客ごとに何にするか（通話精度 / 一次返信時間 / 新人教育の問い合わせ削減 等）》
