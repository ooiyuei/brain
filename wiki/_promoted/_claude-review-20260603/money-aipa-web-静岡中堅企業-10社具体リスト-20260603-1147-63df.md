# AIpa Web 静岡モニター候補 — 選定基準・10社枠・アプローチキット

> 元ドラフト（qwen3:8b）は「静岡◯◯株式会社」という**架空の社名**を10件並べただけで、課題仮説も接触経路もほぼ全行コピペだった。実在企業名が無いリストは営業に使えない。
> そこで Claude が **(A) 実企業を埋めるための選定基準と確定済みの検索手順、(B) 10社の確定枠（実名は要記入）、(C) そのまま送れるアプローチメール・想定問答** に作り直した。
> **実在の社名・担当者・URL は「捏造しない」方針**。該当箇所は《要記入》で空けてある。埋めるのは大井のローカル知識（清水・静岡市内の中堅企業）か、下記検索手順による1時間のリサーチ。

---

## なぜ「架空10社」では1社も獲れないか

採用サイト制作の営業は、**相手の今の採用サイトの弱点を具体的に指摘できるか**で初回返信率が決まる。架空の「静岡建設株式会社」には今のサイトが存在しないので、刺さる一言が作れない。
だから本キットは「社名を埋める」ではなく「**埋めれば刺さる一言が自動で作れる**」状態を先に用意する。

---

## A. 実企業を埋める選定基準（5条件）

[[entities/aipa-web]] の初期ターゲット定義に厳密に従う。10社は**全条件AND**で絞る。

1. **所在地**: 静岡県内（清水・静岡市内・浜松・沼津を優先。大井のホームグラウンド＝清水を最初に当たる）
2. **規模**: 従業員 30〜300名（中堅。大企業は内製、零細は予算が出ない）
3. **業種**: 建設 / 製造 / 物流 / 介護 / 飲食 / 美容 / 小売 / 地方サービス（採用の痛みが売上・現場崩壊に直結する業種）
4. **採用シグナル**: Indeed / engage / AirWORK / 求人ボックスに**今まさに求人を出している**（採用にお金を払う意思が確認できる）
5. **サイトの弱点**: 採用サイトが無い、または「会社概要に求人PDFを貼っただけ」「スマホで崩れる」「応募フォームが無くメールだけ」のいずれか

→ 5を満たす会社ほど刺さる。**5が一番大事**。「採用に金は払ってるのに受け皿が弱い」会社が最優先。

## C. アプローチ実務

### C-0. 社名の埋め方（確定手順・所要約1時間）

1. Indeed で「静岡県 + 業種」（例: 静岡 建設 採用）で検索 → 求人を出している会社を10〜20社リストアップ
2. 各社の会社名で再検索し、**コーポレートサイトの採用ページの有無・出来**を実際に開いて確認
3. 上記Aの条件5（弱点）に当てはまる会社だけ残す
4. 残った会社を上の表に転記。担当者名が分かれば記入、不明なら「採用ご担当者様」で進める
5. 清水・静岡市内を優先。大井の知り合い・地元経営者・商工会議所ルートで紹介が取れる会社は最優先（初回返信率が段違い）

### C-1. 接触経路（実際に使う3ルート）

元ドラフトは「求人代理店にAIpa Webをアピールして依頼を出す」など意味の通らない記述だった。実際に動く3ルートに整理する。

| ルート | 具体的な動き | 向く相手 |
|---|---|---|
| **直アプローチ** | C-0で見つけた会社の問い合わせフォーム/採用担当へC-2のメール | サイトの弱点が明確な会社 |
| **紹介（最優先）** | 大井の地元（清水）人脈・経営者仲間・商工会議所経由で1社挟む | 紹介が取れる会社（返信率が一番高い） |
| **求人広告代理店と組む** | 「媒体に出す前に受け皿（採用サイト）を整えましょう」と代理店に提案し、代理店の顧客に同梱提案 | 媒体は出すがサイトが弱い顧客を抱える代理店 |

補助金は**入口ではなく後押し材料**。商談中に「IT導入補助金2026（上限450万・ウェブ制作対象）が使えます」と添えるとクロージングが早い。最初の一言には使わない（補助金目当ての冷やかしを呼ぶため）。

### C-2. 初回コールドメール（直アプローチ・そのまま送れる完成形）

> 弱点の一文だけ各社で差し替える。それ以外は固定。

```
件名: 御社の採用ページについて（株式会社《自社名》・大井）

《担当者名 or 採用ご担当者様》

突然のご連絡失礼いたします。静岡で採用サイト制作を行っている
《自社名》の大井と申します。

御社が《媒体名・例: Indeed》で人材を募集されているのを拝見し、
ご連絡しました。求人広告は出していらっしゃる一方で、
《弱点の一文・例: 応募者が会社の魅力を確認できる採用ページがまだ無い状態》
のように見受けられました。

採用広告は「興味を持った人がクリックした先」で決まります。
その受け皿となる採用サイトを、フォーム入力をもとに最短7日で
公開し、公開後の更新まで一括で代行しています。

打ち合わせを何度も重ねる必要はありません。
御社にご記入いただくのはフォーム1枚で、構成・文章・デザインの
初稿はこちらで用意します（最終確認は御社で行っていただきます）。

今、事例づくりとして数社限定で、通常30〜50万円相当の採用サイトを
初期15万円（事例として実績掲載にご協力いただける条件）で
お受けしています。

一度、御社の採用ページの改善ポイントを5分ほどでご説明できればと思います。
今週か来週で15分、オンラインかお電話のお時間をいただけませんでしょうか。

──────────
《自社名》 大井湧瑛
《電話》 / 《メール》
《サイトURL or 実績ページ》
```

### C-3. 想定問答（商談・返信での反論対応）

| 相手の一言 | 返し方 |
|---|---|
| 「AIで作るんでしょ？安っぽくない？」 | 「AIは下書きの効率化に使いますが、給与・条件・会社の事実は必ず御社に確認いただき、最終的な文章とデザインは人が仕上げます。だから速くて、かつ事故が起きません」 |
| 「うちは今のサイトで足りてる」 | 「会社サイトと採用サイトは役割が違います。求職者は『ここで働く自分』を想像できるかで応募を決めます。今のページでスマホから応募まで進めるか、一度ご一緒に見てみませんか」 |
| 「15万は安すぎて逆に不安」 | 「事例として実績掲載にご協力いただける数社限定の価格です。通常は30〜50万円です。先に1社で品質を見ていただき、納得いただけたら継続（月額更新）をご相談する形です」 |
| 「月額って何に払うの？」 | 「公開後の更新です。募集要項の変更、写真差し替え、お知らせ追加などを月1回〜代行します。採用サイトは作って終わりではなく、募集が変わるたび直す必要があるので、その担当を外注する感覚です」 |
| 「補助金は使える？」 | 「IT導入補助金2026がウェブ制作も対象で、上限450万円です。申請サポートも込みでご案内できます」 |
| 「他社（STUDIO等）と何が違う？」 | 「ビルダー系は結局ご自身で構成と文章を考える必要があります。うちはそこをこちらで用意し、採用に特化したテンプレと、公開後の更新まで含めて代行します」 |
| 「効果あるの？」 | 「応募『増』を保証する話ではなく、まず『応募者が会社を理解して応募できる受け皿』を作る話です。広告費を払う前に受け皿を整える方が、同じ広告費でも応募の質が上がります」 |

## D. このリストを使った次の3アクション

1. **C-0を1時間回して表Bの社名を10件埋める**（清水・静岡市内優先）。埋まった瞬間に各社の「刺さる一言」が決まる。
2. **紹介が取れる1〜2社を最優先で当てる**。直アプローチより返信率が高いので、最初の事例はここから作る。
3. **モニター3社獲得**（初期15万・事例掲載条件）。[[concepts/aipa-web-form-design]] の16セクションフォームを渡し、1社目をClaudeで初稿生成 → 実物を完成させる。

> ゴールは「10社の名前を持つこと」ではなく「**3社のモニター事例を持つこと**」。事例が1つできれば、次の営業は「これと同じものを御社に」で済む。

## 元ドラフトからの主な修正

- 架空社名10件 → 実名は《要記入》にし、**埋めるための選定基準＋検索手順**を新設（捏造を排除）
- 全行コピペの課題仮説 → 業種別の痛みと「サイトの弱点」を起点に再設計
- 意味の通らない接触経路（「代理店にアピールして依頼を出す」） → 実際に動く3ルートに整理
- frontmatter の虚偽（「実在企業名・確証あり」） → 事実に合わせて撤回
- 使えない「結論」「補足」の水増し → そのまま送れるメール・想定問答に置換
- 価格・納期・条件は [[entities/aipa-web]] の正本（初期15万モニター・最短7日）に統一
