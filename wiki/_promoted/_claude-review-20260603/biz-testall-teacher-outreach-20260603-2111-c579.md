# Testall 教育者連携プログラム — 初期5名募集パッケージ

> 目的: Testall を「個人課金プロダクト」だけでなく「教育者が生徒に勧めるツール」にする。
> このパッケージは **(1) 連携の狙いと提供価値 → (2) 募集要項 → (3) アウトリーチメール → (4) 連携契約ドラフト → (5) インセンティブ設計 → (6) 想定問答** の順で、そのまま送れる形にしてある。

---

## 0. なぜ教育者連携をやるのか（大井の意図）

俺がTestallで最初に取るべきは、塾なし独学層（高1〜3、約70万人）。でもこの層は広告で個別に取りにいくとCACが重い。**いちばん効くのは「信頼している先生に勧められる」導線**だ。塾講師・予備校講師・高校教師は、すでに「この子は何が弱いか」を見ている。そこに「で、今日の45分は何をやればいいか」を自動で出すTestallを挟めば、先生の手間を増やさずに生徒の自走を作れる。

だから最初は**売上目的じゃなく検証目的**で5人だけ取る。狙いは3つ:

1. **生徒へのリーチ**: 先生1人の裏に生徒10〜30人。5人で50〜150人の見込み接点。
2. **プロダクト検証**: 現場のプロに「AI診断は的を射ているか」をぶつけて、診断ロジックを直す。
3. **推薦の言葉**: 「現役の先生が使っている」という事実とコメントが、その後のLP・SNS・営業の弾になる。

---

## 1. 連携で先生に渡せる価値（先生視点で書く）

先生が連携する理由は「Testallが便利」じゃなく「**自分の指導が楽になる/成果が上がる**」こと。3点に絞る。

| 先生のペイン | Testallが渡すもの |
|---|---|
| 一人ひとりの弱点把握に時間がかかる | テスト結果入力 → AIが分野別の弱点優先度を自動で可視化。先生は診断結果を見るだけ |
| 「次これやって」と毎回指示するのが大変 | 生徒側に「今日の45分タスク」が自動で出るので、宿題の細かい指定を肩代わり |
| 授業外で生徒が勉強を続けない | 連続学習日数・週計画で、授業と授業の間の自走を仕組み化 |

連携先生には次を無償提供する（初期5名の特典）:

- 担当生徒分のTestall **プレミアム機能を無料開放**（通常1,500円/月相当・《要記入: 開放する生徒上限人数》）
- 生徒の診断結果サマリーを先生がまとめて見られる **指導者ビュー**（《要記入: このビューは現状未実装。MVPに含めるか、Phase2扱いか大井判断》）

> 注: 「先生が全生徒の進捗をまとめて見る画面」は entities/testall.md の現機能表に存在しない。連携の核になる機能なので、**約束する前に実装可否を確定させること**。無いまま「リアルタイムで把握できます」と言うと初回で信頼を失う。

## 2. 初期5名 募集要項

| 項目 | 内容 |
|---|---|
| 募集人数 | 5名（先着・少数で深く回す） |
| 対象 | 塾講師 / 予備校講師 / 高校教師。**現在、受験生を担当している人** |
| 必須条件 | ① 担当生徒が3人以上いる ② スマホ/PCでツールを生徒に勧められる ③ 月1回20分のフィードバック面談に応じられる |
| 不問 | Testall利用経験（新規でOK。むしろ初見の反応が欲しい） |
| 募集期間 | 2026-06-09 〜 2026-06-30（※元ドラフトは6/1開始だが本日6/3時点で過去日のため修正） |
| 連携期間 | 初回は **3ヶ月**（2026-07-01 〜 2026-09-30）。受験直前期の効果を見たいので延長は9月末に相談 |
| 費用 | 先生・生徒とも **無料**（検証フェーズのため） |
| 先生の稼働 | 担当生徒へTestallのURL共有 + 月1回20分の面談のみ |

**選定フロー:**
1. アウトリーチメール送付（下記§3）
2. 返信あり → 15分のオンライン顔合わせ（大井が直接話す）
3. 連携同意書（下記§4）に合意 → 生徒向けセットアップ案内を渡す

## 3. アウトリーチメール（そのまま送れる完成形）

宛先別に2パターン。署名は共通。

### 3-A. 知人・紹介経由の先生向け（ややカジュアル）

```

○○先生

「じゃあ今日の45分は何をやるべきか」を自動で出すアプリです。
生徒が「何を勉強すればいいか分からない」で止まる時間を無くすのが狙いです。

いま、現場で受験生を見ている先生5名に、3ヶ月だけ無料でモニターをお願いしています。
```

## 4. 連携同意書（ドラフト）

正式な業務委託契約ではなく、**検証フェーズの軽い合意書**にする。重くすると先生が引く。

```
Testall モニター連携 同意書（ドラフト）

本同意書は、Testall（開発: 大井湧瑛、以下「運営」）と
連携いただく教育者（以下「先生」）との間で、以下の通り取り決めるものです。

第1条（目的）
本連携は、受験生の学習支援に資するTestallを現場で検証し、
プロダクトおよび指導への活用方法を共に確立することを目的とする。

第2条（期間）
2026年7月1日 〜 2026年9月30日（3ヶ月）。
延長は期間満了前に双方協議のうえ決定する。

第3条（先生の役割）
(1) 担当する生徒にTestallの利用を案内する（強制はしない）。
(2) 月1回20分程度、運営とのフィードバック面談に応じる。
(3) 利用上で気づいた点・診断の精度について率直な意見を提供する。

第4条（運営の提供）
(1) 担当生徒分のプレミアム機能を本期間中、無料で提供する。
(2) 《要記入: 指導者ビュー提供の有無 — 実装が間に合う場合のみ記載》
(3) 先生・生徒から利用料は一切徴収しない。

第5条（謝礼）
インセンティブの内容は別紙「インセンティブ設計」に定める。

第6条（データの取り扱い）
生徒の学習データは指導目的の範囲でのみ利用し、
本人・保護者の同意なく第三者に提供しない。
先生のフィードバックは、匿名化のうえプロダクト改善・広報に利用する場合がある。
広報での実名・コメント掲載は、事前に先生の許可を得る。

第7条（解除）
いずれの当事者も、相手方に通知することでいつでも本連携を終了できる。
終了後も第6条のデータ取り扱いは継続する。

2026年__月__日
運営: 大井湧瑛　　先生: ___________
```

> 法務注: これは**有償の紹介報酬を伴う場合に景表法・特定商取引法・各塾の就業規則に触れうる**。とくに先生に金銭インセンティブを払う設計（§5）にするなら、勤務先が副業/紹介報酬を禁じていないか先生本人に確認させること。検証フェーズは金銭ナシ（§5-Aを推奨）にするのが安全。

## 5. インセンティブ設計

元ドラフトの「双方に報酬」「生徒に報酬を払う教育者には報酬」は意味が通らないので破棄。**検証フェーズと拡大フェーズで分ける。**

### 5-A. 検証フェーズ（初期5名・推奨）— 金銭ナシ

金を払うと「お金のために良いことを言う」バイアスが入り、検証が濁る。だから**非金銭の価値で組む**:

| 提供 | 内容 |
|---|---|
| 担当生徒のプレミアム無料 | 3ヶ月、1,500円/月相当 ×生徒数 |
| 共同開発者の立場 | フィードバックを実際の機能改善に反映。「この先生の意見でこう変えた」と伝える |
| 先行クレジット | 正式リリース時に「初期協力者」として（許可制で）名前/塾名を掲載 |
| 拡大フェーズの優先条件 | 有償紹介プログラム開始時、初期5名は最優遇レート |

### 5-B. 拡大フェーズ（検証後・有償紹介）

検証で「先生経由でちゃんと生徒が定着する」と分かったら、紹介報酬を入れる。**ロジックが追える形にする:**

- 先生の紹介で生徒が**有料プランに転換し3ヶ月継続**したら、その生徒の月額の **《要記入: 紹介報酬率（例: 20%）》を継続課金中ずっと**先生に還元。
- 例: 生徒5人がプレミアム（1,500円）で継続 → 1,500円 × 20% × 5人 = **月1,500円**を先生に。
- 不正防止: 紹介は専用リンク/コードで計測。1ヶ月未満の解約は報酬対象外。

> 数字（報酬率・上限・支払い方法）は大井が決める。Stripe側で紹介トラッキングをどう実装するかは別途設計が要る（現状Stripe課金自体が未実装＝最優先タスク。entities/testall.md参照）。

## 6. 想定問答（顔合わせ・面談で先生に聞かれること）

**Q. 生徒のデータは安全ですか？保護者にどう説明すれば？**
A. 学習データ（テスト結果・進捗）は指導目的にしか使いません。第三者提供はしません。保護者向けには「先生が勧める無料の学習サポートアプリ」と案内できる説明文を運営が用意します。《要記入: 保護者向け1枚説明の作成》

**Q. 私の手間はどれくらい？**
A. 生徒へのURL共有と、月1回20分の面談だけです。日々の声かけや採点をお願いすることはありません。むしろ「次に何をやらせるか」の指示出しをアプリが肩代わりします。

**Q. AIの診断って当てになるんですか？**
A. 正直、そこを先生に検証してほしいんです。テスト結果から弱点を出すロジックはありますが、現場のプロの感覚と合っているかは現場でしか分かりません。ズレていたら教えてください、直します。

**Q. 自分の塾の方針/教材とぶつからない？**
A. Testallは「何を教えるか」ではなく「生徒が一人の時間に何をやるか」を埋めるツールです。先生の指導を置き換えるのではなく、授業と授業の間を埋める設計です。

**Q. 無料なのは分かったけど、後で課金されて生徒が困らない？**
A. モニター期間の3ヶ月は完全無料です。期間後も、生徒に断りなく自動課金することはありません。続けたい生徒だけが有料に進む形です。

**Q. atama+とかスタディサプリと何が違うの？**
A. あれらは「教材を提供する」サービスです。Testallは教材を持たず、**生徒が今持っている教材・テスト結果を起点に「今日の45分」を出す**ツールです。だから既存の塾教材と競合しません。
