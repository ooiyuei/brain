# ビルドインパブリック X投稿 3本（2026-06-03）

> 編集メモ: build-in-public は「契約確定」を叫ぶ宣伝ではなく、**作っている過程・数字・つまずき**を見せて信頼を積む手法。元ドラフトは宣伝アナウンス＋絵文字過多＋未確認の数字（Gymeeβ）だったため、確認済みの事実（AIpaX 4社契約、Testall 開発実態、18歳起業家）に寄せて全面書き直し。
> 一人称・等身大・1ポスト1メッセージ。絵文字は1ポスト最大1個に削減。

## 投稿1 — 数字を1つだけ出す（フック）

```
18歳。AIコンサル「AIpaX」、4社目の契約が決まった。

中小企業に「会社専用のAI保存庫」を作る事業。
契約書・料金・FAQ・顧客別の例外条件まで、AIが全部引っ張ってこれる状態にする。

派手じゃない。でも現場の「あれ何だっけ」が消えると、社員の動きが変わる。

#個人開発 #AI #buildinpublic
```

**狙い**: 「4社」という確定数字＋年齢のギャップでフック。事業の中身を1文で説明。
**事実根拠**: AIpaX = 既に4社契約済（entities/aipax.md）。「会社別AI保存庫コア」が核。


**事実根拠**: AIpaX = オンボーディング3ステップ（入庫前処理→確認→反映）が「今やること」。Testall = MVP実装済・Stripe課金が最優先残タスク（entities/testall.md 活動ログ）。

## 投稿しない方が良い元ドラフトの主張（要確認）

- **「Gymee 10人βテスト開始 / 応募締め切り」**: Gymee の現状を裏付ける正本（entities/gymee.md）が無く、10人・締切日も根拠不明。投稿すると約束になる。出すなら《要記入: Gymeeβの実際の募集状況・締切日・応募先URL》を大井本人が確認してから。今回は3本から除外。
- **「社員AI教育の需要、データ出たら共有」**: AIpaX school（中高生向け）はあるが「社員AI教育の実データ」は未取得。数字が無い段階で「データ出たら」は空手形。出すなら実数値が出てから。
