# [[entities/aipax-school]] 1年カリキュラム5モジュール完成版

本カリキュラムは、AI時代のビジネスパーソンおよびエンジニアが、単なるツールの使用者から、AIを駆使して価値を創造する「AIプロフェッショナル」へと進化するための完全なロードマップです。全5モジュール、合計12ヶ月（各モジュール2ヶ月）で構成され、理論と実践を密接に結びつけることで、即戦力となるスキルセットの習得を目指します。

## モジュール1：AIマインドセットと基礎リテラシー（第1-2ヶ月）

**学習目標**
AIの基本原理を理解し、LLM（大規模言語モデル）との効果的な対話（プロンプトエンジニアリング）技術を習得する。また、AI倫理とセキュリティの基本を学び、AI時代における適切な思考習慣を確立する。

**週次トピック**
*   **第1週：AIの歴史と現在の技術基盤**
    *   機械学習、ディープラーニング、LLMの基本概念。
    *   Transformerアーキテクチャの直感的な理解。
    *   生成AIの限界とハルシネーション（幻覚）のメカニズム。
*   **第2週：プロンプトエンジニアリングの基礎**
    *   役割付与、コンテキスト提供、出力形式の指定。
    *   Few-shot promptingとChain-of-Thought（思考の連鎖）の活用。
    *   反復的なプロンプト改善プロセスの実践。
*   **第3週：高度なプロンプト技法と構造化データ処理**
    *   複雑なタスク分解とステップバイステップ指示。
    *   JSONやCSV形式での構造化出力の制御。
    *   複数モデルの比較評価とベストプラクティスの確立。
*   **第4週：AI倫理、セキュリティ、著作権**
    *   バイアス、プライバシー、データ漏洩リスクの理解。
    *   企業におけるAI利用ガイドラインの策定と遵守。
    *   著作権法と生成コンテンツの法的地位。
*   **第5-6週：統合プロジェクト「AIリテラシー診断と改善計画」**
    *   自身の業務におけるAI活用可能性の棚卸し。
    *   プロンプトライブラリの構築と共有。
    *   AI倫理チェックリストの作成と適用。
*   **第7-8週：レビューとフィードバック**
    *   受講生間のプロンプト共有と相互評価。
    *   講師による個別フィードバック。
    *   次モジュールへの準備知識の確認。

**課題**
*   日常業務の3つのタスクをAIで自動化するプロンプトチェーンの設計と実装。
*   「AI倫理に関する社内ガイドライン草案」の作成。

**必要ツール**
*   ChatGPT Plus / Claude Pro / Gemini Advanced などの主要LLMアカウント。
*   Notion / Obsidian などの知識管理ツール。
*   プロンプト管理用スプレッドシート。

**評価方法**
*   プロンプトの精度と出力の再現性（40%）。
*   AI倫理に関するレポートの論理性（30%）。
*   受講生間のピアレビュー参加度（30%）。

## モジュール2：日常業務自動化とワークフロー最適化（第3-4ヶ月）

**学習目標**
APIや自動化プラットフォームを活用して、メール、スプレッドシート、カレンダーなどの日常業務を自動化する。RPAとAIの組み合わせによる効率化を実現し、業務負荷の大幅な削減を図る。

**週次トピック**
*   **第1週：自動化の基礎とAPIの概念**
    *   REST APIの仕組みと認証（OAuth2, API Key）。
    *   Postmanを用いたAPIテストの基本。
    *   JSONデータ構造の理解と操作。
*   **第2週：No-Code/Low-Code自動化プラットフォーム**
    *   Make (Integromat) / Zapier の基本操作。
    *   トリガー、アクション、ループの概念。
    *   例：Gmail受信→スプレッドシート記録→Slack通知の自動化。
*   **第3週：スクリプトによる自動化の導入**
    *   Python基礎文法とpandasライブラリ。
    *   Google Apps Script / Microsoft Power Automate の基本。
    *   既存の自動化フローへのスクリプト組み込み。
*   **第4週：AIと自動化の融合**
    *   AIを用いたメール要約・返信草案の自動生成フロー。
    *   画像認識APIを用いた書類データ抽出の自動化。
    *   エラーハンドリングとリトライメカニズムの実装。
*   **第5-6週：統合プロジェクト「業務自動化ダッシュボード」**
    *   部門固有の業務フローの分析と自動化対象の選定。
    *   自動化フローの設計図作成と実装。
    *   効果測定（時間節約率）のシミュレーション。
*   **第7-8週：レビューと最適化**
    *   自動化フローのデバッグとパフォーマンス改善。
    *   スケーラビリティの検討。
    *   自動化によるリスク管理策の策定。

**課題**
*   少なくとも1つの主要業務プロセスを自動化し、その前後の所要時間を比較したレポート。
*   エラー発生時の対応フローを含む自動化ドキュメント。

**必要ツール**
*   Make / Zapier / n8n の無料または有料プラン。
*   Python環境（Anaconda / VS Code）。
*   Google Workspace / Microsoft 365 APIアクセス権。

**評価方法**
*   自動化フローの安定性とエラー耐性（40%）。
*   業務効率化の数値的効果の提示（30%）。
*   ドキュメントの明確さと再現性（30%）。

## モジュール3：エンジニアリング思考と外部連携（第5-6ヶ月）

**学習目標**
ソフトウェア開発の基本的な設計パターンを学び、AIモデルを外部システムと連携させるためのアーキテクチャを構築する。ベクトルデータベース、エージェント、および外部APIとの統合技術を習得する。

**週次トピック**
*   **第1週：ソフトウェア設計パターンとAI統合**
    *   モジュール設計と関心の分離。
    *   AIモデルを呼び出すためのラッパー関数の設計。
    *   状態管理とセッション管理の重要性。
*   **第2週：ベクトルデータベースとRAG（検索拡張生成）**
    *   埋め込み（Embedding）の概念とベクトル空間。
    *   Pinecone / Weaviate / ChromaDB のセットアップ。
    *   RAGパイプラインの構築：文書分割→埋め込み→検索→生成。
*   **第3週：AIエージェントの基礎**
    *   ツール呼び出し（Function Calling）の仕組み。
    *   ReAct（Reasoning and Acting）パターン。
    *   LangChain / LlamaIndex によるエージェント構築。
*   **第4週：外部APIとの高度な連携**
    *   天気、地図、決済などの外部サービスAPIとの統合。
    *   非同期処理とWebhookの活用。
    *   セキュアなシークレット管理（環境変数、Vault）。
*   **第5-6週：統合プロジェクト「自律型情報収集エージェント」**
    *   特定トピックに関するニュースやデータを自動収集・要約するエージェント。
    *   RAGを用いた社内ドキュメントへの質問応答インターフェース。
    *   エージェントの出力品質評価とチューニング。
*   **第7-8週：レビューとアーキテクチャレビュー**
    *   システムの拡張性と保守性の評価。
    *   コスト最適化（トークン使用量の監視と制御）。
    *   技術的負債の特定とリファクタリング。

**課題**
*   RAG機能を持つチャットボットのプロトタイプ開発。
*   エージェントの動作ログと判断根拠の分析レポート。

**必要ツール**
*   LangChain / LlamaIndex / CrewAI。
*   Pinecone / Weaviate / ChromaDB。
*   FastAPI / Flask（バックエンド用）。
*   GitHub（バージョン管理）。

**評価方法**
*   RAGの検索精度と回答の関連性（40%）。
*   エージェントの自律性とエラー処理能力（30%）。
*   コードの品質とドキュメント化（30%）。

## モジュール4：本格アプリ開発とリリース（第7-8ヶ月）

**学習目標**
AI機能を組み込んだ本格的なWebアプリケーションを開発し、クラウド環境にデプロイする。UI/UX設計、認証、データベース設計、およびCI/CDパイプラインの構築を通じて、製品としての完成度を高める。

**週次トピック**
*   **第1週：フルスタック開発の基礎**
    *   フロントエンド（React/Vue）とバックエンド（Node.js/Python）の連携。
    *   RESTful API設計のベストプラクティス。
    *   データベース設計（SQL vs NoSQL）。
*   **第2週：AIアプリ特有のUI/UX**
    *   生成コンテンツのストリーミング表示とローディング状態の設計。
    *   ユーザー入力の補完と自動修正機能の実装。
    *   フィードバックループのUI設計（ thumbs up/down ）。
*   **第3週：認証、認可、セキュリティ**
    *   OAuth2.0 / JWTを用いたユーザー認証。
    *   ロールベースのアクセス制御（RBAC）。
    *   AIアプリ特有のセキュリティリスク（プロンプトインジェクション対策）。
*   **第4週：クラウドデプロイメントとインフラ**
    *   AWS / GCP / Vercel / Railway へのデプロイ。
    *   コンテナ化（Docker）とオーケストレーション（Kubernetes基礎）。
    *   環境ごとの設定管理とシークレット管理。
*   **第5-6週：統合プロジェクト「AI支援ビジネスツール」の開発**
    *   要件定義、ワイヤーフレーム、データベーススキーマの設計。
    *   機能開発：ユーザー管理、AI機能、データ可視化。
    *   テスト駆動開発（TDD）の導入と単体テスト。
*   **第7-8週：リリース準備と運用**
    *   パフォーマンステストとセキュリティ監査。
    *   CI/CDパイプラインの構築（GitHub Actions）。
    *   リリースノートとユーザーマニュアルの作成。

**課題**
*   公開可能なAI機能付きWebアプリケーションの完成。
*   デプロイメント手順書と運用保守ガイドライン。

**必要ツール**
*   React / Next.js / Vue / Svelte。
*   FastAPI / Express.js。
*   PostgreSQL / MongoDB。
*   AWS / Vercel / Docker。
*   GitHub Actions。

**評価方法**
*   アプリケーションの機能性とバグの少なさ（40%）。
*   UI/UXの使いやすさとレスポンス速度（30%）。
*   セキュリティとデプロイメントの適切性（30%）。

## モジュール5：データ分析とマネタイズ（第9-12ヶ月）

**学習目標**
収集したデータを用いた分析基盤を構築し、AIモデルの性能評価を行う。さらに、開発したソリューションのビジネスモデルを設計し、収益化戦略を立案・実行する。

**週次トピック**
*   **第1週：データ収集と前処理**
    *   ログ収集とデータパイプラインの構築。
    *   データクリーニングと特徴量エンジニアリング。
    *   分析用データベース（BigQuery / Snowflake）の活用。
*   **第2週：AIモデルの評価と監視**
    *   精度、再現率、F1スコアなどの評価指標。
    *   プロンプト出力の品質評価（自動評価と人間評価）。
    *   モデルドリフトの検知と再学習サイクル。
*   **第3週：データ可視化とストーリーテリング**
    *   Tableau / Power BI / Metabase によるダッシュボード作成。
    *   分析結果に基づく意思決定の提案。
    *   ステークホルダーへの効果的なプレゼンテーション。
*   **第4週：ビジネスモデルと価格設定**
    *   SaaS、サブスクリプション、従量課金などのモデル比較。
    *   競合分析と差別化戦略。
    *   CAC（顧客獲得コスト）とLTV（顧客生涯価値）の計算。
*   **第5-6週：統合プロジェクト「AIソリューションのビジネスプラン」**
    *   既存のアプリまたは新規アイデアのビジネスモデルキャンバス作成。
    *   市場調査とターゲット顧客の特定。
    *   収益シミュレーションと財務予測。
*   **第7-8週：マーケティングとリリース戦略**
    *   デジタルマーケティングの基本（SEO、コンテンツマーケティング）。
    *   パイロットユーザーの募集とフィードバック収集。
    *   投資家向けピッチデッキの作成。
*   **第9-10週：最終プロジェクト「AIビジネスプロトタイプの立ち上げ」**
    *   収益化機能の実装（決済統合など）。
    *   マーケティングキャンペーンの実施。
    *   初期ユーザーの獲得と分析。
*   **第11-12週：総括とキャリアパス**
    *   プロジェクトの振り返りと学習成果の整理。
    *   ポートフォリオの完成と公開。
    *   今後の学習ロードマップとコミュニティ参加の提案。

**課題**
*   収益化されたAIソリューションのプロトタイプとビジネスプラン。
*   市場調査レポートとマーケティング戦略書。

**必要ツール**
*   Python (pandas, matplotlib, seaborn)。
*   Tableau / Power BI / Metabase。
*   Stripe / PayPal API。
*   Google Analytics / Mixpanel。

**評価方法**
*   ビジネスプランの実現可能性と論理性（40%）。
*   プロトタイプの収益化機能とマーケティング成果（40%）。
*   最終プレゼンテーションとポートフォリオの質（20%）。

このカリキュラムを通じて、受講生はAIの技術的深さとビジネス的応用の両方をマスターし、持続可能なAI駆動の価値創造を実現する能力を獲得します。各モジュールの修了時には、具体的な成果物（プロンプトライブラリ、自動化フロー、エージェント、Webアプリ、ビジネスプラン）が蓄積され、これがそのままポートフォリオとして機能します。
