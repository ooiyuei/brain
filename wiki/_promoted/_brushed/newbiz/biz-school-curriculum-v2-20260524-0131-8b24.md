# AIpaX school 1年カリキュラム v2 完成版

## 🎯 概要と教育哲学

**AIpaX school**は、単なるツールの操作法を教えるのではなく、「AIを手足のように使いこなす思考回路」を構築することを目的とした、中高生向けの実践型AIスクールです。

本カリキュラムは、大井自身が受講した**HR高等学院**の教育ノウハウ（思考フレームワーク）を統合し、**「マインドセット→自動化→論理構築→開発・リリース→分析・収益化」**という5つの段階で構成されます。全5モジュール（各2ヶ月、合計1年）を通じて、受講生はAIリテラシーだけでなく、問題解決能力、論理的思考、そして起業家精神を身につけます。

ターゲットは「AI時代の不安を持つ親」と「自身の未来を切り拓きたい中高生」です。既存のプログラミングスクールとの差別化ポイントは、「コードを書くこと」ではなく「AIを活用して価値を生み出すこと」に焦点を当てている点にあります。

## 📚 カリキュラム詳細：5モジュール構成

### Module 1: AIマインドセット & バイブコーディング体験
**期間**: 1ヶ月目 - 2ヶ月目
**テーマ**: 「AIは道具ではなく、思考の拡張装置である」

このモジュールでは、AIとの対話を通じて「正しい質問（プロンプト）の設計」を学びます。技術的な深さよりも、AIとの信頼関係構築と、創造的な発想の引き出し方を重視します。

#### 学習目標
- AIの基本原理と限界を理解する。
- 「バイブコーディング（直感とAIの共鳴）」による初期プロトタイピングを体験する。
- HR高等学院フレームワーク **F1 ストーリーテリング3軸**（普通の思考×武器×提案）を用いて、自分のアイデアを言語化する。

#### 週次トピック（8週間）
- **Week 1: AIとの出会い**
    - ChatGPT/Claudeの基本操作。プロンプトエンジニアリングの基礎（役割付与、文脈付与）。
    - 課題：「自分自身をAIに紹介する」プロンプト作成。
- **Week 2: 思考の可視化**
    - マインドマップ作成ツールとAIの連携。アイデアのブレスト手法。
    - 課題：「好きな趣味」をテーマにAIと対話し、新しい視点を3つ導出する。
- **Week 3: ストーリーテリングの基礎 (F1)**
    - 「普通の思考（現状）」→「武器（AIの力）」→「提案（解決策）」の構造理解。
    - 課題：身近な悩みをAIに相談し、ストーリーテリング形式で回答を整理する。
- **Week 4: バイブコーディング入門**
    - Replit/CursorなどのAI統合開発環境（IDE）に触れる。コードを書かずに自然言語でUIを設計する体験。
    - 課題：テキストだけで簡単なゲームのロジックをAIに記述させる。
- **Week 5: 画像・動画生成AI**
    - Midjourney/DALL-E/Stable Diffusionの基礎。ビジュアルプロンプトの設計。
    - 課題：Module 1のテーマに関連するキャラクター画像を生成し、ストーリーに組み込む。
- **Week 6: 情報リテラシーと倫理**
    - ハルシネーション（嘘）の見分け方、著作権、プライバシー保護。
    - 課題：AI生成コンテンツと事実確認の比較レポート作成。
- **Week 7: 初期プロジェクト企画**
    - 「AIで解決したいこと」を1つ選び、F1フレームワークで企画書をドラフト作成。
    - 課題：チーム（または個人）でのアイデア共有セッション。
- **Week 8: Module 1 総括と発表**
    - 企画書のフィードバック。次のモジュールへの準備。
    - 課題：「私のAIとの関係性」に関するエッセイ提出。

#### 必要ツール
- ChatGPT Plus / Claude Pro
- Replit / Cursor (無料枠)
- Midjourney / Canva AI
- Notion (ノート管理)

#### 評価方法
- プロンプトの精度（40%）
- アイデアの独創性と論理性（40%）
- 提出物のクオリティとプロセスの記録（20%）

### Module 2: 日常業務自動化 & チーム合意形成
**期間**: 3ヶ月目 - 4ヶ月目
**テーマ**: 「AIで時間を買い、チームで合意形成する」

個人での作業効率化から、チームでの協業へ。AIを活用した資料作成、データ処理、そしてHR高等学院フレームワーク **F2 合意形成4ステップ**を用いた議論のファシリテーション技術を学びます。

#### 学習目標
- 定型業務の自動化（RPAの基礎概念）をAIで実現する。
- AI支援による資料作成の高速化。
- チーム内でAIを仲介役として使い、合意形成を促進する。

#### 週次トピック（8週間）
- **Week 9: 自動化の基礎**
    - Zapier/Makeの概念理解。APIとは何か（簡易版）。
    - 課題：メール受信→スプレッドシート記録の自動化フロー設計。
- **Week 10: AIによるデータ処理**
    - Excel/Google SheetsでのAI関数活用。大量データの要約と分析。
    - 課題：適当なデータセットから傾向を読み取り、レポートを作成する。
- **Week 11: 資料作成の自動化 (Gamma/Notion AI)**
    - プレゼン資料の自動生成。デザインとコンテンツの分離。
    - 課題：「AIpaX schoolの紹介」スライドをAIで作成し、手修正を加える。
- **Week 12: チーム合意形成 (F2) - ステップ1,2**
    - 「現状共有」と「課題定義」におけるAIの活用。
    - 課題：チームで「学校でやりたいこと」をAIで整理し、共通課題を抽出する。
- **Week 13: チーム合意形成 (F2) - ステップ3,4**
    - 「案の創出」と「決定プロセス」。AIによる優劣比較の支援。
    - 課題：複数の解決案をAIで比較分析し、合意形成の根拠を作る。
- **Week 14: プロジェクト管理とAI**
    - Trello/AsanaとAIの連携。タスクの自動分割と進捗管理。
    - 課題：個人プロジェクトのタスクをAIに分解し、スケジュールを組む。
- **Week 15: 中間プロジェクト制作**
    - 「日常の悩み」をAI自動化で解決するミニプロジェクト。
    - 課題：自動化フローのデモ動画を作成する。
- **Week 16: 中間発表と振り返り**
    - 自動化の成果発表。チームワークの評価。
    - 課題：「自動化によって得られた時間」の使い道について考察。

#### 必要ツール
- Zapier / Make (無料枠)
- Gamma / Tome
- Notion AI
- Slack / Discord (チームコミュニケーション)

#### 評価方法
- 自動化フローの機能性と安定性（40%）
- チーム内での合意形成プロセスの質（40%）
- 資料の明確さとプレゼン力（20%）

### Module 3: エンジニアリング思考 & 個人探究テーマ設定
**期間**: 5ヶ月目 - 6ヶ月目
**テーマ**: 「論理的に問題を分解し、探究テーマを確定する」

直感から論理へ。HR高等学院フレームワーク **F3 11項目ロジックモデル**を用いて、卒業課題のテーマを具体化します。エンジニアリング思考（問題定義→解決策の設計→検証）を習得します。

#### 学習目標
- 複雑な問題をAIと論理的に分解する。
- HR高等学院 **F3 11項目ロジックモデル**を用いて、探究テーマの骨格を作る。
- MVP（最小実行可能製品）の設計。

#### 週次トピック（8週間）
- **Week 17: エンジニアリング思考入門**
    - 問題定義の重要性。5W1HとWhy-Why分析。
    - 課題：「なぜ、その問題が解決されないのか」をAIと対話して深掘りする。
- **Week 18: ロジックモデルの理解 (F3)**
    - 11項目（目的、対象、現状、課題、解決策、手段、成果、指標、リスク、リソース、継続性）の解説。
    - 課題：ロジックモデルのテンプレートに自分のアイデアを埋める。
- **Week 19: テーマの具体化**
    - 抽象的なアイデアを具体的なアクションプランに変換。
    - 課題：ロジックモデルの「解決策」部分をAIに批判され、改善する。
- **Week 20: 技術選定とアーキテクチャ**
    - 必要なAIモデル、API、データベースの選定。
    - 課題：技術スタックの比較表を作成し、理由を説明する。
- **Week 21: プロトタイプ設計**
    - ワイヤーフレーム作成。ユーザーフローの設計。
    - 課題：Figma/Whimsicalでプロトタイプを描く。
- **Week 22: 調査とリサーチ**
    - 競合分析、既存ソリューションの調査。
    - 課題：AIを用いた市場調査レポートを作成する。
- **Week 23: 探究テーマの確定**
    - メンターとの1on1。テーマのブラッシュアップ。
    - 課題：最終的な探究テーマとロジックモデルの提出。
- **Week 24: Module 3 総括**
    - 探究テーマのピアレビュー。
    - 課題：「私の探究テーマ」のポスターセッション風プレゼン。

#### 必要ツール
- Figma / Whimsical
- Miro / FigJam (論理展開)
- ChatGPT/Claude (深掘り用)

#### 評価方法
- ロジックモデルの整合性（50%）
- 問題定義の深さと具体性（30%）
- 技術選定の妥当性（20%）

### Module 4: 本格アプリ開発 & サービス企画
**期間**: 7ヶ月目 - 8ヶ月目
**テーマ**: 「AIを駆使して、世に出すプロダクトを作る」

コーディングの基礎から、AI支援開発の本格化。HR高等学院フレームワーク **F4 CHINTAI型5項目企画**を用いて、サービスの市場適合性（PMF）を検証します。

#### 学習目標
- Python/JavaScriptの基礎とAIコード生成の活用。
- F4 CHINTAI型5項目企画（顧客、課題、解決策、価値、収益）に基づくサービス設計。
- Webアプリまたはモバイルアプリのプロトタイプ完成。

#### 週次トピック（8週間）
- **Week 25: プログラミング基礎**
    - Python/JSの基本構文。変数、ループ、関数。
    - 課題：AIの解説を受けながら、簡単な計算プログラムを作る。
- **Week 26: AIコード生成の高度活用**
    - Cursor/GitHub Copilotの活用。バグ修正とリファクタリング。
    - 課題：既存のコードをAIに改善させる。
- **Week 27: フロントエンド開発**
    - React/Vueなどのフレームワーク概要。UIコンポーネントのAI生成。
    - 課題：簡単なUI画面をAIと共同で作成する。
- **Week 28: バックエンドとデータベース**
    - Firebase/Supabaseの活用。データの保存と取得。
    - 課題：ユーザー情報を保存する機能を実装する。
- **Week 29: サービス企画 (F4) - 顧客と課題**
    - ペルソナ設定。共感マップの作成。
    - 課題：ターゲットユーザーのインタビューシミュレーションをAIで行う。
- **Week 30: サービス企画 (F4) - 解決策と価値**
    - USP（他との違い）の明確化。プロトタイプの機能追加。
    - 課題：プロトタイプに「核心機能」を追加する。
- **Week 31: サービス企画 (F4) - 収益モデル**
    - 収益化のシミュレーション。コスト計算。
    - 課題：収益モデルのシートを作成する。
- **Week 32: プロジェクト完成とテスト**
    - バグテスト、ユーザーテスト。
    - 課題：プロトタイプの完成版を提出する。

#### 必要ツール
- Cursor / VS Code + GitHub Copilot
- React / Next.js / Supabase
- Figma (UI設計)

#### 評価方法
- プロダクトの完成度と機能性（40%）
- サービス企画書の論理性（F4準拠）（40%）
- コードの品質とドキュメント（20%）

### Module 5: データ分析 & マネタイズ & 卒業課題発表
**期間**: 9ヶ月目 - 12ヶ月目
**テーマ**: 「データで検証し、価値を収益に変える」

最終段階。HR高等学院フレームワーク **F5 産業課題×AI 4項目**を用いて業界分析を行い、データを基にした意思決定と、実際のビジネスモデルの構築を行います。

#### 学習目標
- データ分析によるユーザー行動の可視化。
- F5 産業課題×AI 4項目（業界構造、競合、技術トレンド、参入障壁）に基づく戦略策定。
- 卒業課題の発表と、実際のリリースまたはピッチ。

#### 週次トピック（8週間）
- **Week 33: データ分析入門**
    - Google Analytics/Heatmapの概念。ユーザー行動の計測。
    - 課題：プロトタイプの仮想的なデータからインサイトを得る。
- **Week 34: A/Bテストと改善**
    - 仮説検証の手法。AIによるデータ解釈。
    - 課題：改善案を提案し、その根拠をデータで示す。
- **Week 35: 産業分析 (F5) - 業界構造**
    - 対象業界のバリューチェーン理解。
    - 課題：業界マップを作成する。
- **Week 36: 産業分析 (F5) - 競合と技術トレンド**
    - 競合優位性の再確認。最新AI技術の適用可能性。
    - 課題：SWOT分析をAIと行う。
- **Week 37: マネタイズ戦略の深化**
    - サブスクリプション、広告、課金などの詳細設計。
    - 課題：財務予測モデルを作成する。
- **Week 38: ブランディングとマーケティング**
    - SNS活用、コンテンツマーケティング。
    - 課題：プロモーション用のSNS投稿をAIで作成する。
- **Week 39: 卒業課題の仕上げ**
    - 最終プレゼン資料の作成。リハーサル。
    - 課題：ピッチデッキの完成。
- **Week 40: 卒業発表会**
    - 外部メンター・保護者向けプレゼン。
    - 課題：卒業証書授与。

## 📊 評価方法の詳細

AIpaX schoolの評価は、単なる正解不正解ではなく、**「プロセスの質」**と**「成長の度合い」**を重視します。

1.  **ポートフォリオ評価 (40%)**:
    - 各モジュールで提出したプロンプト、コード、企画書、プロトタイプなどの成果物の蓄積。
    - 思考過程の記録（ログ）が含まれているか。
2.  **対話とフィードバック (30%)**:
    - メンターとの1on1での議論の深さ。
    - ピアレビューでの他者への建設的なフィードバック能力。
3.  **最終プロジェクト (30%)**:
    - Module 5の卒業課題に対する、論理性、独創性、実行可能性。
    - プレゼンテーションにおける説得力と情熱。
