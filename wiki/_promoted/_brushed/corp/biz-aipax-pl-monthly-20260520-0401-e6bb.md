# AIpaX β1社月50万+運用継続のPL予測と6社契約までの戦略

## 1. 概要と前提条件

本文書は、B2B AI導入支援事業「AIpaX」のβ1期（初期顧客4社契約済み）における、月間売上50万円超えを達成するための損益構造と、6社契約（通常価格移行期）までの財務予測を記述する。

AIpaXの核心は、「会社別AI保存庫コア（マザーボード）」の構築と、その上に乗せる業務AIツールの提供にある。本モデルは、単発の開発費収益ではなく、保存庫の維持・更新・検索精度向上を目的とした「継続的な月額運用収益」を軸としたビジネスモデルである。

現在の状況は、初回事例価格（初期費用50〜80万円、月額5万円）で4社契約済みであり、5社目以降は通常価格（初期100〜150万円、月額10〜15万円）へ移行する段階にある。本予測では、既存4社の継続と、新規3社の獲得による合計6社契約体制を想定し、PL（損益計算書）の構造を解明する。

## 2. 売上構成の分析

AIpaXの売上は、大きく「初期構築費（一時的なキャッシュフロー）」と「月額運用費（継続的な収益）」の2つに分かれる。

### 2.1 初期構築費（初期費用）
顧客ごとに「保存庫コア」の構築と、最初の業務ツール（例：リアルタイム電話支援）の実装を行う。
*   **β1期価格帯**: 50〜80万円（初回事例価格）
*   **通常価格帯**: 100〜150万円
*   **特徴**: 顧客の資料量、対象領域の広さ、既存データの整理難易度によって変動するが、基本は「保存庫の土台作り」に対する対価である。

### 2.2 月額運用費（継続収益）
保存庫内のデータ更新、入庫前処理、検索条件の調整、回答カードの改善、軽微な不具合修正などを行う。
*   **β1期価格帯**: 5万円/月
*   **通常価格帯**: 10〜15万円/月
*   **特徴**: 顧客の業務量やデータ更新頻度に応じて設定される。この月額収益が事業の安定性を担保する。

## 3. 原価構造の解明

AIpaXの原価は、主に「人件費（実装工数）」と「変動費（AI API + 保管庫運用）」に分類される。固定費は最小限に抑えられている。

### 3.1 実装工数（人件費）
保存庫コアの構築とツールの実装には、高度な技術力と業務理解が必要である。
*   **初期構築**: 1社あたり約40〜60時間（ヒアリング、データ整理、Supabase構築、OpenAI API連携、テスト）。
*   **月額運用**: 1社あたり月間5〜10時間（データ更新、検索精度のチューニング、顧客対応）。
*   **コスト換算**: 開発者単価を時給5,000円と仮定すると、初期構築で20〜30万円、月額運用で2.5〜5万円/社となる。

### 3.2 AI API + 保管庫運用（変動費）
*   **OpenAI API**: 埋め込み（Embedding）、検索（Search）、生成（Chat）の使用量に応じて課金される。
*   **Supabase**: データ保存量、API呼び出し数に応じて課金される。
*   **推定コスト**: 1社あたり月額1,000〜3,000円程度。初期構築時のデータインデックス作成にも一時的なコストがかかるが、運用段階では非常に低い。

## 4. 損益分岐点の算出

AIpaXの損益分岐点を求めるには、固定費と変動費、そして単価を明確にする必要がある。

### 4.1 固定費
*   **インフラ費**: Supabase、ドメイン、メールサーバーなど。月額約1万円。
*   **管理費**: 経理、法務、営業のための間接コスト。月額約2万円。
*   **合計固定費**: 月額約3万円。

### 4.2 変動費
*   **API・保管庫費**: 1社あたり月額2,000円。
*   **運用工数**: 1社あたり月額4万円（時給5,000円 × 8時間）。
*   **合計変動費**: 1社あたり月額4.2万円。

### 4.3 損益分岐点の計算
*   **β1期顧客（月額5万円）の場合**:
    *   1社あたりの貢献利益 = 5万円 - 4.2万円 = 0.8万円
    *   損益分岐点顧客数 = 固定費3万円 ÷ 0.8万円 = 3.75社
    *   つまり、β1期顧客が4社いれば、月額ベースで黒字となる。

*   **通常価格顧客（月額10万円）の場合**:
    *   1社あたりの貢献利益 = 10万円 - 4.2万円 = 5.8万円
    *   損益分岐点顧客数 = 固定費3万円 ÷ 5.8万円 = 0.5社
    *   つまり、通常価格顧客が1社いれば、月額ベースで大幅な黒字となる。

## 5. 6社契約までの予測PL

現在4社契約済み（β1期価格）であり、5社目以降を通常価格で獲得することを想定する。

### 5.1 現状（4社契約時）
*   **売上**:
    *   初期費用: 4社 × 60万円（平均） = 240万円（一時的）
    *   月額売上: 4社 × 5万円 = 20万円
*   **原価**:
    *   運用工数: 4社 × 4万円 = 16万円
    *   API・保管庫費: 4社 × 0.2万円 = 0.8万円
    *   固定費: 3万円
*   **営業利益**:
    *   月額: 20万円 - 16万円 - 0.8万円 - 3万円 = 0.2万円（ほぼ損益分岐）
    *   初期費用回収後、月額収益が純利益に直結し始める。

### 5.2 5社目獲得時（通常価格移行）
*   **売上**:
    *   月額売上: (4社 × 5万円) + (1社 × 10万円) = 30万円
*   **原価**:
    *   運用工数: 5社 × 4万円 = 20万円
    *   API・保管庫費: 5社 × 0.2万円 = 1万円
    *   固定費: 3万円
*   **営業利益**:
    *   月額: 30万円 - 20万円 - 1万円 - 3万円 = 6万円

### 5.3 6社目獲得時（目標）
*   **売上**:
    *   月額売上: (4社 × 5万円) + (2社 × 10万円) = 40万円
    *   ※5社目も通常価格に移行した場合: (3社 × 5万円) + (3社 × 10万円) = 45万円
*   **原価**:
    *   運用工数: 6社 × 4万円 = 24万円
    *   API・保管庫費: 6社 × 0.2万円 = 1.2万円
    *   固定費: 3万円
*   **営業利益**:
    *   月額: 40万円 - 24万円 - 1.2万円 - 3万円 = 11.8万円
    *   ※通常価格移行が進むと、利益率はさらに向上する。

## 6. 戦略的考察と結論

### 6.1 月50万円の達成可能性
現在の4社契約（月額20万円）では、月50万円には届かない。しかし、初期費用の回収と並行して、5社目、6社目の獲得を進めることで、月額収益は指数関数的に増加する。
特に、5社目以降を通常価格（月額10万円以上）で契約できれば、顧客1社あたりの貢献利益が大幅に向上するため、月50万円の達成は容易になる。

### 6.2 6社契約までのアクションプラン
1.  **既存4社のオンボーディング徹底**: 保存庫コアの価値を最大化し、月額運用の必要性を顧客に実感させる。これにより、解約リスクを低減し、将来の価格改定（通常価格移行）の基盤を作る。
2.  **5社目・6社目の獲得**: 通常価格（初期100〜150万円、月額10〜15万円）で契約を獲得する。この段階では、β1期の事例を活用した営業が有効である。
3.  **標準化と効率化**: 保存庫コアの構築プロセスを標準化し、実装工数を削減する。これにより、1社あたりの原価を下げ、利益率を向上させる。

### 6.3 結論
AIpaXのβ1期は、月50万円の売上を初期費用の回収と月額収益の積み上げで達成する段階にある。既存4社の継続と、通常価格での新規2社獲得により、月額収益は40万円〜45万円に達し、営業利益も10万円以上を見込むことができる。今後は、この基盤の上に、より高単価な本格導入（月額20万円以上）の顧客を獲得し、事業の安定性と成長性を両立させることが重要である。
