# [[entities/aipax]] 深掘り (2026-06-04)

> 原ドラフト(qwen3:8b)は市場規模・競合価格・顧客属性を捏造していたので全削除した。
> 数値は [[sources/aipax-spec-v1-2026-05-19]] と [[entities/aipax]] に書いてある事実だけを残し、
> 裏が取れていないものは《要出典》《要記入》にしてある。埋める前に営業文や提案書に流用しない。

## いまの立ち位置（事実だけ）

俺のAIpaXは、単発のAIツールを売る事業じゃない。
顧客企業ごとに「会社別AI保存庫コア（マザーボード）」を作り、その上に必要な業務AIツール（拡張機能）を載せていく構造で組んでる。

- **契約状況**: 4社契約済み・5社目を営業中
- **5社目以降は通常価格へ移行**する（初回事例価格を5社目に使わない、ここは絶対に混同しない）
- **最初の主力商品**: 保存庫コア + リアルタイム電話／営業回答支援ツール

## ターゲット顧客（canonで確定している範囲）

| 項目 | 内容 | 出典 |
|------|------|------|
| 誰に | 中小企業の経営者 | entity 30秒サマリー |
| 条件 | AI導入意欲がある・予算50万円を出せる層 | entity 30秒サマリー |
| 主な課題 | 営業電話で回答がズレる／新人が社内ルールを何度も聞く／フォロー漏れ／問い合わせ対応の属人化 | spec §2 |

ここから先（業種の偏り・年商レンジ・営業人数）は、4社の実顧客プロファイルを棚卸しして埋める。
原ドラフトは「製造・小売・サービス業／年商5,000万〜3億／営業1〜5人」と書いてたが、これは裏取りゼロの推測なので捨てた。

- 既存4社の業種: 《要記入：契約済み4社の実データから》
- 年商レンジ: 《要記入》
- 組織規模（営業人数）: 《要記入》

## 価格構造（段階制・ここが事業の肝）

| 段階 | 初期 | 月額 | 誰に出すか |
|------|------|------|-----------|
| 初回事例価格 | 50〜80万円 | 5万円前後 | 1〜4社目（事例化が目的） |
| 通常価格 | 100〜150万円 | 10〜15万円以上 | **5社目以降** |
| 本格導入 | 200〜500万円 | 20〜50万円 | 全社・複数部署・重い責任範囲 |

出典: [[sources/aipax-spec-v1-2026-05-19]] §15 / entity「価格感」。

### 価格を決める要素（資料の枚数では決めない）
管理対象領域 / 更新頻度 / ツール数 / 利用量・実費 / 外部連携 / 運用対応速度 / 紙資料・OCR / 大量旧資料整理 / 自社側か顧客側か。
対象領域内なら顧客別条件・例外条件は削らない。そこまで扱えるのが保存庫の価値だから、値付けもそこに乗る。

### 追加ツール（単体メニュー）
保存庫コア導入後の積み増し。これも canon にあるレンジだけ:

| メニュー | 価格帯 |
|---------|--------|
| 導入前簡易設計 | 3万円前後 |
| 軽い1業務フロー導入 | 20〜40万円 |
| 単体上位／軽い連携あり | 40〜80万円 |
| パッケージ導入 | 40〜90万円 |
| 保存・管理ありパッケージ | 80〜150万円以上 |

## 収益の作り方（月収100万円の道筋）

entity に置いてある試算をそのまま使う。これは俺が引いた線で、捏造じゃない。

| 構成 | 月額 |
|------|------|
| 通常価格顧客 5社（10万×5） | 50万円 |
| 本格導入 1社（20万） | 20万円 |
| 初回事例価格 2社（5万×2） | 10万円 |
| AI単体メニュー（スポット） | 20〜30万円 |
| **計** | **100〜110万円** |

顧問×継続収益（月額運用）＋単発スポット（単体メニュー）の組み合わせ。
月額運用に「含むもの／含まないもの」の線引きは spec §16-17 に確定済みなので、追加料金がどこから発生するかは詰まってる。ここが曖昧だと月額が赤字化するから、提案時に必ず範囲表を見せる。

## 競合（ここは埋まっていない・捏造禁止ゾーン）

原ドラフトは Notion+AI＝月10万、Salesforce AI＝月30万…と具体価格を並べてたが、**全部出典なしの捏造**なので削除した。AIpaXの forbidden に「競合価格の新規追加禁止」が明記されてる。

competitive-intel として埋めるべき項目だけ枠で残す:

| 競合候補 | 価格帯 | 顧客層 | AIpaXとの差 |
|---------|--------|--------|------------|
| 《要出典：実在の競合名》 | 《要出典》 | 《要出典》 | 《要記入》 |

AIpaX側の差別化軸（これは事実）:
- 会社別AI保存庫コアを汎用基盤として標準化 → 毎回ゼロから作らない
- 「保存庫コアとツール出力を混ぜない／AIが勝手に正式情報を作らない」という5原則で、顧客資料の正しさは顧客側が決める設計
- 対象領域内の顧客別条件・例外条件を削らない

競合の実価格・実顧客層は WebSearch か商談ヒアリングで取ってから [[meta/competitive-intel]] に入れる。それまでは差別化の優劣を断定しない。
