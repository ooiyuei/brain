# 森岡毅式 コンセプト診断テンプレ集

> シンコード式が「**数字で判定**」なら、森岡式は「**概念で判定**」。
> 両方揃って初めて事業が確実に動く。

---

## 🎯 使い方

シンコード式と並走する。

| 判定軸 | フレーム | 出力先 |
|---|---|---|
| 数字 (CPA/PMF/LTV) | シンコード式 | `wiki/_final/decisions/cpa-check-*.md` 他 |
| 概念 (コンセプト強度) | **森岡式 (このファイル)** | `wiki/_final/decisions/concept-diagnosis-*.md` |

両方クリアで「**売れるコンセプト × 売れる数字**」の二重補強になる。

---

## 📊 1. 絶対に買う1人 定義シート (全事業必須)

各事業の「**絶対に買う1人**」を実名 (仮名) で具体的に定義する。

### フォーマット

\`\`\`markdown
# 絶対に買う1人: {事業名}

## プロファイル
- **仮名**: __ (例: 山田太郎)
- **年齢**: __
- **居住地**: __ (具体的な市町村まで)
- **職業/役職**: __
- **家族構成**: __
- **年収/予算**: __
- **デバイス/SNS**: __ (どこで接触可能か)

## 痛み (深層)
- 表層の痛み: __
- 中層の痛み: __
- 深層の痛み (本能レベル): __

## なぜこの商品を「絶対に買う」のか
- 機能的価値: __
- 情緒的価値: __
- 自己実現価値: __

## 競合との比較
- 既存代替手段: __
- なぜそれを使わないのか: __
- 自社商品を選ぶ決定的理由: __

## 接触経路
- 認知のきっかけ: __
- 検討のきっかけ: __
- 購入のきっかけ: __
\`\`\`

**判定**: この1人を見て「絶対に買う」と100%断言できるか?
- Yes → コンセプト合格
- うーん → コンセプト未完成・WHO/WHATの精緻化必要

## 📊 2. WHO/WHAT/HOW 設計シート (新規事業/コンセプト見直し時)

森岡鉄則: **WHO → WHAT → HOW の順** で設計。逆は失敗。

\`\`\`markdown
# {事業名} ブランド設計

## STEP1: WHO (誰の脳内に構築するか)
- ターゲット人格: __ (上記「絶対に買う1人」と同じ)
- 上位カテゴリーでの位置: __ (脳内でSKU→ブランド→カテゴリーの順で選ばれる)
- 凡人 or 狂人 憑依: __ (どちらを優先するか)

## STEP2: WHAT (何のブランドエクイティを構築するか)
4要素に分解する:

1. **信頼できる○○**: __
2. **効果的な○○**: __
3. **継続的な○○**: __
4. **(独自要素)**: __

### POD (差別化ポイント): __
### POP (同等であるべきポイント): __

## STEP3: HOW (どうやって構築するか)
- タッチポイント1: __
- タッチポイント2: __
- タッチポイント3: __

各タッチポイントで「WHATを脳内に積む」設計になってるか?
- [ ] 全タッチポイントで一貫したエクイティ表現
- [ ] ベネフィット・ステートメント (ブランド名+短い便益) を必ず添える
- [ ] Brand Character (人格) が定義されている

## 判定
- [ ] WHO/WHAT/HOWの順で設計したか
- [ ] WHATは4要素に分解されてるか
- [ ] HOWはWHATを脳内に積む手段になってるか
- [ ] 「絶対に買う1人」が実在するか
- [ ] 全て満たす → コンセプト合格・実装GO
\`\`\`

## 📊 3. 凡人 vs 狂人 憑依選定シート

| 判断軸 | 凡人憑依 | 狂人憑依 |
|---|---|---|
| 市場規模 | 大 (マス) | 小 (カルト) |
| 1人あたりLTV | 低 | 桁違いに高 |
| 浸透率上限 | 高 | 低 |
| 購入頻度 | 標準 | 異常に高 |
| 口コミ拡散 | 弱い | 強い (オタク仲間に伝染) |
| 競合参入難度 | 低 | 高 (深い文脈理解必要) |
| 適する事業 | 日用品・食品・大量消費財 | エンタメ・趣味・教育・SaaS |

**大井5事業の推奨憑依**:
- AIpaX → **狂人** (AIで会社変えたい社長)
- AIpaX school → **狂人** (起業したくて燃えてる高校生)
- Testall → **狂人** (医学部絶対志望)
- AIpa Web → **狂人** (採用LPで本気で人を採りたい中小企業)
- EEMUS → **狂人** (静岡を盛り上げたい経営者)

→ 大井の主力5事業は全て **狂人憑依が最適**。マス狙わない。

## 📊 4. プレファレンス vs 認知 vs 配荷 投資配分シート (週次)

森岡: **戦略変数はたった3つ・どれに集中するか決めろ**

\`\`\`markdown
# 投資配分判定: {事業名} ({日付})

## 現状診断
- **プレファレンス** (相対好意度・1番選ばれる確率): __/10
- **認知** (ブランドを知ってる人の割合): __/10
- **配荷** (買える場所にある率): __/10

## 弱点特定
- 最も低い変数: __
- 原因: __

## 投資配分判断
- プレファレンス強化: __% (LP改善・コンセプト磨き・商品改良)
- 認知強化: __% (広告・SNS・PR)
- 配荷強化: __% (販路・チャネル・営業)

## 来週の最大コミット
- 何にいくら投資するか: __円
- 期待効果: __
\`\`\`

**判定ルール**:
- プレファレンス1番低い → 商品・コンセプト改良が最優先
- 認知1番低い → 広告/SNS強化
- 配荷1番低い → 営業/販路強化

## 📊 5. ブランドエクイティ設計図 (LP/広告作成前必須)

森岡: **広告の設計図 = ブランドエクイティ**。これ無しで広告打つな。

\`\`\`markdown

## 1. 脳内につくりたい最終形 (版画)
- 消費者がこのブランドを思い浮かべた時の感情: __
- 連想する3つの記号: __, __, __
- 競合と比較した時の差別化記憶: __

## 2. このエクイティを実現するためのマーケティングコンセプト (版木)
- 一言で言うと: __ (15字以内)
- なぜそれが効くのか: __
- 脳内記号として既存のレールに乗るか: __

## 3. 全タッチポイント設計
- LP ヒーロー: __
- LP セクション1: __
- LP セクション2: __
- LP セクション3: __
- LP CTA: __
- 広告コピー1: __
- 広告コピー2: __
- 商品名/サービス名: __
- カスタマーサポート対応: __

→ 全部 上記「版画」の方向にエクイティを積み上げる設計になってるか?
\`\`\`

## 🚨 森岡の鉄則 (判定時の指針)

1. **WHO → WHAT → HOW の順を絶対守る** (逆から発想すると顔のないブランドに)
2. **狭めるな・拡げよ** (ターゲット早期絞り込みは数学的に間違い)
3. **絶対に買う1人が実在するか** (Yes/No明確に出るレベル)
4. **広告打つ前にエクイティ設計図** (無いと広告止めたら売上消える)
5. **凡人憑依より狂人憑依** (LTV桁違い・拡散力強い)
6. **コンセプトは "コンセプチュアル・セル効果"** = 事前に脳内に刷り込んで価値実感を増幅
7. **マインドオープニング or ハートオープニング** どちらで攻めるか決める
