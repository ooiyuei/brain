# シンコード式 数字判定テンプレ集

> シンコードの本質: **「数字を実測して判定する」**。AIに数字を作らせない。大井が記入する。
> このファイルは毎週・毎月のGO/PIVOT判断の起点になる。

---

## 🎯 使い方

1. `shincoder_loop.ps1` が定期的に `_final/decisions/` に空テンプレを出力する
2. 大井が **実測値** を埋める (推測値で埋めない)
3. 判定ルールに従って GO/HOLD/PIVOT を決める
4. 結果を `wiki/daily/` に1行ログとして残す

---

## 📊 1. CPA判定テンプレ (週1)

**シンコード基準**:
| CPA | 判定 | 次のアクション |
|---|---|---|
| ≤ 100円 | **超GO** | 即追加投下5万円・本開発スタート |
| 100-300円 | **GO** | 本開発スタート・継続広告 |
| 300-500円 | **HOLD** | LPコピー1箇所だけ変えて再検証 (3日) |
| ≥ 500円 | **即PIVOT** | LP or オーディエンス全変更 |
| 登録0 | **即PIVOT** | 別アイデアへ |

**計算式**: `CPA = 広告費 ÷ ウェイトリスト登録数`

## 📊 2. PMF判定テンプレ (週1)

**3軸全てクリアでPMF達成宣言**:

### 軸1: 定量 (3つ全て)
- [ ] 継続率 15%以上 / 30日
- [ ] CVR 3%以上 (LP→課金 or 訪問→アクション)
- [ ] オーガニック成長 (広告止めても新規増える)

### 軸2: 訂正 (1つ以上)
- [ ] 「なくなったら残念」回答 40%以上 (Sean Ellis Test)
- [ ] 紹介口コミ自然発生

### 軸3: 経済
- [ ] LTV ≧ CAC × 3

**判定パターン**:
- 3軸全部クリア → スケール (集客拡大期へ)
- 軸1-2クリア・軸3未達 → 価格見直しorAI原価圧縮
- 軸1未達 → 改善2-3週間継続・出ない場合ピボット
- 6ヶ月かけても軸1未達 → 寝かせる or 終わらせる

## 📊 3. ファネル分析テンプレ (週1)

**各事業共通の段階**:

| 段階 | 測定方法 | 改善焦点 |
|---|---|---|
| 認知 | LP訪問数 / 営業開封率 | 広告コピー・件名 |
| 興味 | 返信数 / 問い合わせ | LPコピー・差別化 |
| 検討 | デモ予約 / 商談 | 提案資料・実例 |
| 決定 | 契約 / 課金 | クロージング・価格 |
| 継続 | 30日継続率 | オンボーディング・サポート |

**離脱率最大の段階 = 最優先改善箇所**

**マイクロコピー1個変更でCVR 1.3倍**実例 (シンコード):
- ボタン文言: 「無料登録」→「今すぐ試す (3秒で完了)」
- 価格表示: 「月額980円」→「月額980円 (1日33円)」
- CTA色: グレー→オレンジ

## 📊 4. ユニットエコノミクス テンプレ (月1)

**計算式**:
- **LTV** = 平均月額 × 平均継続月数 - AI API原価
- **CAC** = (広告費 + 営業時間×時給) ÷ 新規獲得数
  - 大井1人運営時は時給0円扱い
- **倍率** = LTV ÷ CAC

**判定**:
- 倍率 ≥ 3: スケール (広告・インフルエンサー・紹介プログラム全開)
- 倍率 1-3: 改善余地 (LTV伸ばす or CAC下げる)
- 倍率 < 1: 即停止 or 価格見直し

**AI APIコスト圧縮テク**:
- 軽量モデル併用 (重い処理だけ高価モデル)
- プロンプトキャッシュ活用 (Anthropic 5分TTL)
- ローカルLLM活用 (qwen3.6等で前処理)

## 📊 5. 逆算思考フレーム (月初)

**月収目標 → 1日あたり → 1事業あたりへ分解**:

```
月収100万 (大井 2026/6/21期限・残26日)
↓
1日あたり: 33,000円必要
↓
事業分散:
  AIpaX (1社=月10-15万): 月3-5社追加 = 1日1-2社接触必要
  Testall (β月980円×100名 = 98,000円): 1日3-5名募集
  AIpa Web (モニター月10万×3社 = 30万円): 1日1社接触必要
  → 合計 73-83万円 / 月
↓
残20-30万 = リファラル+EEMUS夢AWARD賞金狙い
```

**毎月初に再計算する**。

## 📊 6. アイデア→ローンチ ステップチェックリスト

新規事業候補のアイデアが出たら、このチェックを順番に進める:

### Step 1-2: アイデア+LP (1日)
- [ ] マイクロフラストレーション特定
- [ ] 競合調査 (存在=需要証明・1機能特化で勝つ)
- [ ] Claude Design + GPT Image 2 でLP生成
- [ ] ウェイトリスト機能 (Form Spree) 実装

### Step 3-4: 広告検証 (1-2週間)
- [ ] X広告 5,000円投下
- [ ] Meta広告 5,000円投下
- [ ] **CPA計算** → 上記判定テンプレ参照
- [ ] **GO** or **PIVOT** 決定

### Step 5-7: 開発+リリース (1-3週間)
- [ ] MVP機能絞り込み (Google認証 + コア機能1 + 課金のみ)
- [ ] 4-5日で初版リリース
- [ ] Google Analytics + PostHog 必須

### Step 8-9: 分析+改善 (継続)
- [ ] **ファネル分析** → 上記テンプレ参照
- [ ] ABテスト (週1)
- [ ] フィードバックループ

### Step 10: PMF達成 (1-6ヶ月)
- [ ] **PMF判定** → 上記テンプレ参照
- [ ] 達成シグナル全部クリア確認

### Step 11: スケール (PMF後)
- [ ] **ユニットエコノミクス計算** → 上記テンプレ参照
- [ ] LTV/CAC ≥ 3 確認
- [ ] 広告・インフルエンサー・リファラル全開
- [ ] バイアウト or 続行 判断

## 🚨 シンコード鉄則 (判定時の指針)

1. **PMF達成前は広告費かけない** (湿った木材に火つけても無駄)
2. **完璧主義 = 現実逃避** → 4-5日でMVP出す
3. **CPA300円超は即ピボット** → 8割の事業で発生する
4. **継続率 < 15% = PMF未達** → 改善2-3週間で出ない=ピボット
5. **「自分で完璧に」は罠** → Claude Codeに丸投げが正解
6. **数字は実測のみ** → AIに数字を作らせない (このテンプレの最重要原則)
