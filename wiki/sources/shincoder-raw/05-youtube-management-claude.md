# クロードコードに YouTube 運営丸投げ (仮想組織構築)

## 📊 サマリー
- クロードコード仮想組織で業務自動化
- YouTube制作6ステップ
- スキル化・サブエージェント・ハーネス設計
- 1サンプル→任意の業務に応用可

## 🔥 6ステップ YouTube制作ワークフロー
1. **企画・リサーチ** (Web検索・トレンド分析)
2. **台本構成** (テンプレ・フック+本編+CTA+エンディング)
3. **サムネ・タイトル** (GPT Image 2・他人と被らない)
4. **動画撮影** (人間・AI動画はYouTube収益化止リスク)
5. **動画編集** (Videoy / FFmpeg / ElevenLabs・無音カット+テロップ)
6. **投稿** (人間・YouTube API投稿不可)

## 💡 核心ノウハウ

### 業務効率化前の必須作業
**「現状ワークフローを書き出して言語化する」**
- これしないとクロードコードに伝えられない
- 部署 → スキル化 → サブエージェント化 の順

### スキル化 vs サブエージェント
- **スキル** = ワークフロー・マニュアル
- **サブエージェント** = 重い作業用・独立コンテキスト
- 例: 動画編集は重いから sub-agent 化推奨
- 親 1個 + サブ複数 = 出力品質向上

### クロードコード組織図 (本動画事例)
```
秘書室 (日常雑務・思考吐き出し)
└── YouTube部
    ├── リサーチチーム
    ├── 台本チーム
    ├── サムネチーム
    └── 編集チーム (サブエージェント化推奨)
```

### マニュアル育成
- 1発で完璧は無理
- 出力品質チェック → CLAUDE.md修正
- 新人部下マネジメントと同じ
- Tips集 → ナレッジ集 → 専門エージェント化

### 重要発見
- 「言語化できる業務 = AI委任可能」
- 「抽象化能力・言語化能力・委託する力」が必須スキル
- 「サブエージェントで出力品質が上がる」(独立コンテキスト)
- 「業務全体マニュアル化 → スキル → サブエージェント のレイヤー化」
