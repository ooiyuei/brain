# 9割が知らないクロードコード スラッシュコマンド10選

## 📊 サマリー
- クロードコードの隠れた機能・コマンド
- 生産性UP・コンテキスト管理・並列処理

## 🔥 10選 + チップス

### 1. /effort max
- デフォルトは high
- 推論深さ調整 (low/medium/high/max)
- 複雑な要件定義・複雑実装は max 推奨
- スラッシュ /model で確認可能

### 2. btw/ (By The Way)
- 待機中の質問
- コンテキスト汚染なし
- ちょっとした学習・調査用
- esc で抜ける

### 3. /resume
- セッション復活
- 過去の会話履歴の続きから
- 自動でセッション名つく (rename可能)
- 引き継ぎ・並列クロード作業時に必須

### 4. design.md ファイル
- アプリのデザイントークン定義
- ノーション風・クロード風・リセンド風 配布されてる
- アプリ作成前に必ず指定
- デザイン特化エージェント+ design.md = 一貫性UP

### 5. /rewind
- 巻き戻し
- ESC 2回でも可
- 出力ダメな時・指示変えてやり直したい時
- レジウム と 違い: 会話含めて巻き戻す

### 6. /batch (並列処理)
- 似た実装の並列処理
- 例: ダークモード対応・各コンポーネントへのテストコード
- git worktree 自動活用 = 別部屋で独立実装
- 1個のクロードより並列の方が正確に実装

### 7. オートランモード (デフォルト推奨)
- /config の `default permission mode = auto`
- バイパスより安全 (危ない時人間に聞く)
- act edit on と切り替え可能

### 8. /fewer-permission-prompts
- 過去履歴から自動許可リスト
- 毎回許可してたコマンドを記憶
- セキュリティ厳しい会社向けにdナイ設定追加可
- 設定先: settings.json

### 9. /insights
- クロードコード使用分析
- 「あなたの使い方」レポート
- 改善案 (短期+長期)
- 完全文・改善 + 強み弱み分析
- 1度実行で大量分析

### 10. /CC max (自分のCLI・宣伝)
- ターミナル分割
- VS Code 依存しない
- 軽量
- 概要欄に公開・無料

## 💡 補足チップス

### bash !
- クロードコード内で ! 先頭 = bash実行
- ローカルサーバー立ち上げなど
- ctrl+b でバックグラウンドタスク確認
- ターミナル分割不要

### グローバル MCPの活用
- リセンドのデザイントークン使う場合: クロードコードに直接貼り付け
- DUI MCP (図解化) で仕組み理解
- バイブラーニング (図でAIに教えてもらう)

### git worktree (中級)
- 同じ部屋で複数クロード = 競合
- git worktree = 別部屋でコピー実装
- batch コマンドが自動でやってくれる
