## ⭐ 大井に直接効くTOP10 (隅から隅まで抽出)

### 1. CPA = 100-300円が需要証明ライン
**手法**: LP作って X/Meta 広告に1万円投下→ウェイトリスト登録数測定 = CPA
- 100-300円 → 需要あり・開発GO
- 300円超 → ピボット検討
**大井応用**: AIpaX 5社目契約の前にLP作って広告検証→1社あたり獲得コストを定量化

### 2. PMF達成シグナル3軸
- **定量**: 継続率15%+/30日 / CVR 3%+ / オーガニック成長
- **訂正**: 「なくなったら残念」回答 40%+ / 紹介口コミ自然発生
- **経済**: LTV ≧ CAC × 3
**大井応用**: Testall β顧客100名集まったらこの基準でPMF判定→広告投下するか決める

### 3. アイデア発掘=自分の痛みから (Maicro frustration)
- ペインポイント = 小さなイライラを毎日メモ
- 重複するペインが本物
- 競合存在しても「1機能特化」で勝てる(AIpaXがGPT/Claude直接利用に勝つロジック)
**大井応用**: 大井のメモリ ChatGPT 789件分析と同じパターン。SNSでバズる構造作りに反映

### 4. 競合いる方が良い (需要証明)
- キャンバや Photoshop ある中、シンコードがサムネAIで月3万円
- AIpaXは AIコンサル大手 ある中、4社契約取れてる
**大井応用**: 「競合いるから無理」捨てる。「1機能特化で大手の隙間」狙う

### 5. 逆算思考フレーム (月20万円→分解)
```
月20万 → 1日6,300円
→ 1,000円買切なら 1日6個販売
→ CVR 1%なら 1日600ダウンロード
→ X バズ or ASO or SNS集客
```
**大井応用**: 月収100万 (6/21期限) ・残30日 → 1日約33,000円必要
- AIpaX 1社契約=月10-15万 = 月3-5社追加で達成
- Testall β100名月980円=98,000円
- → 残り20-30万 を AIpa Web モニター+紹介で

### 6. MVP=最小限機能で1-3日リリース
- 「完璧主義 = 現実逃避」(シンコード名言)
- 完成度45%でも出す
- Google認証だけでOK・メアド認証不要
- 競合と差別化されたUI/UX1点で勝てる
**大井応用**: Testall 9月MVPまで時間あるが、機能絞って最小版を6月中に β5名にリリース→ feedback取得

### 7. 「拡散される1ページ」を最初から設計
- LP上で1秒で理解できるデザイン
- スクショ取りやすい1画面 (映え/インスタ映え)
- これだけで X バズ100万 impression 獲得実績
**大井応用**: AIpaX/Testall/EEMUS 各事業に「1秒で理解1画面」のスクショ用デザイン用意

### 8. リファラル (口コミ拡大)
- アフィリエイト=お金 vs リファラル=クレジット
- 指数関数的拡散 (2→4→8→16...細胞分裂)
- スクショ映えポイント = 自然拡散の起点
**大井応用**: AIpaX 既存4社→紹介プログラム / Testall β顧客→紹介で10クレジット付与

### 9. ファネル分析 = バケツの穴埋め
- LP訪問→新規登録→課金 各段階で離脱率測定
- 離脱多い箇所=穴
- 例: マイクロコピー変更だけで CVR 1.3倍
**大井応用**: AIpaX 商談1回目→2回目→クロージング のファネル測定・離脱箇所特定

### 10. PMF達成までは8割ピボット覚悟
- スタートアップで1番難しいフェーズ
- 個人開発の有利点=やめやすい (チーム抱えてないから)
- 失敗=スキル蓄積・無駄じゃない
**大井応用**: Gymee寝かす判断は正解・他事業も「PMFシグナル出ない=即ピボット」のクセつける

## 🛠️ 技術スタックベストプラクティス (シンコード推奨)

### Web SaaS
| 層 | 推奨 | 補足 |
|---|---|---|
| フロント+バック | **Next.js App Router** | 大井既に採用 |
| DB | **Turso** (推奨) or Supabase | Tursoは100枠+10GB無料・MCP対応・SQLiteベース |
| 認証 | Supabase OAuth / Better Auth / Clerk | 大井は Google認証だけでOK |
| 決済 | **Stripe** (手数料3.6%) | Polarは手数料高い |
| 画像ストレージ | **Cloudflare R2** (10GB無料) | Supabaseは1GBで満杯 |
| メール | Resend | 安定 |
| インフラ | Vercel ($20/月) or **Netlify** (無料商用OK) or Cloudflare Workers | 迷ったらVercel |

### Mobile (iOS含む)
| 層 | 推奨 | 補足 |
|---|---|---|
| フロント | **Expo** (Windows対応) | Mac不要・iOSビルド可 |
| ネイティブ機能必須 | Mac + Swift / Flutter | Widget等は Expo不可 |
| 決済 | **RevenueCat** | Apple 30%手数料は不可避 (15%申請可) |
| 分析 | **PostHog** (MCP対応) | ファネル分析・離脱検知 |

### バイブコーディング
- **Cursor or Claude Code** (どちらでもOK・ツール論争に巻き込まれない)
- 「アプリのスタイル」と「中身」と「セキュリティ」が大事・ツールは手段
- 完全自動化は無理・最終レビューは人間

## 💡 クロードコード フル活用 (大井のbrain Systemに反映済+追加)

### 既に大井が実装済 (本日確認)
- 仮想組織 (部署別エージェント)
- スキルズ (ワークフロー化)
- サブエージェント (専門特化)
- バッチコマンド (並列処理)

### 大井がまだ取り入れていない (推奨)
1. **`/effort max`** (デフォルトhigh→max推奨・複雑業務用)
2. **`/bashmode`** (`!npm run dev` でターミナル分割不要)
3. **`/resume`** (会話継続)
4. **`/rewind`** (出力が微妙な時に巻き戻し)
5. **`/insights`** (クロードコード使用分析レポート)
6. **`/fewer-permission-prompts`** (過去パーミッション履歴から自動許可リスト作成)

### 9つのチップス TIPS要約
- `effort high→max` で品質UP
- `btw` (待機中の質問) でコンテキスト汚染なし
- `bash !` でターミナル統合
- `design.md` でデザイン方向統一
- `rewind` で品質悪い時の巻き戻し
- `batch` で並列処理 (refactor/test/dark mode)
- オートランモード (auto edit on) 推奨
- フューパーミッションプロンプツ で許可リスト自動化
- `insights` で使い方分析

## 🚨 シンコード名言・原則

1. **「完璧を目指すのは現実逃避」** - MVPで出す
2. **「銀の弾丸はない」** - PDCAサイクル回せる人が勝つ
3. **「成功は失敗の先・高く飛ぶにはしゃがむ」** - 沈み込み期間必須
4. **「J カーブで成果出る」** - 短期で諦めない
5. **「アプリは在庫じゃない」** - 来年使うため保管できない (機会損失思考の罠)
6. **「個人の有利点=やめやすい」** - チーム抱えてないからピボット早い
7. **「結果出るまで時間かかる・幸せは没頭中にある」** - お金より過程
8. **「専門家を巻き込めば勝てる」は半分嘘** - 専門家=証拠補強・順番は「痛み→検証→専門家→発信」
9. **「日本人は変化遅い」** - 海外で起きてる変化 (AI労働解放) を先取りすれば勝てる
10. **「外貨を稼ぐ」** - 日本市場小さい・グローバル展開でレバ効かせる
