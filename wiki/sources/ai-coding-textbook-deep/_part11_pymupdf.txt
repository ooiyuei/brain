// console.logは削除済み
// 使わない変数は削除済み

CLI とは何か
これまでの章でも、Antigravity 内のターミナルでコマンドを実行してきました。npm

一方、CLI はテキストベースで操作します。マウスでポチポチとフォルダを開いていく代わり
に、コマンド一発で目的のフォルダに移動できます。GUI が人間にとって分かりやすいのに対

CLIのメリット
CLI には大きく 3 つのメリットがあります。
まず、高速性です。GUI では、フォルダを開いて、さらに下の階層に移動して、ファイルを探し

rm -rf / の恐怖
CLI の世界で最も恐れられているコマンドがあります。それがrm -rf / です。
/ は PC の根っこのフォルダを指し、rm -rf は強制的にすべてを削除するコマンド
です。つまり、PC の全ファイルを消すコマンドということになります。
実際にこれを実行することはほとんどありませんが、AI にフォルダ削除を任せた際に、
誤ってホームディレクトリごと消してしまった...という事故は時々耳にします。
権限を与えることの恐ろしさを理解し、特に削除系のコマンドは実行前に必ず確認し
ましょう。AI は便利ですが、最終的な判断は人間が行う必要があります。

絶対パスと相対パス
ファイルやフォルダの場所を指定する方法には、絶対パスと相対パスの 2 種類があります。
絶対パスは、最初から最後まで全部書く住所のようなものです。例え
ば/Users/username/Documents/report.txt という書き方で、誰がどこから見ても場所が特
定できます。
一方、相対パスは、今いる場所を基準にした道案内です。例えば./report.txt は「今いるフ
ォルダの report.txt」を意味し、../Documents は「一つ上のフォルダの Documents」を意
味します。

CLI 作業を爆速にする 2 つのテクニック
CLI を使い始めると、「毎回同じコマンドを打つのが面倒...」「長いファイル名を全部打
つのが大変...」と感じることがあります。でも、実は簡単に効率化できる機能が用意さ
れています。
↑キーで履歴を呼び出す
矢印キー↑を押すと、過去に実行したコマンドの履歴を遡れます。同じコマンドを何
度も実行する時は、↑キーで履歴から呼び出すだけで OK です。わざわざ打ち直す必要
はありません。
例えば、開発サーバーを起動するnpm run dev というコマンドは毎回使いますが、2
回目以降は↑キーを押すだけで済みます。この小さな積み重ねが、作業効率を大きく
改善します。
Tab 補完で入力を省略
ファイル名やフォルダ名を最後まで打つ必要はありません。途中まで打って Tab キーを
押せば、自動で補完してくれます。例えばcd gem まで入力して Tab を押せば、cd
gemini-chat と補完されます。
候補が複数ある場合は、Tab を 2 回押すと一覧で表示されます。長いファイル名でも、
最初の数文字だけ打てば良いので、タイピング量を大幅に減らせます。
この 2 つの機能を使いこなすだけで、CLI 作業が圧倒的に快適になります。最初は意識
して使う必要がありますが、慣れると無意識に使えるようになり、もうマウス操作に
は戻れなくなるはずです。
