#### なぜ「様々な形態」のツールを学ぶのか（p141-142）

開発支援ツールの形態：
- グラフィカルなエディタ / コマンドライン（CLI）
- ローカル動作 / クラウド実行

それぞれの形態の強み:

| 形態 | 具体例 | 強み |
|---|---|---|
| **エディター型** | Antigravity、Cursor | コード補完＋AIエージェント機能、自分のPCで本格開発、複数ファイルプロジェクトに最適 |
| **CLIツール** | Claude Code、Codex、Gemini CLI | ターミナルから起動、テキストベース、**現時点で最も高い自律性とコーディング性能** |
| **クラウド型** | Jules、Claude Code Web | 自分のPCを使わず、ブラウザやSlackから依頼、移動中スマホからもタスク投げ可能 |

組み合わせ例:
- 普段 → Antigravityでビジュアル重視の開発
- コア開発・日々の業務自動化 → CLIツール
- 定期実行や簡単なタスク → クラウド型AIにポイポイ依頼

#### 第9章プレビュー：指示の出し方とコンテキスト管理（p143-144）

中心概念: **コンテキスト**

「『明日何を着ればいい？』と聞かれても、天気や行き先が分からなければ答えられないように、AIも必要な情報がなければ的確な回答はできません」

「コンテキストとは『最良の回答をするために、選択肢を狭める情報』のこと」

ルールファイル（AGENTS.md）に書くべきもの:
- 目的・対象・内容（**Why・Who・What**）
- 例と制約
- 参照先

「フィードバックを繰り返すことで、AIはどんどん自分好みのパートナーに育っていきます」

最後に動作確認:
- AIが作ったもののチェック方法
- チェックリストの作り方
- 「AIに専門家の視点でレビューしてもらう」

#### 第10章プレビュー：CLI開発（p144）

CLI（コマンドライン・インターフェース）= 「黒い画面」

CLIの魅力:
- マウスでポチポチ→コマンド一発で完了する**高速性**
- すべてテキスト操作→**AIに指示を出しやすい**

基本6コマンド:
```bash
cd     # ディレクトリ移動
ls     # ファイル一覧
mkdir  # ディレクトリ作成
mv     # 移動/リネーム
cp     # コピー
rm     # 削除
```

「これらさえ覚えれば、日常的な操作はほとんどこなせるようになります」

**Warp**（AI対応ターミナル）→「やりたいこと」を日本語で伝えるだけでAIが適切なコマンド生成
実用例: PDFの結合、動画の編集

**Gemini CLI** → プロジェクトのドキュメント自動生成

#### 1. 単発スクリプト：ブラウザで手軽に試す（p147-148）

定義: 「チャットツール内でちょっとした自動化が手軽にできるもの」

例: **Gemini Canvas**で「テトリスを作って」→ AIがコードを書き、その場で動かして遊べる

向いている用途:
- 環境構築なしですぐ始められる
- プログラミング初心者向け
- 簡単なゲーム作成
- データの集計・整形
- 単純な自動化スクリプト
- **一つのファイルで完結する作業**

弱点:
- 複数ファイルにまたがる開発には対応していない
- ライブラリ（外部パッケージ）の利用が制限される
- 本格的なアプリ開発には物足りない

#### 本書で使うツール（p155-157）

「なぜかGoogleがすべての形態で無料で使えるツールたちを提供してくれています」

##### エディター：すべてのローカル作業の起点（p156）

メインツールは **Antigravity**

「**エディターは、自分のPC上で行うすべての開発作業の起点**となります。コードを書く、読む、編集する、ファイルを整理するといった基本操作はすべてエディター上で行います。お気に入りのエディターを一つ持っておくことは、プログラミングを続けていく上でとても大切」

「最近はどのエディターも**VSCodeをベース**にしているため、操作感が似通っています。一つに慣れれば、他のツールへの移行もスムーズ」

##### CLIツール：現状最強のコーディング性能（p156-157）

「現時点で最も高いコーディング性能を持っているのは、Claude Code や Codex といった CLI 型のツール」

CLIの利点:
- **AIにとって扱いやすい**（すべてテキスト操作）
- 余計な画面表示などに処理を使わない → **AIの能力を最大限引き出せる**
- 自律性の高さを実感できる

##### クラウド型ツール：どこからでも仕事を依頼（p157）

独特の利点:
- 自分のPCを使わない → どこからでも仕事依頼
- 「移動中にスマホから『この機能を実装しておいて』と依頼し、後で結果を確認」
- 非同期に作業が進む → AIがほぼ自力で達成できるタスクなら細かいレビューに時間をかけずに依頼可能

> ちょっとした修正や定型的な実装を次々と任せられる便利さは、一度体験すると手放せなくなります。

**注意**: クラウド型ツールを使いこなすには **GitHub の知識が必要**

##### まとめ（p157）

> AIコーディングツールは大きく3つのカテゴリに分類できます。ブラウザで手軽に試せる単発スクリプト、本格的な開発ができるしっかり開発（エディター、CLIツール、拡張機能、クラウド型）、Webサイトを一気に構築するオールインワンです。

進めていく流れ:
1. Antigravityをメインエディターとして使いこなす
2. CLI操作を学ぶ
3. CLIツールに挑戦

### 第8章：AIコーディングツールの機能を知る（p158-160）

#### 章のイントロ（p158）

「ツールごとの細かな違いを網羅するのではなく、『どんなツールを使っても活かせる基本的な機能』に絞って解説」

「AIコーディングエージェントは、単なる『コード補完ツール』ではありません。**膨大な情報を理解し、自律的に判断し、行動してくれるパートナー**です」

紹介する6機能:

| 機能 | 説明 |
|---|---|
| ファイルのメンション | AIに「このファイルを参考にして」と情報を渡す |
| Webサーチ | AIの知識にない最新情報を取得する |
| MCP | AIに新しい能力を追加する |
| ルールファイル | AIへの「作業マニュアル」を書く |
| カスタムコマンド | よく使う指示を登録する |
| ガードレール | AIが危険な操作をしないよう制御する |

#### ファイルのメンション：必要な情報だけを渡す（p159）

「AIは渡された情報をもとに作業するため、何も情報を渡さなければ的外れな結果になることがあります」

##### メンションの基本

Antigravity含む多くのAIコーディングエージェントで使える **@ 記号**

実例コマンド:
```text
@writing-rule 通りに @07_ai_coding.md の章を校正して
```

「このように書くと、AI はwriting-rule の内容を読み込んでから作業してくれます」

#### メンションの種類と実例（p160）

第3章で作成した Gemini Chat プロジェクトを例に解説。

##### 1. ファイルメンション（@ファイル名）

「特定のファイルを参照したい時に使います」

**実例1：スタイルを変更する**

```text
@style.css の背景色をダークブルー系に変更してください
```

> AI はstyle.css ファイルだけを読み込んで、背景色を変更してくれます。ファイルを指定することで、AI は「どのファイルを編集すればいいか」を正確に把握できます。

**実例2**: 既存のコードを参考に機能を追加する（p161以降に続く）

## 💡 Tips・原則

### 1. 5章のスキルは「普遍的」原則
- AIコーディング5章のスキルは、文章執筆・資料作成・調査など、あらゆる知的作業に応用できる
- 著者自身も「文書作成や動画の台本制作にAIコーディングエージェントを育てて活用」

### 2. ツール形態の組み合わせ原則
- エディター = 普段使い、ビジュアル重視
- CLI = コア開発、業務自動化
- クラウド型 = 定期実行、移動中の細かいタスク

### 3. プロのスタイル
- CLIツール（Claude Code、Codex）を**主軸**
- エディタ（Cursor、Antigravity）を**併用**
- クラウドAIに**細かいタスク**を依頼

### 4. CLI型最強原則
- 「現時点で最も高いコーディング能力」=CLIツール
- 理由: テキスト操作 → AIが扱いやすい → 能力を最大限引き出せる

### 5. AI時代のバージョン管理必須化
- 「『AIという勝手にいじる同僚』が現れた今、バージョン管理は個人の作業でも必須」

### 6. コンテキストの定義
- **「最良の回答をするために、選択肢を狭める情報」**
- 比喩: 「明日何を着ればいい？」 → 天気・行き先がなければ答えられない

### 7. ルールファイルの3要素（Why/Who/What）
- 目的（Why）
- 対象（Who）
- 内容（What）
- ＋ 例と制約、参照先
