「temperature を 0 にすれば毎回同じ回答になる」と説明しましたが、実は完全に同一

これは、AI の計算がコンピュータ（GPU）上で高速に並列処理されているためです。
小数点を含む大量の計算を超高速で行う際、計算の順番がわずかに変わることがあり、

料金の仕組み
AI API の料金は、多くの場合従量課金制です。使った分だけ料金がかかる仕組みですね。
非エンジニアのための AI コーディングの教科書
286


===== PAGE 287 =====
料金はトークン単位で計算されます。トークンとは、AI がテキストを処理する際の最小単位で
す。日本語では「1 文字≒0.5〜0.7 トークン」くらいのイメージで大丈夫です。つまり、1000
文字のプロンプトを送れば、約 500〜700 トークン分の入力料金がかかります。
例えば、GPT-4.1 の場合の料金を見てみましょう：
入力：100 万トークンあたり 2 ドル（約 300 円）
出力：100 万トークンあたり 8 ドル（約 1,200 円）
「100 万トークン」と聞くとピンと来ないかもしれませんが、日本語で約 150 万文字と仮定する
と、400 字詰め原稿用紙約 3,750 枚分に相当します。つまり、原稿用紙 3,750 枚分のテキスト
を送っても、たった 300 円程度ということです。個人利用であれば、ほとんどの場合、月に数
百円〜数千円で収まるでしょう。
面白いのは、入力より出力の方が料金が高いという点です。これを知っておくと、「長いプロン
プトを送るより、長い回答を求める方がコストがかかる」という感覚が身につきます。
料金を節約する方法
AI API には、料金を節約するための仕組みがいくつか用意されています。本格的に使い始めた
ときのために、知っておくと便利です。
1. プロンプトキャッシュ
多くの AI サービスでは、同じプロンプトの計算結果を再利用する「キャッシュ」機能がありま
す。一度計算した内容を覚えておいて、同じ内容が来たら再計算せずに済ませる仕組みです。
特に OpenAI では、プロンプトの先頭部分が同じであれば、前回のキャッシュが自動的に使わ
れます。

2 段階目がワークフローです。ワークフローは「事前に定義された枠組みの中で AI を使って自

例えば、「特定のキーワードを含むニュース記事を毎日チェックして、その要約を AI に作らせ、

「ニュースをチェック」「AI で要約」「チャットに投稿」という、決まったステップを自動でこな

がある方はイメージしやすいかもしれません。
ワークフローで大事なのが「環境に作用する力」があることです。「環境」というのは、AI 以外
の様々な存在、つまり外部のサービスや人間、他の AI のことを指します。AI 単体でできること
は狭く、こういった様々なものを組み合わせることによって、よりインパクトの大きい自動化が

ReAct：思考と行動の連鎖
ReAct
 は Reason（思考すること）と Action（行動すること）をセットで繰り返していく
考え方です。
例えば、「最新の AI トレンドを調査してまとめて」と AI エージェントにお願いしたとします。
1. 思考：AI エージェントはまず「思考」をします。「よし、まずは『AI トレンド 2025』で
Web 検索をかけてみよう」と。
2. 行動：次にその思考に基づいて「行動」、つまり Web 検索ツールを実行します。
3. 観察：検索結果という「観察」を得て、また「思考」します。「いくつか記事が見つかった。
この記事とこの記事が信頼できそうだ。まずは内容を要約してみよう」と。

このように、「思考→行動→観察」のサイクルを、最終的な目的を達成するまで自律的にぐるぐ
ると回し続ける。これが ReAct の基本的な仕組みです。
状況に応じて自分で次の一手を考えられる、まさに「自律性」の源泉と言える考え方ですね。
現在多くの AI エージェントは、このような枠組みに則って作られています。
脳と手足
これまでの自動化の段階を振り返ると、まず「単一の返信」から「ワークフロー」になる段階
で、AI は外部ツールを操作する「手足」を手に入れました。しかし、それだけでは決まった動
きしかできません。
そこに、ReAct のような「自分で考える脳」が加わることで、AI エージェントが誕生するわけ
です。
この「脳」と「手足」の両方を持っているからこそ、人間の仕事を高度に、そして柔軟に"代
理"できる。これが AI エージェントの正体なんですね。

昨今の AI エージェントたちは、ズバリ「プロンプトと Tool Calling の連鎖」でできています。
Tool Callingとは
前章で少し触れましたが、ここでより深く理解していきましょう。
Tool Calling とは、Web 検索やファイル読み込み、メール送信など、AI の外の世界にある様々
な機能を使えるようにする仕組みです。
実際にどんな風に行われているか見てみましょう。
まず、AI に「こんなツールが使えるよ」と伝えます：
const tools = [
  {
    type: "function",
    function: {
      name: "web_search",
      description: "Webを検索して情報を取得します",
      parameters: {
        type: "object",
        properties: {
          query: {
            type: "string",
            description: "検索クエリ"
          }
        },
        required: ["query"]
      }
    }
  }
];
このツール定義には、ツールの名前、どんなツールなのかの説明、ツールにはどんな情報を渡
すのかなどが含まれています。
ここでは「Web 検索」ツールを定義していて、どんなことができるのかの説明や、渡すべき情
報（検索クエリが必要）を伝えています。

そして AI がプロンプトに応じて、このツールを「使う」と判断します。「『AI トレンド 2025』
という検索クエリで Web 検索ツールを使うべきだ！」と提案してくれるのです。
ここで重要なのは、前章でも触れた通り、AI はテキストを生成することしかできないという点
です。AI が「Web 検索を使うべき」と言っても、実際に検索を実行するのは別のプログラムの
仕事です。
例として次のコードのように、実際の行動はプログラムが AI の回答をもとに行います。
// AIがツールを呼び出すと決めた場合
if (response.toolCalls) {
  for (const toolCall of response.toolCalls) {
    if (toolCall.function.name === "web_search") {
      const args = JSON.parse(toolCall.function.arguments);
      // 実際にWeb検索を実行する
      const searchResult = await executeWebSearch(args.query);
      // 結果をAIに返す
      messages.push({
        role: "tool",
        content: JSON.stringify(searchResult)
      });
    }
  }
}
このようにお膳立てすることで、AI は環境に作用する「手足」を手に入れることができるので
す。
プロンプトもツールになる
ここからが面白いところです。この「ツール」には、「他のプロンプト」も含めることができま
す。
例えば、ディープリサーチのような AI エージェントを作る時、最初に計画を立てた後には、
Web 検索をする、内容が十分かを判断するプロンプト、結果をまとめ上げるプロンプト、のよ
うに連鎖させていく必要があります。
プロンプトも「ツール」として選択できるようにすることで、AI がどんどん次のアクションを
自律的に進められるわけですね。
