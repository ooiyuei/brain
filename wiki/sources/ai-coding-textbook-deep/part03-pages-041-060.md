### p45 — コラム「TypeScript って何？」

- **TypeScript = JavaScript をより厳格に書けるようにしたプログラミング言語**
- 「データの種類を明示する機能」を追加した言語
- 例：「この変数には数値だけを入れる」「この変数には文字列だけを入れる」

#### JavaScript の例（型エラーが実行時まで気づけない）

```javascript
let age = 25;   // 数値を入れた
age = "25歳";   // 後から文字列を入れることもできてしまう
```

#### TypeScript の例（実行前にエラーが出る）

```typescript
let age: number = 25;
age = "二十五歳";  // エラー！数値型の変数に文字列は入れられません
```

- 「昨今の現場では、TypeScript を使うことが基本になっています」（型がある方が保守性が高い）
- ただし「取り急ぎの学習では不要というか、敷居を高く感じてしまうかもしれません」
- 本書では JavaScript から始める。「JavaScript をより厳格に書けるものなんだな」と理解しておけばOK

## 🚀 大井AI改善適用

### Claude Code 運用への転用

#### 1. Plan モード = Antigravity の "Planning" モード

- 大井の Claude Code は既に Plan モードを活用
- Antigravity の Planning/Fast スイッチと同じ思想
- **複雑タスク → Plan、単純タスク → 即実装**

#### 2. プロンプト構成テンプレート（著者の Gemini Chat プロンプトから抽出）

dispatch.ps1 で OpenClaw に投げるとき、以下の構造を採用：

```markdown
# 全体像
[1行で何を作るか]

## デザイン/出力フォーマット
- テーマ/スタイル
- 配色/構造
- 特徴

## 技術スタック/制約
- フレームワーク
- API
- モデル

## 実装のガイドライン
1. **使うもの**: ○○
2. **使わないもの**: ○○
3. **初期化方法**: コード例

## 作業の注意点
- 何を後でやるか
- バージョン指定
- 設定ファイル

[要件まとめ]
[公式ドキュメント URL]
```

#### 3. 「ゆるい→詳細」のグラデーション活用

- 大井が雑に「○○作って」 → Claude が詳細プロンプトに展開してから OpenClaw に dispatch
- 著者の「ゆるくても動く」例は、Claude が中間翻訳できる証拠

#### 4. Accept/Reject 文化を秘書ワークフローに

- OpenClaw の成果物に対して大井が逐一見る必要はない
- Claude が「Accept all」相当の判断を代行
- 不満があるときだけ「再 dispatch（指示改善）」のループ

#### 5. .env / .gitignore の自動化フック

- 新規プロジェクト作成時に **必ず** `.env.example` と `.gitignore` を生成するフック
- → 大井のセキュリティリスクを自動排除

#### 6. モデル切り替え戦略

著者の `gemini-3-flash-preview` → `gemini-2.5-flash` パターンと同じく：
- Claude Code でも Opus → Sonnet → Haiku のフォールバック戦略
- リミット時の自動切り替えロジックを scripts/ に組み込む

#### 7. トラブルシュートを「AI に質問」で完結させる

- 大井がエラーで詰まったら → ブラウザ開発者ツールのスクリーンショット → Claude に貼る
- これを Claude Preview MCP で自動化できる

#### 8. AIpaX school の差別化材料に

- 著者本は Antigravity（Google）特化
- 大井版は **Claude Code + Antigravity の使い分け** を教える
- 「Google エコシステム派 vs Anthropic エコシステム派」の比較表が新しい教材になる

#### 9. 「コピペで動く」体験を最重要視

- 公式リポジトリにプロンプト集を置く著者の戦略は秀逸
- 大井版でも `~/business/brain/wiki/教材/prompts/` を作って公開
- AIpaX school 受講生がそのまま使えるテンプレ集

#### 10. 「危険な作業は行わないはず」リスク認識

- 著者は「Web アクセスなどもせず...とりあえず許可いただいて大丈夫」と書いた
- 大井の Claude Code 運用では：
  - フォルダー閉じた範囲：自動許可OK
  - Web アクセスを伴う：要確認
  - 金銭/権限変更：必ず人間判断
- → settings.json の hook ルールに反映
