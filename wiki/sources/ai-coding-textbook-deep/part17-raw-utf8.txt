プログラミング

ソースコード

プログラミング言語で書かれた、コンピュータへの命令文のことです。「コード」と略されるこ
とも多いです。
人間が読み書きできる形式で書かれており、これをコンピュータが実行できる形に変換（解釈）
することで、プログラムが動作します。本書では主に JavaScript というプログラミング言語で
ソースコードを書いています。

開発環境

IDE（統合開発環境）

Integrated Development Environment の略で、コードを書くためのエディタ、実行環境、
デバッグツールなどがひとまとめになったソフトウェアです。
本書で使用する Cursor や VS Code も、厳密にはテキストエディタですが、拡張機能によって

コンピュータに文字（コマンド）で命令を出すためのアプリケーションです。「コマンドライ
ン」「コンソール」とも呼ばれます。
マウスでアイコンをクリックする代わりに、キーボードで命令を打ち込んで操作します。最初は
難しく感じるかもしれませんが、慣れると効率的に作業できるようになります。

Node.js

JavaScript をブラウザの外（パソコン上）で動かすための実行環境です。
元々 JavaScript は Web ブラウザの中でしか動きませんでしたが、Node.js の登場によって、パ
ソコン上でファイルを操作したり、サーバーを立てたりといった処理も JavaScript で書けるよ

プロンプト

AI に対して送る指示や質問のテキストのことです。「入力文」とも言えます。

コンテキスト

AI が回答を生成する際に参照する「文脈情報」のことです。

ガードレール

AI エージェントが危険な操作を行わないように設ける「安全柵」のことです。

モデル

機械学習によって作られた「学習済みの頭脳」のことです。AI サービスでは、様々なモデルが

AI の「性格」や「役割」を設定するための特別なプロンプトです。ユーザーとの会話の前に設

例えば「あなたは親切なアシスタントです」「敬語で話してください」といった指示を与えるこ

データ分析など正確性が重要な場面では低く、ブレインストーミングなど創造性が必要な場面

AI からの出力を、決められた形式（JSON 形式など）で受け取る機能です。

トークン

AI がテキストを処理する時の最小単位です。おおよそ「単語」や「文字のかたまり」と考えて
ください。
日本語の場合、1 文字が 1 トークン以上になることが多いです。API 利用料は通常、トークン数
に基づいて計算されます。

Tool Calling

AI が外部のツール（Web 検索、ファイル操作など）を「使いたい」と宣言する仕組みです。
重要なのは、AI 自身がツールを実行するわけではないということです。AI は「このツールを使
いたい」と提案するだけで、実際の実行は別のプログラムが行います。

プロンプトキャッシュ

同じプロンプト（システムプロンプトなど）を繰り返し使う際に、その処理結果を一時的に保
存しておく仕組みです。
一度処理した内容をキャッシュしておくことで、2 回目以降のリクエストが高速になり、API の
利用コストも削減できます。特に長いシステムプロンプトを使う場合に効果的です。

Batch API

複数のリクエストをまとめて送信し、非同期で処理する方式の API です。
リアルタイムで応答が必要ない場合（例：大量のデータを一括処理したい場合）に利用します。
通常の API より割安で、大量処理に向いています。結果は後からまとめて取得します。

AI エージェント

AIエージェント

人間に代わって自律的にタスクを遂行する AI システムです。単なるチャットボットとは異なり、
自分で考えて、必要な行動を選択し、目標達成まで進んでいきます。

「脳」（思考・判断する力）と「手足」（環境に作用する力）の両方を持っているのが特徴です。

 ReAct

Reason（思考）と Action（行動）を組み合わせた、AI エージェントの設計思想です。「リアク
ト」と読みます。
「思考→行動→観察」のサイクルを繰り返すことで、複雑なタスクを自律的に進めていきます。
 1. 思考（Reason）: 「次は何をすべきか」を考える
 2. 行動（Action）: ツールを実行する
 3. 観察（ ） Observation : 結果を確認する

ワークフロー

事前に定義された手順に従って、処理を自動実行する仕組みです。
AI エージェントとの違いは、「自分で判断する力」がない点です。決まった手順を確実に実行す
るのには適していますが、想定外の状況には対応できません。

コンテキストエンジニアリング

AI に渡す「文脈情報（コンテキスト）」を設計・管理する技術です。
AI エージェントが複数のステップを踏む時、各ステップで適切な文脈を持っていないと、精度
が落ちてしまいます。どの情報を、どのタイミングで、どう渡すかを設計することが重要です。
