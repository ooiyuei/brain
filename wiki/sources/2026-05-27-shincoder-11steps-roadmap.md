# クロードコードスタートアップの教科書 — 11ステップ・ロードマップ

> **シンコーダーがサブスクボックス (売上40万/月) を作って得た知見をもとに、Claude Code 時代の「1人スタートアップ」の再現性ある手順を11ステップに体系化した動画。**
> 大井の主力5事業 (AIpaX / AIpa Web / AIpaX school / EEMUS / Testall) すべてに適用可能。

---

## 動画の核となる主張

1. **Claude Code 時代は誰でもアプリが作れる** → 差別化はアイデア力・共感能力・実行速度
2. **「完璧を目指す」のは現実逃避** — リリースしないことのリスクの方が大きい
3. **市場検証は広告 1万円でCPA測定が再現性高い** — SNS フォロワーゼロからでも可能
4. **PMF判定の3シグナル** (定量/訂正/経済) を明確に持って、ピボット判断を素早く
5. **作って終わりにしない** — 売上に変えるロードマップが必要

---

## 11ステップの全体像

| # | ステップ | 期間目安 | 判定基準 |
|---|---|---|---|
| 1 | **アイデアの発掘** | - | バズりやすい/共感されやすい仮説か |
| 2 | **要件定義 + LP作成** | 30分 | ChatGPT 5.5 (orClaude) で要件壁打ち → Claude Design でLP一気生成 |
| 3 | **広告で需要検証** | 1日 | X広告+メタ広告 各1万円、Form Spree でウェイトリスト集計 |
| 4 | **CPA計算で判定** | 即時 | 100-300円 = 確実需要、400-500円 = 需要可能性、それ以上 = ピボット |
| 5 | **Claude Code で開発スタート** | 数時間〜2週間 | MVP優先、課金機能必須 (バズった時の取りこぼし防止) |
| 6 | **SNS運用** (開発と並行) | 開発期間中 | 関連アカウントいいね・コメント、ビルドインパブリック発信 |
| 7 | **MVP ローンチ** | 1日 | 価値発揮できる最小機能 + 課金 |
| 8 | **分析 (課金されたか)** | リリース後 200DL地点 | CVR 1% = 価値あり、CVR 0% = ピボット検討 |
| 9 | **アップデート** | 数週間ループ | PostHog でファネル分析、UI/UX改善+機能追加/削除 |
| 10 | **PMF判定** | 1-3ヶ月後 | 3シグナル (継続率15%+ / なくなったら残念40%+ / LTV ≧ CAC×3) |
| 11 | **スケール** | PMF後 | マーケ強化 / チーム化 / バイアウト or 連続起業 |

## ステップ別 詳細

### 1. アイデアの発掘
- **競合がいる = 需要がある** (差別化は UI/UX で可能)
- 海外バズアプリ→日本ローカライズも有効
- SNS タイムラインを「なぜこれがバズったか?」を考えながら見る癖
- AI には代行しづらい (人間の経験・共感が必要)

### 2. 要件定義 + LP作成
- ChatGPT 5.5 (推奨) or Claude 4.7 と壁打ち
- アイデアを画像化 → GPT image 2.0 で UI イメージ生成
- 要件確定 → **Claude Design** で LP一気生成 (3分で完成)
- LPには **ウェイトリスト (メアド収集) を必ず配置** ← Form Spree 推奨
- LP公開: Netlify Drop (簡単デプロイ)

### 3. 広告で需要検証
- X広告 1万円 ≈ 30万インプレッション (シンコーダー実績)
- メタ広告 (Facebook/Instagram) も並行
- 計 2万円で市場の手応えを掴む
- フォロワーゼロからでも検証可能 = 再現性高い

### 4. CPA計算で判定
- **CPA = 広告費 ÷ 獲得顧客 (ウェイトリスト登録数)**
- CPA 100-300円 = **確実に需要あり** → 開発GO
- CPA 400-500円 = 需要ある可能性 (判断は人による)
- CPA 1000円超 = **ピボット推奨** (アイデア変えて再検証)
- 大井のサブスクボックス: 1万円→62件 = CPA 約161円 (確実需要)

### 5. Claude Code で開発
- ツール選択 (Claude Code / Codex / Cursor) は手段、本質ではない
- MVP数時間〜数日で作成 (認証+DB+課金あり)
- 大規模なSNS等は 1-2ヶ月かかる
- セキュリティ・ハーネス設計はクロードコードアカデミー参照

### 6. SNS運用 (開発と並行)
- 新アカウントゼロから運用OK (シンコーダー実例)
- 関連アカウントへの「いいね回り」「コメント」
- ビルドインパブリック (開発過程を毎日発信)
- フォロー牽引性のため「フォローはしない」戦略もアリ

### 7. MVP ローンチ
- **「完璧」=「現実逃避」** ← 重要原則
- 価値発揮できる最小機能 + **課金機能必須** (バズった時の取りこぼし防止)
- 出してから現実を見る方が早い

### 8. 分析 (課金されたか)
- 最初の指標は単純 = 「課金されたか」だけでOK
- 200DL中 0課金 = **アプリ価値発揮できてない** → ピボット検討
- 200DL中 1課金 = 一人よがりじゃない → アップデート継続
- CVR (Conversion Rate) = 課金者 ÷ DL × 100
- CVR 1% = 普通、CVR 2-5% = 優秀

### 9. アップデート (分析と並行ループ)
- **PostHog** でファネル分析 (オンボーディング離脱・課金前離脱の特定)
- MCP サーバー対応あり (PostHog)
- 2-3週間ループしても改善ない = ピボット検討
- UI/UX 微細修正 + 機能追加/削除を泥臭く繰り返す

### 10. PMF達成シグナル

#### 定量シグナル
- 継続率 **15%+ / 30日**
- CVR **3%+**
- オーガニック成長 (広告止めても増える)

#### 訂正シグナル (定性)
- **Sean Ellis Test**: 「なくなったら残念」40%以上
- 紹介・口コミの自然発生

#### 経済シグナル
- **LTV ≧ CAC × 3** (顧客生涯価値 ≧ 顧客獲得コスト × 3)
- LTV計算時は **粗利ベース** (売上 - AI料金 - インフラ料金)

### 11. スケール / バイアウト / 連続起業
- PMF達成 = 「乾いた木材」状態、燃料投入で広範拡散
- マーケティング強化 (広告増額 / インフルエンサー / ショート動画)
- 月500万-1000万超 = チーム化検討
- **バイアウト** = 連続起業者の選択肢 (シンコーダーも検討中)
- **シリアル個人開発者** = 大井のタイプ (0→1, 1→1 が好き)
