# AIpa Web LP ヒーローセクション 3案

[[entities/aipa-web]] のLP、ヒーローセクション（ファーストビュー）の構成案3つ。
森岡式「狂人憑依（本気で採用したい中小企業オーナー）」の心理に刺さるよう、痛みと解決の構造を最適化した。

---

## 案A: 採用に困る中小企業オーナー憑依 (痛み→解決)

**コンセプト**: 「求人票を出す前に、応募者の受け皿が壊れている」事実を突きつけ、AIがその穴を埋めることを提示。
**ターゲット**: 現場崩壊寸前、人手不足で焦っている建設・製造・物流のオーナー。

| 要素 | 内容 |
| :--- | :--- |
| **キャッチコピー** | **「求人票を出す前に、応募者の受け皿を整えませんか？」** |
| **サブコピー** | 建設・製造・物流の現場崩壊を防ぐ。<br>フォーム入力だけで、最短7日で公開。原稿から更新まで一括代行。 |
| **ヒーロー画像** | 手書きのメモや、散らかった求人票、そして「採用できない」焦る社長の顔。その向こうに、整然とした美しい採用LPが浮かぶビフォーアフター。 |
| **CTA文言** | **無料で採用サイトの現状診断を受ける** |

**意図**:
「求人票（出口）」ばかりに目が行きがちだが、[[entities/aipa-web]] の本質は「入口（LP）」の質にある。
中小企業オーナーは「なぜ応募が来ないのか」で悩んでいる。
「AIpa Web」は単なるWeb制作会社ではなく、**「社外Web担当者」**として、その悩みを解決する存在であることを示す。

---

## 案B: AI×LPの新時代訴求 (テクノロジー先進性)

**コンセプト**: 「AIで速く・安く」ではなく、「AIが裏側で全部やってくれる」安心感と効率性を強調。
**ターゲット**: DX推進に意欲はあるが、リソースがない中堅企業の採用担当者。

| 要素 | 内容 |
| :--- | :--- |
| **キャッチコピー** | **「AIpa Webは、Web制作の面倒を『フォームとAIで消す』サービス。」** |
| **サブコピー** | 打ち合わせだらけの受託から解放される。<br>あなたの回答が、そのまま採用サイトの原稿になる。 |
| **ヒーロー画像** | 複雑なコードやデザインツールではなく、シンプルなGoogleフォームへの入力画面。その横で、AIが自動的にページを構築していくアニメーション。 |
| **CTA文言** | **フォームで試す（無料）** |

**意図**:
「AIで作る」は安っぽく聞こえる。
「AIが裏側で動く」ことで、**「人間は入力するだけ」**という圧倒的な楽さを訴求する。
[[entities/aipa-web]] のDeep Researchで得た「STUDIOはAI機能+Figma取込を備える」という警報に対し、**「AIで速いだけ」ではなく「運用代行まで含む」**差別化を図る。

## 案C: モニター3社限定 緊急性訴求

**コンセプト**: 「今だけ」「3社限定」という希少性と、通常価格とのギャップで行動を促す。
**ターゲット**: 補助金（IT導入補助金等）を活用したい、コスト意識の高い企業。

| 要素 | 内容 |
| :--- | :--- |
| **キャッチコピー** | **「通常30万円相当の採用サイトが、初期15万円で。」** |
| **サブコピー** | 事例掲載条件付きモニター3社募集中。<br>最短7日公開・月額更新込み。IT導入補助金対象。 |
| **ヒーロー画像** | 「モニター3社限定」のバッジが目立つ、シンプルで信頼感のある企業ロゴ風のレイアウト。数字（15万/7日）を大きく強調。 |
| **CTA文言** | **モニター応募（残り○枠）** |

**意図**:
[[entities/aipa-web]] の商品設計にある「モニター価格（初期15万円）」を前面に出す。
3万円などの安売りは修正地獄を招くため、**「15万円」という正当な価格感**を保ちつつ、通常価格との比較で「お得感」を演出する。
「IT導入補助金対象」を明記し、実質負担額を下げていることを示唆する。

---

## 推奨と次の一手

**推奨: 案A をベースに、案C の価格感を組み合わせた構成**

1.  **見出し**: 「求人票を出す前に、応募者の受け皿を整えませんか？」
2.  **副見出し**: 建設・製造・物流の現場崩壊を防ぐ。フォーム入力だけで、最短7日で公開。
3.  **価格提示**: 通常30万円相当 → **モニター3社限定 初期15万円**
4.  **CTA**: 無料で採用サイトの現状診断を受ける

**理由**:
[[entities/aipa-web]] のターゲットである「地方の30〜300名、建設・製造・運輸・サービス業」は、**「痛み（人手不足）」**に対して最も敏感。
テクノロジーの先进性（案B）よりも、**「どうやって人を集めるか」という結果**に直結する訴求が有効。
また、[[entities/aipa-web]] のDeep Researchで示された「求人代理店・士業・採用SaaS経由を主軸」というGTM戦略においても、**「採用サイトの質」**を問う案Aが、代理店からの紹介にも強い。

**[要確認]**
- モニター3社の具体的な業種選定基準（例：建設業1社、物流1社、製造1社など）
- 「最短7日公開」の裏付けとなる、現在のフォーム設計とAI処理の速度検証結果
