## 結論

kasunote の技術資産を 80% 流用し、配達・訪問業者向けのカスハラ記録 SaaS を 1 週間で LP 公開、1 ヶ月で MVP リリースする。コア差別化は「玄関先 1 タップ録音 + GPS 自動付与 + 法的証拠 PDF 即時出力」。2026 年 10 月のカスハラ対策義務化 直前に CPA 測定を完了させ、Phase 2 以降の本格展開に備える。

---

## プロダクトの目的

tobira-log は、配達・引越し・訪問修理など「玄関先で仕事をする」業種向けに、カスハラ被害を 1 タップで記録し、法的証拠として使える PDF を即時出力する SaaS である。記録の習慣化によりドライバー・スタッフの離職を防ぎ、示談・委託元への報告コストを下げる。月額 980 円 / 人、管理者ダッシュボード付き。

## ユーザーストーリー

- ドライバーとして、クレーマーに絡まれた瞬間に画面を開かずに録音を開始したい。なぜなら両手がふさがっているから。
- ドライバーとして、訪問後 30 秒以内にカスハラ内容を分類タグで記録したい。なぜなら次の配達先に急いでいるから。
- 社長として、ドライバーが記録を残した瞬間にスマホ通知を受けたい。なぜなら深刻なケースをすぐ把握したいから。
- 社長として、委託元（ヤマト・佐川）に報告する際、PDF 1 枚で済ませたい。なぜなら報告書を作る時間がないから。
- 社長として、月次でカスハラ件数が多い住所や時間帯を把握したい。なぜなら事前にドライバーに注意喚起できるから。

## 画面一覧

### 1. LP（マーケティングページ）

- 目的: 訪問業者の管理者・ドライバーに価値訴求し、無料トライアル or CTA を踏ませる
- 主な表示情報: ヘッドコピー・3 つの解決課題・料金表・FAQ
- ユーザー操作: 「14 日間無料で試す」ボタンタップ
- 遷移先: サインアップ
- エラー時: なし

### 2. サインアップ / サインイン

- 目的: メールアドレス + パスワードによる Supabase Auth 認証
- 主な表示情報: メール入力・パスワード入力・利用規約同意チェック
- ユーザー操作: 登録 / ログイン
- 遷移先: オンボーディング（初回）/ ホーム（2 回目以降）
- エラー時: 「メールアドレスが既に使われています」「パスワードが短すぎます」をインライン表示

### 3. オンボーディング

- 目的: 役割（管理者 / ドライバー）と所属チームを設定する
- 主な表示情報: 役割選択（2 択）→ チーム名入力（管理者）or 招待コード入力（ドライバー）
- ユーザー操作: 役割選択 → 次へ → チーム設定 → 完了
- 遷移先: ホーム
- エラー時: 「招待コードが無効です」をインライン表示

### 4. ホーム（ドライバー）

- 目的: 玄関先で即座に録音を開始できる
- 主な表示情報: 大きな「記録開始」ボタン（画面 1/3 以上）・本日の記録件数・最近 3 件のサマリー
- ユーザー操作: 記録開始ボタンタップ → 録音中表示 → 停止 → タグ選択 → メモ → 保存
- 遷移先: 記録保存後は記録一覧に自動遷移
- エラー時: マイク権限未許可の場合「設定からマイクを許可してください」とダイアログ表示。GPS 未取得の場合「位置情報なし」として保存継続（保存はブロックしない）

### 5. 記録作成フロー（ドライバー）

録音画面 → タグ選択画面 → メモ入力画面 の 3 ステップ。各ステップは戻れる。

- 録音画面: 経過時間表示・停止ボタン。最大録音時間 30 分（それ以上は自動停止）
- タグ選択画面: 怒鳴り / 不当要求 / 暴言 / 長時間拘束 / その他 の 5 タグ（複数可）。タグ未選択でも保存可
- メモ画面: テキストエリア（最大 500 文字）・スキップ可能
- エラー時: 録音ファイル保存失敗の場合「保存に失敗しました。再度お試しください」表示。リトライボタン付き

### 6. 記録一覧（ドライバー）

- 目的: 自分の記録を時系列で確認・PDF 出力する
- 主な表示情報: 日付・タグ・録音時間・GPS 住所（逆ジオコード）。1 件あたり 2 行
- ユーザー操作: 一覧タップ → 詳細 / PDF 出力ボタン
- 遷移先: 記録詳細画面
- エラー時: データ取得失敗時は「読み込みに失敗しました」と再読み込みボタン

### 7. 記録詳細（ドライバー）

- 目的: 1 件の記録を確認し PDF を出力する
- 主な表示情報: タイムスタンプ・GPS 座標・タグ・メモ・音声プレーヤー・PDF 出力ボタン
- ユーザー操作: 音声再生 / PDF 出力 / 削除（確認ダイアログあり）
- 遷移先: PDF 出力 → ブラウザ PDF ビューワー（新タブ）
- エラー時: 音声ファイル読み込み失敗は「音声ファイルが見つかりません」表示。PDF 生成失敗は「PDF の作成に失敗しました」表示

### 8. 管理者ダッシュボード

- 目的: チーム全体のカスハラ記録をリアルタイムで把握する
- 主な表示情報: ドライバー別件数（本日 / 今月）・最新記録タイムライン・未確認件数バッジ
- ユーザー操作: ドライバー名タップ → そのドライバーの記録一覧 / 記録タップ → 詳細
- 遷移先: ドライバー別記録一覧 / 記録詳細
- エラー時: Supabase Realtime 切断時は「リアルタイム更新が停止しています」バナー表示。手動更新ボタンを提供

### 9. チーム管理（管理者）

- 目的: ドライバーの招待・無効化・プラン確認
- 主な表示情報: ドライバーリスト（名前・メール・ステータス）・招待リンク発行ボタン・支払い状況
- ユーザー操作: 招待リンクコピー / ドライバー削除（確認ダイアログ） / Stripe ポータル遷移
- 遷移先: Stripe Customer Portal（外部）
- エラー時: 招待リンク発行失敗時は「リンクの発行に失敗しました」インライン表示

## 機能要件

### 録音・保存
- 最大録音時間: 30 分。30 分到達で自動停止し「録音が自動保存されました」トースト表示
- 音声ファイル形式: WebM (Opus)。Supabase Storage に保存
- GPS: `navigator.geolocation.getCurrentPosition` で取得。精度 50m 以内を目標。取得失敗でも録音保存はブロックしない
- タイムスタンプ: UTC で DB 保存。表示は JST

### PDF 出力
- 出力形式: A4 縦。出力項目は以下を必ず含む
  - タイトル「カスタマーハラスメント記録票」
  - 会社名・ドライバー名
  - 記録日時（JST: YYYY年MM月DD日 HH:MM:SS）
  - 住所（GPS 逆ジオコード。住所不明の場合は緯度経度を表示）
  - 録音時間（XX 分 XX 秒）
  - カスハラ分類タグ
  - テキストメモ
  - 「音声ファイル添付あり」または「音声ファイル添付なし」
  - フッター: tobira-log システム生成日時（改ざん防止の表示）
- PDF 生成は Next.js API Route で `@react-pdf/renderer` を使用

### リアルタイムアラート
- ドライバーが保存完了した直後、Supabase Realtime で管理者クライアントに INSERT イベントを送信
- 管理者が受信した場合: ブラウザ Push Notification + ダッシュボードのバッジ数更新
- 管理者がオフラインの場合: 次回ログイン時にダッシュボードの未確認バッジに反映

### 課金
- Stripe Billing: ドライバー 1 席 = 980 円/月
- 管理者は 1 席分として課金（ドライバー兼管理者の場合も 1 席）
- 14 日間無料トライアル（Stripe Trial）
- 席数の増減は Stripe ポータルで管理者自身が操作

## 非機能要件

### パフォーマンス
- 「記録開始」ボタン押下から録音開始まで: 500ms 以内
- 記録一覧の初期表示: 2 秒以内（LTE 環境）
- PDF 生成: 5 秒以内

### セキュリティ・プライバシー
- 音声ファイルは Supabase Storage の署名付き URL（有効期限 1 時間）でのみアクセス可
- RLS（Row Level Security）: ドライバーは自分の記録のみ参照可。管理者は所属チームの記録のみ参照可
- 個人情報（GPS・録音）は日本国内リージョン（Supabase ap-northeast-1）に保存
- HTTPS 必須。Vercel 標準 TLS

### 法的要件
- プライバシーポリシー: 音声・位置情報の取得目的・保存期間・削除方法を明記（MVP リリース前に法務確認）
- 録音告知: 録音開始時「この通話・会話はカスハラ記録のため保存されます」文言をアプリ内に表示し、法的リスクを低減
- 保存期間: デフォルト 3 年。管理者が任意削除可

## 状態遷移

### 録音フロー
```
待機 → [記録開始タップ] → 録音中 → [停止タップ or 30分経過] 
     → タグ選択 → [保存タップ] → 保存完了 → 記録一覧
                → [キャンセル] → 待機（録音データ破棄・確認ダイアログあり）
```

### 記録のライフサイクル
```
作成済（未確認）→ [管理者が閲覧] → 確認済 → [管理者が削除] → 削除済
作成済 → [ドライバー or 管理者が削除] → 削除済
```

### 招待フロー
```
管理者: 招待リンク発行 → リンクをドライバーに共有
ドライバー: リンクを開く → サインアップ → チームに自動参加 → オンボーディング完了
```

## API または内部処理の考え方

### POST /api/incidents

- リクエスト: FormData（audio Blob + JSON メタデータ）
- 処理順序:
  1. Supabase Auth でユーザー検証
  2. 音声 Blob を Supabase Storage に `{team_id}/{driver_id}/{uuid}.webm` でアップロード
  3. incidents テーブルに INSERT（audio_path に Storage パスを格納）
  4. incident_tags に INSERT（タグが 1 件以上の場合）
  5. Supabase Realtime channel `team:{team_id}` に NOTIFY
- レスポンス: `{ success: true, incident_id: string }`
- エラー: Storage 保存失敗時は DB INSERT もロールバック（トランザクション）

### GET /api/incidents/[id]/pdf

- リクエスト: incident_id（パスパラメーター）
- 処理: incidents + incident_tags + users を JOIN して取得 → `@react-pdf/renderer` で PDF Blob 生成
- レスポンス: `Content-Type: application/pdf` のストリーム
- エラー: 記録が存在しない or 権限なしは 404

### Realtime アラート

- チャンネル名: `team:{team_id}:incidents`
- イベント: `INSERT` on incidents table（Supabase Realtime broadcast）
- 管理者クライアントはダッシュボード表示中にこのチャンネルを subscribe
- Push Notification は Web Push API（VAPID）で別途送信。API Route `/api/push/notify` がサーバーサイドで実行

## テスト観点

### ドライバー機能
- マイク権限未許可時に適切なエラーダイアログが表示されること
- 録音開始から 500ms 以内に録音中表示に切り替わること
- GPS 取得失敗時でも録音・保存が完了すること（GPS なしとして保存）
- 30 分録音で自動停止しデータが失われないこと
- タグ未選択・メモ未入力でも保存できること
- 記録削除後に記録一覧から消えること
- PDF が必須項目（日時・GPS・タグ・録音時間）をすべて含むこと

### 管理者機能
- 別ドライバーの記録が管理者に見えること
- 別チームのドライバーの記録が見えないこと（RLS 確認）
- ドライバーが記録保存後 10 秒以内に管理者ダッシュボードの未確認数が増えること
- 招待リンクから登録したドライバーが正しいチームに紐づくこと

### 課金
- 14 日間トライアル終了後に録音・保存がブロックされること
- 席数増加時に Stripe が正しく課金されること

### セキュリティ
- ドライバーが他チームの音声 URL に直接アクセスしても 403 が返ること
- 署名付き URL の有効期限（1 時間）切れ後に 401 が返ること

## リスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| WebM 音声が一部 Android 機種で再生不能 | 高 | MP4 (AAC) へのサーバーサイド変換（Phase 2 対応。MVP は WebM 限定と明記） |
| 録音の法的有効性に疑義（相手の同意なし） | 高 | LP・アプリ内に「本人記録用途・証拠として使用する際は弁護士に確認を」免責表示を必須化 |
| Supabase Storage 容量コスト超過（音声大量保存） | 中 | 無料トライアルは音声保存 1GB 上限を設ける。有料は 10GB。超過分は別途課金（Phase 2） |
| kasunote との競合（同一ターゲットへのマーケ） | 低 | ターゲット（訪問業者 vs 店舗）が明確に分離。クロスセル前提で設計 |
| カスハラ義務化の施行延期 | 中 | LP を「義務化前から備える企業が増えている」訴求に変更できるよう文言を汎用化 |

## 完了条件

以下をすべて満たした時点で MVP 完了とする。

1. ドライバーが iPhone / Android Chrome でホームを開き、1 タップで録音を開始できる
2. 録音停止後にタグ選択・メモ入力を経て保存ができ、GPS 住所が記録詳細に表示される
3. 記録詳細から PDF を出力でき、タイムスタンプ・GPS 住所・タグ・録音時間が全て印刷される
4. 管理者ダッシュボードでドライバーの記録がリアルタイムに表示される
5. Stripe トライアル登録から課金開始まで、管理者が画面内で完結できる
6. 別チームのユーザーが自チームの音声ファイルに直接 URL でアクセスしても 403 が返る
7. LP が Vercel に公開されており、CTA ボタンが機能する
8. プライバシーポリシーが LP に掲載されている
