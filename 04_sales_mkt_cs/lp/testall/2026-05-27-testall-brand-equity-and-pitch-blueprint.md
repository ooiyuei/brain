## §1. 絶対に買う1人 (解像度最大化ペルソナ)

### 「鈴木蓮太郎」(仮名・17歳)

**プロファイル**
- 居住: 静岡県浜松市北区 (政令指定都市の周縁部・車社会)
- 学校: 県立浜松北高校2年 (偏差値63前後・進学校だが最難関ではない)
- 部活: 水泳部 (6月末まで週5日・夕方18時まで練習)
- 家族: 父親(製造業・年収580万)・母親(パート)・妹(中2)
- スマホ: iPhone 15 / 1日5時間以上 / X・YouTube・TikTokメイン
- 成績: 英数が偏差値55-60 / 理社は60-65 / 国語だけ50台で伸び悩み
- 志望校: 名古屋大学工学部 (現状偏差値差は10pt・ギャップ大)
- 塾: 通っていない (部活後19時帰宅のため物理的に無理)
- 参考書: Amazonで3冊買ったが積読
- 年間受験費用: 親が出せるのは最大20万円

**今夜 24時、寝る前に最初に何が頭にあるか**

「今日の模試、数学3問も分からなかった。また同じとこで詰まってる。明日なんの参考書やればいいんだろう。河合塾の友達は授業でやってるらしいけど俺には何もない。塾行けない分、なんか差つきそうで、でも具体的に何すればいいか全然わかんないんだよな。参考書開いても最初から全部やれるわけじゃないし。あーでもまず明日の古文の予習しないと。そっちも詰んでるし。部活もあるし。今日も何もできなかった。結局また寝る。」

眠れない理由は「意思の弱さ」ではない。**何をやれば正しいか分からないから動けないのだ。**

**蓮太郎がTestallを絶対に使うトリガー**

水泳部の先輩 (現在高3・同じ名大工志望) が「模試返ってきたらTestallに写真撮って入れるだけで今日何やるか教えてくれる」とXで言及。蓮太郎がポストをタップ → アプリDL → 最初の模試結果を撮影入力 → AI診断が「英語長文の時制ミスが8割・まず時制のルール1つだけ今日やれ」と出てくる。所要45分で終わる。「これだけか」と思い、やったら終わった達成感。翌日また開く。**これが習慣のトリガーだ。**

**親が課金を絶対に許すトリガー**

母親が「また受験のスマホなの?」と言いかけた時、蓮太郎が「これテスト結果入れたらAIが今日やること教えてくれるやつ。塾行けない分これで補ってる」と言う。月1,980円という価格が塾月謝の10分の1未満。「塾行けない代わりに」という文脈で完全に課金を正当化できる。**親の罪悪感 (塾に行かせてやれない) と Testall を直結させることが鍵だ。**

## §2. WHO (誰に届けるか)

### 一次ターゲット: 塾なし独学高校生 (狂人憑依)

- 全国高校生 約350万人 × 独学受験率 (推測: 約35%) = **約120万人**
- このうち「正しい勉強法が分からず詰まっている層」= 推測で約60-80万人
- 蓮太郎タイプが典型: 進学校在学・部活有・塾なし・志望校ギャップあり

**セグメント別優先度**

| セグメント | 優先度 | 理由 |
|---|---|---|
| 公立進学校・塾なし | 最優先 | 痛みが最大・親の課金意欲高い |
| 地方在住 (県庁所在地以外) | 最優先 | 物理的に予備校アクセスなし = 代替需要が強い |
| 私立中高一貫・自習指向 | 第2優先 | 自走意欲高い・親の教育投資大 |
- 都市圏進学校 | 第3優先 | スタサプ競合あり・塾リッチ層 |

**狂人の定義**: 医学部・旧帝大・難関国立を「塾なしで」「自分の力で」取りに行こうとしている高校生。この層は「使える道具なら何でも使う」。価格感応度が低い。LTVが桁違いになる。

### 二次ターゲット: 親 (課金決済者)

- 子の合格に本気な親
- 塾に行かせられない罪悪感を持っている親
- 「スマホで遊んでいる」から「スマホで勉強している」に変えるだけで許容する
- 月1,980円は塾のコンペアラブル (月3万〜7万) と比べれば誤差

### 三次ターゲット: 浪人生 (推測: 約10万人)

- 推測: 浪人生は動機が強烈・課金意欲も高い
- ただし浪人生は予備校に行っている比率が高いため、第3優先

## §3. WHAT (何の価値を売るのか)

### 機能価値 (Features)

1. テスト写真撮影 → Claude Vision で画像認識 → 弱点抽出
2. AI診断 (今日やる45分タスク 1つだけ自動生成)
3. 2,888冊参考書DB (どの参考書のどのページをやるかまで具体指示)
4. 週次計画 / 進捗可視化 / PWA対応 (ホーム画面追加)

### 情緒価値 (Emotional Value)

- 「今日やることが分かる」 → 迷いが消える安堵
- 「塾なしでも戦える」 → 孤独な受験生の自己肯定感
- 「一人じゃない感」 → AIが毎日見ていてくれる感覚
- 「仕組みが自分を動かしてくれる」 → 意志力に頼らない解放感

### 自己実現価値 (Self-Actualization Value)

- 「自分の力で受かった」という物語の構築
- 「親に塾代をかけさせなかった」という誇り
- 「AI時代の最初の受験生」という先進性

### WHAT 4要素 (森岡式)

| 要素 | Testall の具体 |
|---|---|
| **1. 信頼できる品質** | Claude Vision + 2,888冊参考書DB + Supabase Auth + Sentry監視 の技術基盤 |
| **2. 効果的な学力向上** | テスト返却 → 45分タスク → 実行の一気通貫サイクル。弱点を反復で潰す設計 |
| **3. 継続的な支援** | 毎テスト後に更新されるAI診断 + PWA通知で毎日接触。習慣化まで伴走 |
| **4. 独自価値 (大井が作った)** | 「テスト返却もらった後どうすれば良いか分からない」という原体験が設計思想の根本。他社には出せない当事者性 |

### 競合との圧倒的差 3つ

**差1: 「今日やること」が1つだけ出てくる**
スタディサプリは「コンテンツ無限」。atama+は「適応学習」。しかしどちらも「今日何をやればいいか」という意思決定負担を取り除いていない。Testallは「今日の45分タスク1つだけ」を自動生成する。これが最大の差だ。

**差2: テスト結果が起点**
すべての競合は「学習プランを先に立てる」設計だ。Testallは逆だ。テスト返却という「証拠」が起点になる。「実際に間違えた事実」から逆引きするため、学習の必然性が高い。やらされ感ではなく「確かにここ弱い」という納得感が生まれる。

**差3: 地方・塾なし層への包摂**
スタサプ・atama+は塾または都市圏リッチ層にリーチしている。Testallは月1,980円で地方の塾なし高校生が対象だ。物理的アクセス格差を技術で壊す設計になっている。これは単なる差別化ではなく「社会的使命」として語れる。

## §4. HOW (どう届けるか)

### 価格帯とブランドの格 (クラス)

- **月額 ¥1,980 (スタンダード)** を主力にする
  - 理由: ¥980 は「安すぎて信頼されない」。¥1,980 は塾月謝の25分の1。「これで十分変わる」という期待値を保てる
  - 推測: 高校生の月のお小遣い平均は5,000-10,000円。1,980円は痛い金額ではあるが「受験のため」なら許容範囲
  - 親課金の場合: 月1,980円 = 塾の「1コマ分にも満たない」というフレームで正当化
- **無料プラン (テスト3回・30日)** = 入口
- **プレミアム ¥4,980** (保護者ダッシュボード + 週次AIレポート付き) = 将来追加

### 接触戦略 (HOWのタッチポイント設計)

各タッチポイントは全部「今日やることが分かる + 塾なしで戦える」というWHATを脳内に積む方向にそろえる。

| チャネル | 具体施策 | WHATを積む方法 |
|---|---|---|
| X (Twitter) | 大井自身が「模試返却→Testall診断→45分でここまで変わった」投稿 週3本 | 当事者性+効果実証 |
| TikTok | 「テスト返ってきた直後に撮影→AI診断→今日のタスク」15秒動画 | 体験のビフォーアフター可視化 |
| App Store ASO | キーワード: 受験勉強/塾なし/AI診断/テスト分析 / キャッチ:「テスト結果を、今日の45分に変える。」 | 認知から信頼への橋渡し |
| 口コミ設計 | 「合格した先輩がTestall使ってた」という物語を意図的に仕込む (β合格者の声) | 信頼増幅 |
| 親向け | 子供のTestall利用時に「親ダッシュボード」通知 → 親が学習状況を把握できる安心感 | 保護者WHS確立 |

### 体験設計 (最初の7日間が全て)

```
Day 1: アプリDL → テスト写真撮影 → AI診断 (3分) → 今日の45分タスク提示
Day 2: 前日タスク完了確認 → 次のタスク更新
Day 3: 「3日連続でした」通知 + 弱点変化グラフ表示
Day 7: 「1週間続いた」達成感演出 → 有料プランへの自然な誘導
```

**鍵は Day 1 の体験だ。** 「写真を撮ったら今日やることが出てきた」この体験が全てを決める。初回45分が終わった瞬間、解約率は激減する。

## §5. ブランドの「狂気」(森岡フレーム特有)

### Testallの狂気の入り口

大井湧瑛の原体験はこれだ: **「テスト返却もらった後、どうすれば良いか全然分からなかった」**

これは多くの受験生の「あるある」に見えるが、大井にとっては単なる不便ではない。通信制高校を選び、SFC AO受験に挑んでいる大井自身が「日本の標準教育レールから外れた人間」として感じてきた孤独と、「正しい道を誰も教えてくれない」という怒りから来ている。

Testallはその怒りの産物だ。

「標準レールに乗れた人間が塾で整った道を歩む間、レールの外にいる人間には誰も45分の使い方を教えてくれなかった。だから俺が作った。」

### 17歳代表のリスク + メリット

| リスク | 対処 |
|---|---|
| 「まだ受験も終わってないのに」という懐疑 | SFC AO受験中という事実 + プロダクト完成度 (v0.6) + 2,888冊DB が証拠になる |
| 継続性への不安 (飽きないか) | 法人化済み + 株式会社の代表 + 別売上 (AIpaX 4社契約) がある事実で払拭 |

| メリット | 使い方 |

| 「俺自身がユーザーゼロ号」という説得力 | 自分の模試結果をTestallに入れてX投稿する |

### ストーリーラインのキャッチ (3案)

**案A**: 「17歳が、日本の受験戦争を終わらせる」

## §6. ブランドエクイティ設計 (5階層)

### 1. 使命 (Mission)

**「テスト返却という現実から、今日の45分を作り出す。全ての高校生に、正しい努力の方法を。」**

解説: 塾や予備校にアクセスできない高校生が、テストという「実際の証拠」をもとにAIと共に最短ルートで学べる環境を作ること。勉強の才能や親の収入に左右されず、「仕組み」で次の一手が分かる世界を実現する。

### 2. 約束 (Promise)

**高校生への約束**: 「今日やることが、1つだけ、分かる。」
迷わなくていい。意志力に頼らなくていい。テスト結果を入れれば、今日の45分が出てくる。それだけでいい。

**親への約束**: 「子供が今日何をやったか、見える。月1,980円で、塾の代わりに毎日動く仕組みが手に入る。」
子供の学習状況を可視化することで、「スマホで遊んでいる不安」を「勉強している安心」に変える。

### 3. 個性 (Personality) — ブランドの性格 5つ

1. **正直 (Honest)**: 「これをやれば受かります」とは言わない。「今日やることを1つ出します」と言う。過剰な約束はしない
2. **断言する (Decisive)**: 「どれかやってみては?」ではなく「今日はこれをやれ」と言い切る。受験生に選択肢を与えない
3. **共闘する (Ally)**: 上から教えるのではなく、同じ受験生の立場で「一緒にやる」姿勢
4. **シンプル (Simple)**: テスト結果を入れたら出てくる。それだけ。余計な機能で迷わせない
5. **本気 (Intense)**: 大井自身が現役受験生であり、Testallで自分の模試を診断している。遊びではない

### 4. 物語 (Story) — 200字

テスト返却日、俺は毎回途方に暮れた。赤ペンで直したはいいが、明日何をやればいいかが分からない。塾の友達は先生に聞ける。俺には誰もいない。その夜に「今日やること1つ」を出すプロダクトを作り始めた。Testallは怒りから生まれた。日本中の「誰も教えてくれなかった高校生」の隣に、AIを置く。今日やる45分が、3年後を変える。

### 5. シンボル (Symbol) — ロゴ・色に込める意味

| 要素 | 方向性 | 込める意味 |
|---|---|---|
| ロゴイメージ | テスト用紙が光を放つ / 採点されたXが矢印に変わる | 「失点という現実が、行動に変わる」変換の瞬間 |
| メインカラー | **深い紺 × 電光ブルー** | 夜中の自習 (深い紺) × AI診断の即答性 (電光ブルー) |
| サブカラー | **白** | シンプル・クリア・迷いのなさ |
| フォント | 太いゴシック | 断言する性格を視覚的に体現 |
| NGカラー | 蛍光ピンク・レッド | 受験プレッシャーを煽る色は使わない。Testallは解放する側だから |

## §7. ピッチパッション設計 (「日本の教育を変える」レベル)

### ピッチで使うべき強烈な一行 5案

**A案**: 「テスト返却後、俺には誰も教えてくれなかった。だから17歳で作った。」
→ 原体験から直結。聴衆が「はっ」とする。ハートオープニング型。

**B案**: 「日本の受験生70万人が、今夜、何をすればいいか分からないまま寝る。」
→ 社会問題の規模を数字で叩き込む。マインドオープニング型。

**C案**: 「塾のある子と塾のない子で、合格率に6倍の差がある。Testallはその差を潰す。」
→ 推測ベースだが、教育格差の不公平をデータで語る。投資家に刺さる。

**D案**: 「月3万の塾か、月2千円のAIか。どっちが受験生の現実に近いか、俺には分かっている。」
→ 価格差を使った挑発。「俺には分かっている」という当事者宣言。

**E案**: 「日本の標準教育が見捨てた70万人の代わりに、俺のAIが毎晩45分を設計する。」
→ スケールと使命を同時に宣言。「見捨てた」という言葉が刺さる。

### スライド1枚目の「敵」設定 3案

**敵A: 情報格差**
「テスト返却後の1時間、塾に通う子供は先生に聞ける。通えない子は積読の参考書を眺める。この差が3年間積み重なる。」

**敵B: 意志力への依存**
「『やる気を出せ』『計画を立てろ』。既存の学習アプリは全部、高校生の意志力に丸投げしてきた。意志力は枯渇する。仕組みは枯渇しない。」

**敵C: 教育市場の構造**
「日本の受験産業は2,160億円規模。しかしその恩恵を受けるのは、月3万円以上を出せる家庭だけだ。残りの70万人は手付かずのまま放置されてきた。」

### 「大井がやる必然性」を3行で

1. 大井湧瑛は通信制高校在学の17歳であり、現在SFC AO受験中の現役受験生だ。「テスト返却後どうすればいいか分からない」という体験を今まさに生きている。
2. 日本の標準教育レール (偏差値序列・一斉授業・塾依存) の外側から見ているからこそ、レール上の問題を構造として見抜ける。
3. Testallは「使えるから使う」ではなく「大井自身が必要だったから作った」プロダクトだ。創業者とプロダクトが同一の痛みを持つ、これ以上のFounder-Market Fitはない。

### 「日本教育の標準レール」が誰を見殺しにしているか

日本の公教育とその後の受験産業は暗黙の前提を持っている。「必要な情報は塾で手に入れられる」「意欲がある子は計画を自分で立てられる」「テストで間違えた後の行動は自分で考えられる」。これらは全部嘘だ。

実際には:
- 地方在住・経済的理由・部活・家庭環境で塾に行けない高校生が数十万人いる
- テスト返却後の「次の一手」を教える教師はほぼいない (教員は授業をする人間であって受験戦略家ではない)
- 「参考書を買え」「計画を立てろ」は意志力への丸投げであり、構造的解決策ではない

**推測: 大井自身が通信制高校を選んだことで、この構造をより鮮明に体感している。** SFC AO受験という非標準ルートを選び、自ら情報を取りに行く経験が、Testallの「情報の非対称性を壊す」という設計思想に直結している。

### 聴衆別に刺さるメッセージ

**投資家向け**

## §8. 勝ち筋分析

### 既存教育SaaSが取れていない領域

**スタディサプリ**: 動画コンテンツのラインナップは最大級。しかし「今日何をやるか」の意思決定は全部ユーザーに委ねている。コンテンツ提供者であって戦略家ではない。

**atama+**: AIによる適応学習を実現しているが、塾と学校向けの B2B2C 構造。個人高校生が月1,980円で使える製品ではない。価格と販路が構造的に異なる。

**Studyplus Planning**: 学習記録 + 計画管理は持っているが、起点が「自分で計画を立てる」こと。テスト結果という「証拠」から逆引きする設計はない。

**モノグサ**: 記憶定着 (リテンション系) に特化。テスト診断から今日のタスクを出すOSではない。

**取れていない領域の核心**:
「テスト返却 → 今日の45分 → 実行」の一気通貫を、月2千円以下で個人向けに提供しているプロダクトは存在しない。

### Testallが構造的に勝つ理由 3つ

**理由1: AI診断の起点が「テスト結果 (事実)」**
他社が「ユーザーの申告 (学習計画・目標)」を起点にする中、Testallは「テストで間違えた事実」を起点にする。これは「学びの必然性」を担保する。「苦手っぽいからやる」ではなく「昨日間違えたからやる」は動機の強さが桁違いだ。

**理由2: AI世代との同期 (大井が現役高校生)**
2026年の高校1-3年生はAIネイティブ世代だ。「AIに聞く」ことへの心理的抵抗がない。Testallの「AIに診断させる」体験は、この世代には「当然の行動」として受け入れられる。スタサプ創業者は50代。atama+創業者は30代。大井は17歳。この差は埋まらない。

**理由3: 地方・塾なし層の包摂という構造的優位**
静岡という大井の地元が象徴する「地方の塾アクセス格差」は、日本全国に存在する。都市圏の有名教育SaaSは地方に来ていないか、来ていても価格が高い。Testallは月1,980円で全国対応の設計だ。地方で口コミが発火すれば、競合が同じ価格帯でターゲットを変えるには数年かかる。

### 競合に絶対真似できない武器

**武器1: 大井自身の物語**
「17歳・通信制高校・SFC AO受験中・元スタートアップ経験者・自分のテスト結果をTestallに入れている」という事実の束は、他社が何億円かけても再現できない。この物語が「当事者が作った」という信頼を作る。

**武器2: 「テスト返却もらった後どうすれば良いか分からなかった」という痛みの当事者性**
これはFounder-Market Fitの最深層だ。大井が解決したい問題は「自分が今まさに困っている問題」だ。競合のプロダクトは「ユーザーリサーチで分かった問題」を解いている。大井は「自分の問題」を解いている。この差が設計の精度に出る。

**武器3: 大井株式会社という非典型のブランド**
17歳の代表が持つ「社会への反骨」「日本の標準から外れた視点」「ぎりぎりの実体験」は、大手教育SaaSが「共感できる話」として語れるものではない。Testallは制度の外側にいる人間が、制度の外側の人間のために作ったプロダクトだ。

### 月収10億シナリオの蓋然性 (3年後)

月額 ¥1,980 × 50,000人 = 月 **約9,900万円 (月10億手前)**

**シナリオの蓋然性評価**:

| マイルストーン | 期間 | 想定ユーザー数 | 根拠 |
|---|---|---|---|
| βリリース | 2026年6月 | 100-500名 | X発信 + 口コミ |
| 口コミ拡散 | 2026年夏〜秋 (受験シーズン) | 1,000-5,000名 | 受験前の高い動機 |
| メディア露出 (ビジコン等) | 2026年末〜2027年 | 5,000-20,000名 | 17歳代表という希少性 |
| 垂直成長 | 2027年〜2028年 | 20,000-50,000名 | 合格実績 + 継続口コミ |

推測: 月50,000人の有料ユーザーは「全国の塾なし高校生120万人の約4%」だ。4%のシェアは、教育SaaS市場で現実的に狙える数字だ。ただし以下が前提条件:
1. Stripe 課金実装 (現在最優先残課題) の早期完了
2. βユーザーからの合格実績の確保 (最初の「証拠」が全てを決める)
3. 大井自身のSFC合格 (最大の実証) — 推測: これが達成されれば「Testall使って合格した」は最強のマーケティング素材になる

## §9. 教育系ビジネスの落とし穴と回避

### 教育系で死にやすいパターン

**パターン1: 受験期だけビジネス (季節性の罠)**
3月に全受験生が合格or不合格が確定する → 一斉解約。12月から2月は高需要だが3月〜6月はチャーン地獄。

**Testallの回避策**:
- 定期テスト対策モード (年6回・高1-2年向け) を追加してシーズナリティを分散
- 高1-2年の「受験準備層」を早期に取り込む設計
- 推測: 高1から使い始めれば平均LTV = 2年 × 12ヶ月 × 1,980円 = 約47,000円

**パターン2: 親が決定権を持つBtoB2C構造 (子の熱狂が親を超えない)**
子どもがTestallを欲しがっても、親が「また無駄なアプリ?」と遮断する。課金導線で詰む。

**Testallの回避策**:
- 親向けダッシュボード (子の学習状況可視化) を早期実装
- 「塾代の25分の1」というフレーミングを親向けコミュニケーションの前面に出す
- 推測: 無料プランで子が使い始め → 親がダッシュボードを見て価値を認識 → 親が課金という逆転フローが最強

**パターン3: 成果が見えにくい (Churnの本質的原因)**
「Testallのおかげで成績が上がった」因果関係が見えにくい。解約する言い訳が「効果が分からないから」になる。

**Testallの回避策**:
- 弱点スコアの変化グラフを毎テストで可視化
- 「3ヶ月前と今の弱点比較」を自動生成して見せる
- ベータユーザーの合格体験談を早期収集し、エビデンスとして使う

**パターン4: 卒業即離脱 (大学入学 = 解約)**
高3の3月に入試終了 → 全員退会。ゼロからの再獲得が毎年必要。

**Testallの回避策**:
- 大学生版 (TOEIC/資格試験/定期試験向け) への自然な移行設計を中期的に検討
- 現在は高校生市場に集中で良い。大学生市場は別プロダクトで後追い

## 大井への一言 (森岡から)

お前がTestallで一番やらかしてる間違いはこれだ。「機能を売ろうとしている」。

お前自身がその高校生だったはずだ。それを最前面に出せ。「17歳の俺が、同じ高校生のために作った」が全ての説明を不要にする。

プロダクトの完成度は十分だ。Stripe課金を今週中に動かして、βユーザーを取り始めろ。最初の5人の体験談が、後の50万人に勝る。
