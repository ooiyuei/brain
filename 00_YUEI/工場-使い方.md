# 🏭 工場（OpenClaw）を Mac から動かす — 使い方

> **大井へ。外出先の MacBook の Obsidian で `00_YUEI/工場.md` に1行書くだけで、自宅 Windows の工場（OpenClaw）が動く。**
> Claude を開いてなくてOK。Claudeが寝てても工場は回る。

---

## 仕組み（3行）

1. **Mac で `工場.md` に「- やること」を書く** → obsidian-git が GitHub に push
2. **Windows が4分おきに git pull**（`BrainGitSync`・Obsidian閉じててもOK）
3. **Windows が3分おきに `工場.md` を読んで OpenClaw に投入**（`BrainFactoryBridge`）→ 成果物は `wiki/_inbox/{部署}/` に届く・`工場-受付.md` にレシート

```
[Mac Obsidian] --git push--> GitHub --git pull--> [Windows]
   工場.md に書く                                  factory_bridge が読む
                                                        ↓
                                               OpenClaw(qwen) が生成
                                                        ↓
                                            wiki/_inbox/{部署}/ に成果物
```

---

## 書き方

`工場.md` の「## 投入」の下に箇条書きするだけ：

```
- AIpaXのクロージング想定問答を10個
- Testallの受験あるあるX投稿20本 @marketing !late
- 静岡の助成金を5件まとめて @research !high
```

**タグは任意**（書かなければ newbiz・通常で流れる）:
- `@部署` … research / newbiz / dev / marketing / corp / secretary / misc
- `!優先度` … `!high`=最優先・じっくり / `!normal` / `!late`=軽量・後回しでOK

**一度投入した行は二度と再投入されない**（ハッシュ管理）。消しても残してもOK。

---

## 確認したいとき（Mac から）

- **投入されたか** → `工場-受付.md`（レシートが出る）
- **成果物** → `wiki/_inbox/{部署}/` の新しいファイル
- **工場が生きてるか** → `wiki/live-status.md`（1分ごと更新）

---

## ⚠️ Mac 側で1回だけ確認してほしいこと（これだけ）

このループは **Mac の obsidian-git が「自動 commit + 自動 push」になってること** が前提。
Obsidian（Mac）→ 設定 → Obsidian Git:
- **Auto commit-and-sync interval** を 1〜5 分に
- **Push on commit-and-sync** = ON

これが OFF だと、Mac で書いても GitHub に飛ばず工場に届かない（＝今までの「分断」の原因候補）。

---

## 復旧・分断防止（自動で効く）

| 仕掛け | 頻度 | 役割 |
|---|---|---|
| `BrainGitSync` | 4分 | Mac↔Win を git で同期（Obsidian閉じてても動く） |
| `BrainFactoryBridge` | 3分 | 工場.md → OpenClaw 投入 |
| `BrainQueueGuard` | 10分 | worker 死亡時に自動復活・queue膨張防止 |
| 競合時 | — | 自動マージせず安全に中断 → Discord通知（Claudeが手で直す） |

再起動後も全部 `StartWhenAvailable` で自動復帰する。
